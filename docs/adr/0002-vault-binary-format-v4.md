# ADR-0002: Vaultバイナリフォーマット v4
Status: accepted
Date: 2026-04-03

## Context
vault.pqdは `strings` コマンドでフィールド名・アルゴリズム名が一切露出しない設計が必要。将来の添付ファイル対応(Phase 2)とHQCハイブリッド(Phase 4)に備える。

## Decision
**カスタムバイナリフォーマットを自前実装。** シリアライズクレート不使用。

構造: マジックバイト(8B) → スキーマVer(1B) → KDFソルト(32B) → Legacyソルト(32B) → 検証トークン → PQC鍵ブロック → エントリセクション → ランダムパディング(512-4096B)

エントリレコードに以下を予約:
- legacyフラグ + legacy鍵ブロック (Phase 2 デジタル遺言用、Phase 1では0初期化)
- 添付ファイルカウント(u16) + 添付ブロックオフセット(u64) (Phase 2用、Phase 1では0初期化)

## Consequences
- シリアライズの自前実装にバグリスクがあるが、フォーマットを完全制御できる
- v4→v5マイグレーション時(HQC追加)もヘッダのスキーマバージョン切り替えで対応可能
- 添付ファイル対応時にフォーマットマイグレーション不要

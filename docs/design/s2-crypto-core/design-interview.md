## Q1: 検証トークンによるパスワード正当性チェックのスコープ

**質問**: 検証トークン（パスワード正当性チェック）はS2に含めるか、後続スプリントに回すか？

**回答**: S2に含める。

**詳細**: ダミートークン方式で実装する。32バイトのランダム平文をAES-256-GCMで暗号化し、vault.pqdヘッダに格納する。unlock時にマスターキーで復号し、元の平文と一致するか確認する簡易実装とする。

**信頼性**: 🔵 確定

## Q2: 暗号モジュールの構造

**質問**: crypto.rs を単一ファイルで拡張するか、サブモジュールに分割するか？

**回答**: サブモジュール分割。

**詳細**: S1で存在するcrypto.rsを以下のサブモジュール構成に展開する。

| モジュール | 責務 |
|-----------|------|
| crypto/mod.rs | CryptoEngine実装、公開API |
| crypto/secure_mem.rs | SecureBuffer, ZeroizingKey, MasterKey (S1から移動) |
| crypto/kdf.rs | Argon2id鍵導出、パラメータ検証 |
| crypto/aead.rs | AES-256-GCM暗号化/復号 |
| crypto/kem.rs | ML-KEM-768カプセル化/脱カプセル化 |
| crypto/dsa.rs | ML-DSA-65署名/検証 |
| crypto/hmac_util.rs | HMAC-SHA256 |

**信頼性**: 🔵 確定

## REQ-001〜004: todayコマンド 🔵

**信頼性**: 🔵 *バックログS5 + ヒアリングQ1, Q6より*

### Given（前提条件）
- Vaultが初期化済みで解錠可能
- パスワードが提供可能

### When（実行条件）
- ユーザーが `pq-diary today` を実行

### Then（期待結果）
- 当日の `YYYY-MM-DD` タイトルのエントリが存在すれば$EDITORで開く
- 存在しなければ新規作成して$EDITORで開く
- 新規作成時、「daily」テンプレートがあれば変数展開後の内容を初期本文とする

### テストケース

#### 正常系

- [ ] **TC-001-01**: 当日エントリが存在しない場合、新規作成して$EDITORで開く 🔵
  - **入力**: エントリなしのVault + `today` コマンド
  - **期待結果**: `YYYY-MM-DD` タイトルのエントリが作成され、edit相当の動作

- [ ] **TC-001-02**: 当日エントリが既に存在する場合、既存を開く 🔵
  - **入力**: `2026-04-05` タイトルのエントリがあるVault + `today` コマンド
  - **期待結果**: 既存エントリを$EDITORで開く（新規作成しない）

- [ ] **TC-001-03**: 「daily」テンプレートが存在する場合、新規作成時に適用 🔵
  - **入力**: `daily` テンプレートが登録済み + エントリなし + `today`
  - **期待結果**: テンプレート内容（変数展開済み）が初期本文として設定される

- [ ] **TC-001-04**: 「daily」テンプレートが存在しない場合、空本文で作成 🔵
  - **入力**: テンプレートなし + エントリなし + `today`
  - **期待結果**: 空本文のエントリが作成される

- [ ] **TC-001-05**: `-v VAULT` オプションで別Vaultを指定 🟡
  - **入力**: `today -v work`
  - **期待結果**: work Vaultに対して操作が実行される

#### 異常系

- [ ] **TC-001-E01**: Vaultが未初期化の場合 🔵
  - **入力**: 初期化されていないパス + `today`
  - **期待結果**: `DiaryError::VaultNotFound` エラー

- [ ] **TC-001-E02**: パスワードが不正の場合 🔵
  - **入力**: 不正なパスワード + `today`
  - **期待結果**: `DiaryError::Crypto` エラー（パスワード不正）

## REQ-101〜105: テンプレートCRUD 🔵

**信頼性**: 🔵 *ADR-0005 + バックログS5より*

### Given（前提条件）
- Vaultが初期化済みで解錠可能

### When（実行条件）
- ユーザーがtemplate add/list/show/deleteを実行

### Then（期待結果）
- テンプレートがvault.pqd内に暗号化CRUD操作される

### テストケース

#### 正常系

- [ ] **TC-101-01**: template add でテンプレートを作成 🔵
  - **入力**: `template add meeting-notes` + エディタでコンテンツ入力
  - **期待結果**: vault.pqd内にRECORD_TYPE_TEMPLATE(0x02)のレコードが追加される

- [ ] **TC-101-02**: template list で一覧表示 🔵
  - **入力**: テンプレートが3つ登録済み + `template list`
  - **期待結果**: 名前のアルファベット順で3件表示

- [ ] **TC-101-03**: template show で内容表示 🔵
  - **入力**: `template show meeting-notes`
  - **期待結果**: 復号されたテンプレート内容が表示される

- [ ] **TC-101-04**: template delete で確認付き削除 🟡
  - **入力**: `template delete meeting-notes` + 確認にyes
  - **期待結果**: テンプレートがvault.pqdから削除される

- [ ] **TC-101-05**: TemplatePlaintext の暗号化・復号ラウンドトリップ 🔵
  - **入力**: テンプレートを作成して復号
  - **期待結果**: 元の内容と一致

#### 異常系

- [ ] **TC-101-E01**: 同名テンプレートの重複作成 🟡
  - **入力**: 既存の `daily` テンプレート + `template add daily`
  - **期待結果**: 上書き確認プロンプト表示

- [ ] **TC-101-E02**: 存在しないテンプレートのshow 🟡
  - **入力**: `template show nonexistent`
  - **期待結果**: `DiaryError::TemplateNotFound` エラー

- [ ] **TC-101-E03**: 存在しないテンプレートのdelete 🟡
  - **入力**: `template delete nonexistent`
  - **期待結果**: `DiaryError::TemplateNotFound` エラー

#### 境界値

- [ ] **TC-101-B01**: テンプレート名の最大長（128文字） 🟡
  - **入力**: 128文字のテンプレート名
  - **期待結果**: 正常に作成

- [ ] **TC-101-B02**: テンプレート名の最大長超過（129文字） 🟡
  - **入力**: 129文字のテンプレート名
  - **期待結果**: `DiaryError::InvalidArgument` エラー

- [ ] **TC-101-B03**: 空本文テンプレートの作成 🟡
  - **入��**: `template add empty` + 空の内容
  - **期待結果**: 正常に作成される

## REQ-201〜203, 211: リンク解決とバックリンク 🔵

**信頼性**: 🔵 *ADR-0004 + バックログS5 + ヒアリングQ3, Q5より*

### Given（前提条件）
- Vaultに複数のエントリが存在
- unlock時にバックリンクインデックスが構築済み

### When（実行条件）
- ユーザーが `show <id>` を実行

### Then（期待結果）
- 本文中の `[[タイトル]]` が解決情報付きで表示される
- バックリンクが末尾セクションに表示される

### テストケース

#### 正常系

- [ ] **TC-201-01**: 単一リンクの解決 🔵
  - **入力**: エントリAの本文に `[[エントリB]]` がある + `show <A>`
  - **期待結果**: `[[エントリB]]` の横にエントリBのUUIDプレフィックスが表示される

- [ ] **TC-201-02**: バックリンク表示 🔵
  - **入力**: エントリAの本文に `[[エントリB]]` がある + `show <B>`
  - **期待結果**: `--- Backlinks ---` セクションにエントリAが表示される

- [ ] **TC-201-03**: 複数リンクの解決 🔵
  - **入力**: 本文に `[[A]] と [[B]] を参照` + show
  - **期待結果**: 両方のリンクが解決される

- [ ] **TC-201-04**: 同名タイトル重複時の全候補表示 🔵
  - **入力**: 「メモ」タイトルのエントリが2つ + 本文に `[[メモ]]` + show
  - **期待結果**: 2件の候補がリスト表示される

- [ ] **TC-201-05**: バックリンクインデックスのunlock時構築 🔵
  - **入力**: 相互リンクを持つ3エントリ
  - **期待結果**: unlock後にインデックスが構築され、show時にバックリンクが正確に表示

#### 異常系

- [ ] **TC-201-E01**: リンク先が存在しない場合 🟡
  - **入力**: 本文に `[[存在しないエントリ]]` + show
  - **期待結果**: `[[存在しないエントリ]] (未解決)` と表示

- [ ] **TC-201-E02**: 空リンク `[[]]` の処理 🟡
  - **入力**: 本文に `[[]]` + show
  - **期待結果**: リンクとして解釈せず、そのまま表示

#### 境界値

- [ ] **TC-201-B01**: リンクなしのエントリ 🔵
  - **入力**: `[[` を含まないエントリ + show
  - **期待結果**: バックリンクセクションは空（またはセクション自体を非表示）

- [ ] **TC-201-B02**: バックリンクなしのエントリ 🔵
  - **入力**: どこからも参照されていないエントリ + show
  - **期待結果**: バックリンクセクションは空

## REQ-301〜303: vim補完関数 🔵

**信頼性**: 🔵 *ADR-0004 + バックログS5より*

### Given（前提条件）
- $EDITORがvimまたはnvim
- Vaultにエントリが存在する

### When（実行条件）
- エディタ内で `[[` を入力後に補完をトリガー

### Then（期待結果）
- タイトル候補がポップアップ表示される

### テストケース

#### 正常系

- [ ] **TC-301-01**: vim起動時に補完関数が注入される 🔵
  - **入力**: vim/nvimでedit実行
  - **期待結果**: `-c` オプションで補完関数セットアップコマンドが注入される

- [ ] **TC-301-02**: 補完用タイトル一覧ファイルの生成 🔵
  - **入力**: エントリが5件あるVault + edit実行
  - **期待結果**: secure_tmpdir上に5件のタイトルを含む一時ファイルが作成される

- [ ] **TC-301-03**: エディタ終了後に一時ファイルがzeroize削除される 🔵
  - **入力**: edit実行 → エディタ終了
  - **期待結果**: 補完用一時ファイルが存在しない（zeroize済み）

- [ ] **TC-301-04**: 補完用ファイルにはタイトルのみ含まれる 🔵
  - **入力**: エントリ（タイトル・本文・タグあり）が存在
  - **期待結果**: 一時ファイルにはタイトルのみ（本文・タグ情報なし）

#### 異常系

- [ ] **TC-301-E01**: vim/nvim以外のエディタの場合 🔵
  - **入力**: `$EDITOR=nano` + edit実行
  - **期待結果**: 補完関数を注入せず、通常の編集として動作

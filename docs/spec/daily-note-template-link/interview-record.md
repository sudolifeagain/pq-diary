## 質問と回答

### Q1: todayコマンドで同日に複数エントリがある場合の挙動

**質問日時**: 2026-04-05
**カテゴリ**: ���定義部分詳細化
**背景**: バックログには「YYYY-MM-DDエントリ自動生成/開く」とあるが、同日に既にエントリがある場合の挙動が未定義。

**回答**: **既存を開く** — 同日タイトルのエントリがあればそれを開く。なければ「YYYY-MM-DD」タイトルで新規作成。

**信頼性への影響**:

### Q2: テンプレートに変数展開は必要か

**質問日時**: 2026-04-05
**カテゴリ**: 未定義部分詳細化
**背景**: ADR-0005ではテンプレートの保存方式のみ決定。テンプレート内容の動的展開機能については未定義。

**回答**: **拡張変数** — カスタム変数定義もサポート。ユーザーが変数を定義可能。

**信頼性への影響**:
- テンプレート変数展開が新規要件として追加（REQ-101〜REQ-105）
- 基本変数 + カスタム変数の両方をサポートする設計が必要

## ヒアリング結果サマリー

### 確認できた事項
- todayコマンドは同日エントリの再利用方式（新規作成ではない）
- テンプレートはカスタム変数定義をサポート（テンプレート内埋め込み）
- バックリンクインデックスはunlock時にインメモリ構築
- リンク重複時は全候補リスト表示
- todayコマンドはdailyテンプレートと自動連携
- 全機能がMust Have

### 追加/変更要件
- テンプレート変数展開（基本変数 + カスタム変数）が新規要件として追加
- todayコマンドとdailyテンプレートの自動連携が追加

### 残課題
- カスタム変数のバリデーションルール（変数名に使える文字等）→ 設計フェーズで決定
- バックリンクインデックスのメモリ上限 → 設計フェーズで決定

### 信頼性レベル分布

**ヒアリング前**:
- 🔵 青信号: 3件
- 🟡 黄信号: 8件
- 🔴 赤信号: 2件

**ヒアリング後**:
- 🔵 青信号: 10件 (+7)
- 🟡 黄信号: 4件 (-4)
- 🔴 赤信号: 0件 (-2)

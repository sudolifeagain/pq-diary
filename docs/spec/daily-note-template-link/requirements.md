# daily-note-template-link 要件定義書

## 概要

Sprint 5「デイリーノート + テンプレート + リンク」の要件定義書。todayコマンドによるデイリーエントリの自動生成/表示、テンプレートの暗号化CRUD、`[[タイトル]]` リンク解決とバックリンク表示、$EDITOR内でのvimカスタム補完関数を定義する。

## 機能要件（EARS記法）

**【信頼性レベル凡例】**:
- 🔵 **青信号**: PRD・EARS要件定義書・設計文書・ユーザヒアリングを参考にした確実な要件
- 🟡 **黄信号**: PRD・EARS要件定義書・設計文書・ユーザヒアリングから妥当な推測による要件
- 🔴 **赤信号**: PRD・EARS要件定義書・設計文書・ユーザヒアリングにない推測による要件

### 通常要件 — todayコマンド

- **REQ-001**: `today` コマンド実行時、システムは当日の日付 `YYYY-MM-DD` をタイトルとするエントリを検索し、存在すれば `edit` と同様の方式で$EDITORを起動しなければならない 🔵 *バックログS5 + ヒアリングQ1より*

- **REQ-002**: `today` コマンド実行時に当日のエントリが存在しない場合、システムは `YYYY-MM-DD` をタイトルとして新規エントリを作成し、$EDITORを起動しなければならない 🔵 *バックログS5 + ヒアリングQ1より*

- **REQ-003**: `today` コマンドで新規エントリを作成する際、`daily` テンプレートが存在すれば変数展開後の内容を初期本文として使用しなければならない。存在しなければ空本文とする 🔵 *ヒアリングQ6より*

- **REQ-004**: `today` コマンドは `-v VAULT` オプションに対応し、指定がなければデフォルトVaultを使用しなければならない 🟡 *既存CLIパターンから妥当な推測*

### 通常要件 — テンプレートCRUD

- **REQ-101**: `template add <name>` 実行時、システムは$EDITORを起動してテンプレート内容を入力させ、vault.pqd内に `RECORD_TYPE_TEMPLATE (0x02)` として暗号化格納しなければならない 🔵 *ADR-0005 + バックログS5より*

- **REQ-102**: `template list` 実行時、システムはvault.pqd内の全テンプレートレコードを復号し、名前一覧を表示しなければならない 🔵 *バックログS5より*

- **REQ-103**: `template show <name>` 実行時、システムは指定テンプレートを復号し、内容を表示しなければならない 🔵 *バックログS5より*

- **REQ-104**: `template delete <name>` 実行時、システムは確認プロンプトを表示し、承認後にvault.pqdからテンプレートレコードを削除しなければならない 🟡 *deleteコマンドパターンから妥当な推測*

- **REQ-105**: テンプレートの暗号化ペイロードは `TemplatePlaintext` 構造体を `serde_json` でシリアライズし、エントリと同じ `CryptoEngine::encrypt()` で暗号化しなければならない 🔵 *ADR-0005 + 既存暗号化パターンより*

### 条件付き要件 — new --template

- **REQ-111**: `new --template <name>` が指定された場合、システムは対応テンプレートを復号し、変数展開後の内容を初期本文として$EDITORを起動しなければならない 🔵 *バックログS5より*

- **REQ-112**: テンプレート変数展開時、`{{date}}`, `{{datetime}}`, `{{title}}` はシステムが自動で値を設定しなければならない 🔵 *ヒアリングQ2より*

- **REQ-113**: テンプレート変数展開時、カスタム変数 `{{var_name}}` が存在する場合、システムはユーザーにプロンプトで値の入力を求めなければならない 🔵 *ヒアリングQ2, Q4より*

- **REQ-114**: テンプレート内にカスタム変数がない場合、プロンプトを表示せずに展開を完了しなければならない 🟡 *ヒアリングQ4から妥当な推測*

### 通常要件 — リンク解決

- **REQ-201**: `show` コマンド実行時、システムは本文内の `[[タイトル]]` パターンを検出し、対応エントリのUUIDプレフィックスとタイトルを付加して表示しなければならない 🔵 *ADR-0004 + バックログS5より*

- **REQ-202**: `show` コマンド実行時、システムは表示中のエントリへのバックリンク（他エントリから `[[このタイトル]]` で参照されているもの）を末尾にリスト表示しなければならない 🔵 *バックログS5より*

- **REQ-203**: バックリンクインデックスはVault解錠時に全エントリをスキャンしてインメモリで構築しなければならない 🔵 *ヒアリングQ3より*

### 条件付き要件 — リンク重複

- **REQ-211**: `[[タイトル]]` 解決時に同名タイトルのエントリが複数存在する場合、システムは全候補をリスト表示しなければならない 🔵 *ADR-0004 + ヒアリングQ5より*

### 通常要件 — vim補完関数

- **REQ-301**: `edit` / `new` / `today` で$EDITORがvimまたはnvimの場合、システムは `[[` 入力時にタイトル候補を表示するカスタム補完関数を注入しなければならない 🔵 *ADR-0004 + バックログS5より*

- **REQ-302**: 補完用タイトル一覧は secure_tmpdir 上の一時ファイルに書き出し、エディタ終了後にzeroize削除しなければならない 🔵 *ADR-0004より*

- **REQ-303**: vim/neovim以外のエディタの場合、補完機能は提供せず手動入力にフォールバックしなければならない 🔵 *ADR-0004より*

### 制約要件

- **REQ-401**: テンプレート内容はvault.pqd内に暗号化格納しなければならない。平文での外部ファイル保存は禁止 🔵 *ADR-0005より*

- **REQ-402**: 補完用一時ファイルは `/dev/shm` (Linux) または secure_tmpdir (Windows) に配置し、エディタ終了後にzeroize削除しなければならない 🔵 *ADR-0004 + CLAUDE.mdセキュリティ規約より*

- **REQ-403**: バックリンクインデックスは秘密データ（タイトル一覧）を含むため、lock時にzeroize消去しなければならない 🟡 *CLAUDE.mdセキュリティ規約から妥当な推測*

- **REQ-404**: テンプレートCRUD関数は `core/` に実装し、CLI固有ロジック（$EDITOR起動・プロンプト）は `cli/` に配置しなければならない 🔵 *CLAUDE.md設計原則より*

## 非機能要件

### パフォーマンス

- **NFR-001**: バックリンクインデックス構築は100エントリで200ms以内に完了しなければならない 🟡 *既存パフォーマンス要件（list<500ms）から妥当な推測*

- **NFR-002**: テンプレート変数展開は10ms以内に完了しなければならない 🟡 *テキスト置換のみの処理から妥当な推測*

### セキュリティ

- **NFR-101**: テンプレートの暗号化・復号はエントリと同一の暗号化パイプライン（AES-256-GCM + ML-DSA-65署名 + HMAC-SHA256）を使用しなければならない 🔵 *ADR-0005 + 既存暗号化パターンより*

- **NFR-102**: vim補完用一時ファイルにはエントリタイトルのみを含め、本文やタグ情報を含めてはならない 🔵 *ADR-0004「タイトル候補を表示」より*

### ユーザビリティ

- **NFR-201**: `template list` は名前のアルファベット順でソートして表示しなければならない 🟡 *一般的なCLI慣習から妥当な推測*

- **NFR-202**: バックリンク表示は `show` コマンド出力の末尾に `--- Backlinks ---` セクションとして表示しなければならない 🟡 *Obsidian風UIから妥当な推測*

## Edgeケース

### エラー処理

- **EDGE-001**: `today` 実行時にVaultが未初期化の場合、`DiaryError::VaultNotFound` を返さなければならない 🔵 *既存エラーパターンより*

- **EDGE-002**: `template add <name>` で同名テンプレートが既に存在する場合、上書き確認プロンプトを表示しなければならない 🟡 *一般的なCLI慣習から妥当な推測*

- **EDGE-003**: `new --template <name>` で指定テンプレートが存在しない場合、`DiaryError::TemplateNotFound` を返さなければならない 🟡 *既存エラーパターンから妥当な推測*

- **EDGE-004**: `[[タイトル]]` のタイトルに対応するエントリが存在しない場合、リンクをそのまま表示し、`(未解決)` の注記を付けなければならない 🟡 *Obsidian風挙動から妥当な推測*

- **EDGE-005**: テンプレート変数展開時にユーザーが空文字を入力した場合、変数をそのまま空文字列に置換しなければならない 🟡 *一般的なテンプレートエンジン挙動から妥当な推測*

### 境界値

- **EDGE-101**: テンプレート名は1文字以上・128文字以下とし、範囲外は `DiaryError::InvalidArgument` を返さなければならない 🟡 *一般的なバリデーションから妥当な推測*

- **EDGE-102**: テンプレート本文が空の場合でも正常にテンプレートを作成できなければならない 🟡 *エントリの空本文許容パターンから妥当な推測*

- **EDGE-103**: `[[]]` (空リンク) はリンクとして解釈せず、そのまま表示しなければならない 🟡 *Obsidian互換から妥当な推測*

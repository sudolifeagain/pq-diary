## エピック1: デイリーノート

### ストーリー 1.1: 今日のエントリを素早く開く 🔵

**信頼性**: 🔵 *バックログS5 + ヒアリングQ1, Q6より*

**私は** pq-diary ユーザー **として**
**毎日の日記を `today` コマンド一発で開始したい**
**そうすることで** 日付タイトルの手入力やエントリ検索なしに、すぐ書き始められる

**関連要件**: REQ-001, REQ-002, REQ-003, REQ-004

**詳細シナリオ**:
1. ユーザーが `pq-diary today` を実行する
2. パスワード入力（3段階方式）
3. 当日の `YYYY-MM-DD` タイトルのエントリを検索
4. 存在すれば既存エントリを$EDITORで開く
5. 存在しなければ「daily」テンプレートを適用して新規作成し$EDITORを開く
6. 編集後に保存・暗号化・施錠

**前提条件**:
- Vaultが初期化済み
- パスワードが提供可能

**制約事項**:
- 日付フォーマットは `YYYY-MM-DD` 固定
- テンプレート適用は新規作成時のみ

**優先度**: Must Have

### ストーリー 1.2: dailyテンプレートで日記構造を標準化する 🔵

**信頼性**: 🔵 *ヒアリングQ6より*

**私は** pq-diary ユーザー **として**
**todayコマンドで自動的にdailyテンプレートを適用したい**
**そうすることで** 毎日同じ構造（例: ## 振り返り / ## TODO / ## メモ）で日記を書ける

**関連要件**: REQ-003, REQ-111

**詳細シナリオ**:
1. ユーザーが事前に `pq-diary template add daily` でテンプレートを作成
2. テンプレートに `## {{date}} の記録\n\n## 振り返り\n\n## TODO` 等を入力
3. `pq-diary today` 実行時、テンプレート変数が展開された状態で$EDITORが開く

**前提条件**:
- 「daily」テンプレートがvault内に存在する

**優先度**: Must Have

## エピック2: テンプレート管理

### ストーリー 2.1: テンプレートを作成・管理する 🔵

**信頼性**: 🔵 *バックログS5 + ADR-0005より*

**私は** pq-diary ユーザー **として**
**よく使うエントリ構造をテンプレートとして保存したい**
**そうすることで** 同じ形式のエントリを効率的に作成できる

**関連要件**: REQ-101, REQ-102, REQ-103, REQ-104, REQ-105

**詳細シナリオ**:
1. `pq-diary template add meeting-notes` でテンプレートを作成
2. $EDITORで内容を入力（例: `## 参加者\n\n## 議題\n\n## 決定事項\n\n## TODO`）
3. `pq-diary template list` で保存済みテンプレート一覧を確認
4. `pq-diary template show meeting-notes` で内容を確認
5. `pq-diary template delete meeting-notes` で不要なテンプレートを削除

**前提条件**:
- Vaultが初期化済み・解錠可能

**制約事項**:
- テンプレートはvault.pqd内に暗号化格納
- テンプレート操作にはVault解錠が必要

**優先度**: Must Have

### ストーリー 2.2: テンプレートから変数付きエントリを作成する 🔵

**信頼性**: 🔵 *ヒアリングQ2, Q4より*

**私は** pq-diary ユーザー **として**
**テンプレートにカスタム変数を定義し、エントリ作成時に値を入力したい**
**そうすることで** 毎回変わる部分（プロジェクト名、相手先等）を簡単に埋められる

**関連要件**: REQ-111, REQ-112, REQ-113, REQ-114

**詳細シナリオ**:
1. テンプレート内に `## {{project_name}} ミーティング\n日時: {{date}}\n参加者: {{attendees}}` を定義
2. `pq-diary new --template meeting-notes -t "Title"` を実行
3. `{{date}}` は自動展開、`{{project_name}}` と `{{attendees}}` はプロンプト入力
4. 展開後の内容が$EDITORに表示される

**前提条件**:
- テンプレートが事前に作成済み

**制約事項**:
- 基本変数（`{{date}}`, `{{datetime}}`, `{{title}}`）はシステム自動設定
- それ以外の `{{var_name}}` はユーザー入力

**優先度**: Must Have

## エピック3: エントリ間リンク

### ストーリー 3.1: エントリ間をリンクで繋ぐ 🔵

**信頼性**: 🔵 *ADR-0004 + バックログS5より*

**私は** pq-diary ユーザー **として**
**エントリの本文に `[[別のエントリタイトル]]` と書いてリンクしたい**
**そうすることで** 関連する日記や議事録を相互に参照できる

**関連要件**: REQ-201, REQ-211

**詳細シナリオ**:
1. エントリ本文に `今日は [[プロジェクトA計画]] について議論した` と記入
2. `pq-diary show <id>` で表示時、`[[プロジェクトA計画]]` がリンク先情報付きで表示される
3. 同名タイトルが複数ある場合は全候補がリスト表示される

**前提条件**:
- リンク先エントリが存在する（存在しない場合は「未解決」と表示）

**制約事項**:
- リンク記法は `[[タイトル]]` (Obsidian互換) のみ

**優先度**: Must Have

---

### ストーリー 3.2: バックリンクで参照元を確認する 🔵

**信頼性**: 🔵 *バックログS5 + ヒアリングQ3より*

**私は** pq-diary ユーザー **として**
**あるエントリを `show` した時に、そのエントリを参照している他のエントリを知りたい**
**そうすることで** 知識のネットワークを俯瞰し、関連コンテキストを素早く見つけられる

**関連要件**: REQ-202, REQ-203

**詳細シナリオ**:
1. エントリA の本文に `[[エントリB]]` がある
2. `pq-diary show <エントリBのid>` を実行
3. エントリBの本文の後に `--- Backlinks ---` セクションでエントリAが表示される

**前提条件**:
- Vault解錠時にバックリンクインデックスが構築済み

**優先度**: Must Have

## エピック4: vim補完

### ストーリー 4.1: エディタ内でリンク先を補完入力する 🔵

**信頼性**: 🔵 *ADR-0004 + バックログS5より*

**私は** pq-diary ユーザー（vim/nvim利用者）**として**
**$EDITOR内で `[[` を入力した時にタイトル候補がポップアップ表示されてほしい**
**そうすることで** 正確なタイトルを覚えていなくても素早くリンクを挿入できる

**関連要件**: REQ-301, REQ-302, REQ-303

**詳細シナリオ**:
1. `edit` / `new` / `today` コマンドでvimが起動される
2. 本文入力中に `[[` と入力
3. Ctrl-X Ctrl-U（または自動トリガー）でタイトル候補がポップアップ表示
4. 候補を選択すると `[[選択タイトル]]` が挿入される

**前提条件**:
- $EDITORがvimまたはnvim
- 補完用タイトル一覧がsecure_tmpdir上に生成済み

**制約事項**:
- vim/neovim以外のエディタでは補完不可（手動入力）
- 補完用ファイルはエディタ終了後にzeroize削除

**優先度**: Must Have

## REQ-001: new コマンド 🔵

**信頼性**: 🔵 *PRD §2.3, §4.1, §6.2 + ヒアリングQ7*

### Given（前提条件）
- Vaultが初期化済み
- パスワードが正しい

### When（実行条件）
- `pq-diary new "タイトル" -t "タグ" -b "本文"` を実行

### Then（期待結果）
- UUID v4が生成される
- 暗号化ペイロード `{ title, tags, body }` がAES-256-GCMで暗号化される
- ML-DSA-65で署名が作成される
- HMAC-SHA256でcontent_hmacが計算される
- EntryRecordがvault.pqdに追記される
- 作成結果が表示される: `Created: {ID_PREFIX} "{title}"`

### テストケース

#### 正常系

- [ ] **TC-001-01**: CLIフラグで全パラメータ指定して作成 🔵
  - **入力**: `new "テスト" -t "タグ1" -t "タグ2" -b "本文"`
  - **期待結果**: エントリ作成成功。vault.pqd内にEntryRecord追加。復号して内容一致を確認

- [ ] **TC-001-02**: パイプ入力で本文を渡して作成 🔵
  - **入力**: `echo "パイプ本文" | new "テスト"` (PQ_DIARY_PASSWORD設定)
  - **期待結果**: stdinから本文読み取り。エントリ作成成功

- [ ] **TC-001-03**: $EDITOR経由で本文入力して作成 🔵
  - **入力**: `new "テスト"` (TTY接続、EDITOR=テスト用モックエディタ)
  - **期待結果**: 一時ファイル作成 → エディタ起動 → 保存後にエントリ作成

- [ ] **TC-001-04**: タグなしでエントリ作成 🔵
  - **入力**: `new "テスト" -b "本文"`
  - **期待結果**: tags=[] でエントリ作成成功

- [ ] **TC-001-05**: ネストタグ付きで作成 🔵
  - **入力**: `new "テスト" -t "仕事/プロジェクトA/設計" -b "本文"`
  - **期待結果**: タグ `仕事/プロジェクトA/設計` が正しく格納

#### 異常系

- [ ] **TC-001-E01**: パスワード未指定（非TTY、env未設定） 🔵
  - **入力**: `new "テスト" -b "本文"` (stdinがパイプ、パスワード未設定)
  - **期待結果**: エラー「パスワードが指定されていません」
  - **信頼性**: 🔵 *ヒアリング確認済み*

- [ ] **TC-001-E02**: Vault未初期化 🔵
  - **入力**: `new "テスト" -b "本文"` (vault.pqd不在)
  - **期待結果**: エラー「Vaultが見つかりません」

- [ ] **TC-001-E03**: パスワード不正 🔵
  - **入力**: `new "テスト" -b "本文" --password "wrong"`
  - **期待結果**: エラー「パスワードが正しくありません」

## REQ-002: list コマンド 🔵

**信頼性**: 🔵 *PRD §4.1 + ヒアリングQ4*

### Given（前提条件）
- Vault解錠済み、エントリが存在

### When（実行条件）
- `pq-diary list` を実行

### Then（期待結果）
- 新しい順に最大20件表示
- 各行: IDプレフィックス(4-8文字) + 日付 + タイトル + タグ

### テストケース

#### 正常系

- [ ] **TC-002-01**: デフォルト20件表示 🔵
  - **入力**: 30件のエントリがある状態で `list`
  - **期待結果**: 最新20件が新しい順に表示

- [ ] **TC-002-02**: -n で件数制限 🔵
  - **入力**: `list -n 5`
  - **期待結果**: 最新5件のみ表示

- [ ] **TC-002-03**: --tag でフィルタ（前方一致） 🔵
  - **入力**: `list --tag "仕事"`、エントリに `仕事`, `仕事/設計`, `日記` タグあり
  - **期待結果**: `仕事` と `仕事/設計` のエントリのみ表示

- [ ] **TC-002-04**: -q でタイトル検索 🔵
  - **入力**: `list -q "振り返り"`
  - **期待結果**: タイトルに「振り返り」を含むエントリのみ表示
  - **信頼性**: 🔵 *ヒアリング確認済み（本文検索はS6のsearchで実装）*

- [ ] **TC-002-05**: エントリ0件 🔵
  - **入力**: 空のVaultで `list`
  - **期待結果**: 空リスト表示（エラーではない）

#### 境界値

- [ ] **TC-002-B01**: エントリ数がちょうど20件 🔵
  - **入力**: 20件のエントリで `list`
  - **期待結果**: 全20件表示

- [ ] **TC-002-B02**: -n 0 指定 🔵
  - **入力**: `list -n 0`
  - **期待結果**: エラー「件数は1以上を指定してください」
  - **信頼性**: 🔵 *ヒアリング確認済み*

## エピック1: エントリ作成

### ストーリー 1.1: CLIフラグで素早くエントリ作成 🔵

**信頼性**: 🔵 *PRD §4.1 + ヒアリングQ7より*

**私は** pq-diary ユーザー **として**
**CLIフラグでタイトル・本文・タグを指定してエントリを作成したい**
**そうすることで** エディタを開かずに素早くメモを残せる

**関連要件**: REQ-001, REQ-011

**詳細シナリオ**:
1. `pq-diary new "買い物メモ" -b "牛乳、卵" -t "生活"` を実行
2. パスワードを入力（TTYプロンプト or env変数 or --password）
3. Vault解錠 → エントリ暗号化 → vault.pqd書き込み → Vault施錠
4. `Created: a3f9b2c1 "買い物メモ"` と表示

**前提条件**:
- Vaultが初期化済み

**優先度**: Must Have

---

### ストーリー 1.2: パイプ入力でエントリ作成 🔵

**信頼性**: 🔵 *ヒアリングQ7より*

**私は** スクリプトやClaude連携を行うユーザー **として**
**パイプ経由で本文を渡してエントリを作成したい**
**そうすることで** 自動化ワークフローに組み込める

**関連要件**: REQ-001, REQ-012

**詳細シナリオ**:
1. `echo "会議メモの内容" | pq-diary new "会議メモ" -t "仕事"` を実行
2. stdinから本文を読み取り
3. エントリ暗号化 → vault.pqd書き込み

**前提条件**:
- パスワードは `--password` または `PQ_DIARY_PASSWORD` で指定（stdinがパイプなのでTTY不可）

**優先度**: Must Have

---

### ストーリー 1.3: $EDITORでじっくり日記を書く 🔵

**信頼性**: 🔵 *PRD §4.1, §4.3 + ヒアリングQ7より*

**私は** 日記を書くユーザー **として**
**使い慣れたエディタで本文を編集したい**
**そうすることで** 長文の日記を快適に書ける

**関連要件**: REQ-001, REQ-013, REQ-061〜REQ-064

**詳細シナリオ**:
1. `pq-diary new "今日の振り返り" -t "日記"` を実行（`-b` なし、TTY接続）
2. セキュア一時ディレクトリに一時ファイル作成
3. $EDITORが起動（vim/neovimならswap/backup/undo無効化オプション付き）
4. ユーザーが本文を編集して保存・終了
5. 一時ファイルから本文を読み取り → 暗号化 → vault.pqd書き込み
6. 一時ファイルをzeroize削除

**前提条件**:
- $EDITOR環境変数が設定済み（未設定時はUnix: vi / Windows: notepad）
- TTY接続

**制約事項**:
- 一時ファイルは /dev/shm (Unix) または %LOCALAPPDATA%\pq-diary\tmp\ (Windows) に作成
- $TMPDIR/$TEMP/$TMP はセキュア一時ディレクトリに上書き

**優先度**: Must Have

## エピック3: エントリ編集

### ストーリー 3.1: $EDITORでエントリを編集する 🔵

**信頼性**: 🔵 *PRD §4.1, §4.3 + ヒアリングQ8より*

**私は** pq-diary ユーザー **として**
**既存エントリの本文・タイトル・タグをまとめて編集したい**
**そうすることで** 1回のエディタセッションで全て修正できる

**関連要件**: REQ-004, REQ-082, REQ-061〜REQ-064

**詳細シナリオ**:
1. `pq-diary edit a3f9` を実行
2. エントリ復号 → ヘッダーコメント形式で一時ファイル作成:
   ```
   # Title: 今日の振り返り
   # Tags: 日記/振り返り, 仕事
   # ---

   今日は良い一日だった。
   ```
3. $EDITORが起動
4. ユーザーがタイトル・タグ・本文を編集して保存・終了
5. ヘッダーパース → メタデータ更新 → 本文再暗号化 → vault.pqd更新
6. 一時ファイルzeroize削除

**前提条件**:
- 対象エントリが存在

**優先度**: Must Have

---

**優先度**: Should Have

## エピック4: エントリ削除

4. `Deleted: a3f9b2c1 "今日の振り返り"` と表示

**前提条件**:
- TTY接続（確認プロンプト表示のため）

**優先度**: Must Have

## エピック5: パスワード入力

### ストーリー 5.1: TTYで安全にパスワードを入力する 🔵

**信頼性**: 🔵 *PRD §4.2, ADR-0003 + ヒアリングQ6より*

**私は** pq-diary ユーザー **として**
**ターミナルで安全にパスワードを入力したい**
**そうすることで** パスワードが画面に表示されず、メモリからも安全に消去される

**関連要件**: REQ-051, REQ-053

**詳細シナリオ**:
1. 任意のVault操作コマンドを実行
2. `Password: ` プロンプト表示（エコーなし）
3. 入力 → SecretStringに直接格納 → Argon2idで鍵導出

**優先度**: Must Have

---

**信頼性**: 🔵 *PRD §4.2, ADR-0003より*

**私は** Claude Code連携やCI環境のユーザー **として**
**環境変数でパスワードを渡したい**
**そうすることで** 非対話環境でもVault操作できる

**関連要件**: REQ-051

**詳細シナリオ**:
1. `PQ_DIARY_PASSWORD="..." pq-diary list` を実行

**優先度**: Must Have

## エピック6: ネストタグ

### ストーリー 6.1: 階層タグでエントリを整理する 🔵

**信頼性**: 🔵 *PRD backlog S4, Obsidianタグ仕様より*

**私は** pq-diary ユーザー **として**
**ネスト構造のタグ（#仕事/プロジェクトA/設計）でエントリを分類したい**
**そうすることで** 細かいカテゴリ分けができる

**関連要件**: REQ-071, REQ-072, REQ-073

**詳細シナリオ**:
1. `pq-diary new "設計レビュー" -t "仕事/プロジェクトA/設計" -b "..."` を実行
2. タグ `仕事/プロジェクトA/設計` が格納される
3. `pq-diary list --tag "仕事"` で前方一致 → このエントリもマッチ
4. `pq-diary list --tag "仕事/プロジェクトA"` でも前方一致マッチ

**優先度**: Must Have

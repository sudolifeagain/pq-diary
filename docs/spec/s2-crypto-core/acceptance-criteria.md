## REQ-001: PQCフォーク 🔵

**信頼性**: 🔵 *PRDセクション5.3より*

### Given（前提条件）
- ml-kem / ml-dsa のフォークリポジトリが GitHub 上に作成されている
- `pq-diary` ブランチが存在する

### When（実行条件）
- フォークされたクレートをビルドする

### Then（期待結果）
- ml-kem フォークがビルド成功し、`DecapsulationKey` の zeroize が動作する
- ml-dsa フォークがビルド成功し、秘密鍵4フィールドの zeroize が動作し、CVE-2026-22705 がパッチ済みである

### テストケース

#### 正常系

- [ ] **TC-001-01**: ml-kemフォークのビルド成功 🔵
  - **入力**: `cargo build` (ml-kemフォーク)
  - **期待結果**: exit code 0、警告なし

- [ ] **TC-001-02**: ml-dsaフォークのビルド成功 🔵
  - **入力**: `cargo build` (ml-dsaフォーク)
  - **期待結果**: exit code 0、警告なし

- [ ] **TC-001-03**: ml-kem DecapsulationKey Drop時にzeroize動作 🔵
  - **入力**: `DecapsulationKey` を作成しDropする
  - **期待結果**: Drop後に内部データがゼロ埋めされていることを検証

- [ ] **TC-001-04**: ml-dsa 秘密鍵4フィールドのzeroize動作 🔵
  - **入力**: ml-dsa の `SigningKey` を作成しDropする
  - **期待結果**: `s1_hat` / `s2_hat` / `t0_hat` / `A_hat` がDrop後にゼロ埋めされていることを検証

## REQ-002: Argon2id鍵導出 🔵

**信頼性**: 🔵 *PRDセクション5.5より*

### Given（前提条件）
- パスワードとソルトが入力として与えられる
- Argon2idパラメータが設定されている (memory_cost_kb=65536, time_cost=3, parallelism=4)

### When（実行条件）
- Argon2id鍵導出を実行する

### Then（期待結果）
- 32バイトの対称鍵が導出される
- 同じパスワード・ソルトからは同一の鍵が導出される
- 異なるパスワードまたはソルトからは異なる鍵が導出される

### テストケース

#### 正常系

- [ ] **TC-002-01**: 正しいパスワードで同一鍵導出（同じソルト） 🔵
  - **入力**: パスワード "test_password"、ソルト [0x01; 16] で2回導出
  - **期待結果**: 2回の結果が完全に一致する

- [ ] **TC-002-02**: 異なるパスワードで異なる鍵導出 🔵
  - **入力**: パスワード "password_a" と "password_b"、同一ソルト
  - **期待結果**: 2つの鍵が異なる

- [ ] **TC-002-03**: 異なるソルトで異なる鍵導出 🔵
  - **入力**: 同一パスワード、ソルト [0x01; 16] と [0x02; 16]
  - **期待結果**: 2つの鍵が異なる

- [ ] **TC-002-04**: 出力が32バイト 🔵
  - **入力**: 任意のパスワードとソルト
  - **期待結果**: 導出された鍵の長さが正確に32バイト

#### 異常系

- [ ] **TC-002-E01**: 空パスワード → DiaryError::Password 🔵
  - **入力**: パスワード ""（空文字列）
  - **期待結果**: `DiaryError::Password` が返される

- [ ] **TC-002-E02**: memory_cost_kb < 19456 → 警告（ログ出力） 🔵
  - **入力**: memory_cost_kb=10000 でArgon2idを設定
  - **期待結果**: 警告ログが出力される（導出自体は実行される）

## REQ-003: AES-256-GCM暗号化/復号 🔵

**信頼性**: 🔵 *PRDセクション5.1より*

### Given（前提条件）
- 32バイトの対称鍵が利用可能である
- 平文データが入力として与えられる

### When（実行条件）
- AES-256-GCMで暗号化し、復号する

### Then（期待結果）
- 暗号化時にランダム12バイトnonceが生成される
- 暗号文+16バイトGCMタグが出力される
- 復号で元の平文が復元される

### テストケース

#### 正常系

- [ ] **TC-003-01**: 暗号化→復号でデータ一致 🔵
  - **入力**: 平文 "Hello, pq-diary!" と32バイト鍵
  - **期待結果**: 暗号化→復号後のデータが元の平文と完全一致

- [ ] **TC-003-02**: nonceが12バイト 🔵
  - **入力**: 暗号化を実行
  - **期待結果**: 生成されたnonceの長さが正確に12バイト

- [ ] **TC-003-03**: 暗号文が平文と異なる 🔵
  - **入力**: 平文を暗号化
  - **期待結果**: 暗号文のバイト列が平文と異なる

- [ ] **TC-003-04**: 異なるnonce→異なる暗号文 🔵
  - **入力**: 同じ平文・鍵で2回暗号化（各回でランダムnonce生成）
  - **期待結果**: 2つの暗号文が異なる

#### 異常系

- [ ] **TC-003-E01**: 暗号文1バイト改竄→復号エラー（GCMタグ不一致） 🔵
  - **入力**: 暗号文の1バイトをビット反転して復号
  - **期待結果**: 復号エラーが返される（GCMタグ検証失敗）

- [ ] **TC-003-E02**: 不正鍵→復号エラー 🔵
  - **入力**: 暗号化時と異なる鍵で復号
  - **期待結果**: 復号エラーが返される

#### 境界値

- [ ] **TC-003-B01**: 空データの暗号化/復号 🔵
  - **入力**: 空バイト列 `[]` を暗号化→復号
  - **期待結果**: 空データが正常に暗号化・復号される

- [ ] **TC-003-B02**: 大規模データ(1MB)の暗号化/復号 🔵
  - **入力**: 1MB (1,048,576バイト) のデータを暗号化→復号
  - **期待結果**: 正常に暗号化・復号され、データが一致する

- カプセル化で共有秘密と暗号文が生成される
- 脱カプセル化で同一の共有秘密が復元される

### テストケース

#### 正常系

- [ ] **TC-004-01**: 鍵ペア生成成功 🔵
  - **入力**: ML-KEM-768鍵ペア生成を実行
  - **期待結果**: EncapsulationKeyとDecapsulationKeyが正常に生成される

- [ ] **TC-004-02**: カプセル化→脱カプセル化で共有秘密一致 🔵
  - **入力**: 生成した鍵ペアでカプセル化→脱カプセル化
  - **期待結果**: 両者の共有秘密が完全に一致する

- [ ] **TC-004-03**: DecapsulationKeyのDrop時zeroize 🔵
  - **入力**: DecapsulationKeyを作成しDropする
  - **期待結果**: Drop後に内部データがゼロ埋めされていることを検証

#### 異常系

- [ ] **TC-004-E01**: 不正暗号文→脱カプセル化エラー 🔵
  - **入力**: ランダムなバイト列を暗号文として脱カプセル化を試行
  - **期待結果**: 脱カプセル化エラーが返される

## REQ-006: HMAC-SHA256 🔵

**信頼性**: 🔵 *PRDセクション5.1より*

### Given（前提条件）
- 32バイトの鍵とデータが入力として与えられる

### When（実行条件）
- HMAC-SHA256でMACを計算し、検証する

### Then（期待結果）
- 32バイトのMACが出力される
- 同じ鍵・データで同じMACが得られる
- 異なるデータで異なるMACが得られる

### テストケース

#### 正常系

- [ ] **TC-006-01**: 正しい鍵・データ→32バイトMAC 🔵
  - **入力**: 32バイト鍵とデータ "test data"
  - **期待結果**: 32バイトのMACが出力される

- [ ] **TC-006-02**: 異なるデータ→異なるMAC 🔵
  - **入力**: 同一鍵、データ "data_a" と "data_b"
  - **期待結果**: 2つのMACが異なる

- [ ] **TC-006-03**: 検証: 正しいMAC→成功 🔵
  - **入力**: 計算したMACで検証を実行
  - **期待結果**: 検証が成功する

#### 異常系

- [ ] **TC-006-E01**: 改竄データ→MAC検証失敗 🔵
  - **入力**: データ "original" でMAC計算後、データ "tampered" で検証
  - **期待結果**: MAC検証が失敗する

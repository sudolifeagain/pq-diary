# s2-crypto-core ヒアリング記録

**作成日**: 2026-04-03
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

Sprint 2（暗号コア）の実装範囲・粒度を明確化するため、PRD (requirements.md v4.0) と既存設計文書から特定した判断ポイントについてヒアリングを実施。

## 質問と回答

### Q1: PQCフォークリポジトリの管理方針

**カテゴリ**: 未定義部分詳細化
**背景**: PRDはml-kem/ml-dsaのzeroize補完を要件としているが、フォークリポジトリの作成状況と参照方法が未定義。

**回答**: フォークリポジトリは未作成。S2で作成する。GitHub git依存で参照する (`ml-kem = { git = "...", branch = "pq-diary" }`)

**信頼性への影響**: REQ-001 (PQCフォーク) が 🟡 → 🔵 に向上。フォーク作成タイミングと依存方式が明確になった。

---

### Q2: Argon2idパラメータの確定

**カテゴリ**: 未定義部分詳細化
**背景**: PRDはArgon2idのパラメータデフォルトを示しているが、Sprint 2での実装値を確定する必要がある。

**回答**: PRDデフォルト値を採用 (memory_cost_kb=65536, time_cost=3, parallelism=4)

**信頼性への影響**: REQ-002 (Argon2id) が 🟡 → 🔵 に向上。パラメータが確定した。

---

### Q3: 暗号アルゴリズムの優先度

**カテゴリ**: 優先順位
**背景**: Sprint 2で実装する暗号アルゴリズム4種の優先度を確認。

**回答**: 全4アルゴリズム (Argon2id, AES-256-GCM, ML-KEM-768, ML-DSA-65) がMust Have

**信頼性への影響**: 全REQの優先度が確定。

---

### Q4: HMAC-SHA256の実装スプリント

**カテゴリ**: 未定義部分詳細化
**背景**: PRDでHMAC-SHA256はデータ整合性検証に使用されるが、実装タイミングが未定義。

**回答**: S2で実装する

**信頼性への影響**: REQ-006 (HMAC-SHA256) が 🟡 → 🔵 に向上。実装スプリントが確定した。

---

### Q5: フォーク方式の詳細

**カテゴリ**: 追加要件
**背景**: PQCライブラリのフォーク方式について、patchセクションか別リポジトリかを確認。

**回答**: GitHub別リポジトリ + Cargo.toml git依存

**信頼性への影響**: REQ-402 (フォーク管理方針) が確定。

---

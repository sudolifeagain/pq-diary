## Phase A: 技術的負債修正

### REQ-A01/A02: vault reader サイズ上限 🔵

**Given**: vault.pqd にフィールドサイズが 16 MiB を超えるレコードが含まれる
**When**: `read_vault()` でファイルを読み込む
**Then**: `DiaryError::Vault("field size exceeds maximum")` が返される

#### テストケース

- [ ] **TC-A01-01**: `kem_encrypted_sk` の長さが 16 MiB + 1 バイトの場合にエラー 🔵
- [ ] **TC-A01-02**: `record_len` が 16 MiB + 1 バイトの場合にエラー 🔵
- [ ] **TC-A01-03**: 正常サイズのフィールドは従来通り読み込み可能 🔵

### REQ-A03: init_vault 空パスワードガード 🔵

**Given**: 空のパスワード (`b""`) が渡される
**When**: `VaultManager::init_vault("test", b"")` を呼ぶ
**Then**: `DiaryError::Password` が返される

- [ ] **TC-A03-01**: 空パスワードで `Password` エラー 🔵
- [ ] **TC-A03-02**: 非空パスワードでは正常に vault が作成される 🔵

### REQ-A04: writer try_from 変換 🔵

**Given**: 4 GB を超えるフィールドを含む `EntryRecord`
**When**: `write_vault()` で書き込む
**Then**: `DiaryError::Vault` がサイレント切り捨てではなくエラーとして返される

- [ ] **TC-A04-01**: 正常サイズのレコードは書き込み可能 🔵
- [ ] **TC-A04-02**: padding が 256 バイト以上の場合にエラー 🔵

### REQ-A05: アトミック write 🔵

**Given**: vault.pqd が存在する
**When**: `write_vault()` で書き込み中にプロセスが中断される
**Then**: 元の vault.pqd が破損していない（一時ファイルのみが残る）

- [ ] **TC-A05-01**: 正常書き込み後に一時ファイルが残っていない 🔵
- [ ] **TC-A05-02**: 書き込み後の vault.pqd が読み込み可能 🔵

### REQ-A06: vim セキュリティオプション 🔵

- [ ] **TC-A06-01**: vim 起動オプションに `nowritebackup` が含まれる 🔵
- [ ] **TC-A06-02**: vim 起動オプションに `viminfo=''` が含まれる 🔵
- [ ] **TC-A06-03**: nvim 起動オプションに `shada=''` が含まれる 🟡

### REQ-A07: 環境変数削除 🔵

- [ ] **TC-A07-01**: `PQ_DIARY_PASSWORD` 設定後に `get_password` を呼ぶと、呼び出し後に環境変数が存在しない 🔵

### REQ-A08: tmpfile パーミッション 🔵

- [ ] **TC-A08-01**: Unix で作成された一時ファイルのパーミッションが 0o600 である 🔵

### REQ-A09: HMAC エラーハンドリング 🔵

- [ ] **TC-A09-01**: `compute()` の戻り値型が `Result<[u8; 32], DiaryError>` である 🔵
- [ ] **TC-A09-02**: 既存のテストが新しい戻り値型で通過する 🔵

### REQ-A10: EntryPlaintext zeroize 🔵

- [ ] **TC-A10-01**: `EntryPlaintext` の `title`, `tags`, `body` が drop 後にゼロ化される 🔵

### REQ-A11: list_entries_with_body zeroize 🔵

- [ ] **TC-A11-01**: `list_entries_with_body` の戻り値が `Zeroizing` でラップされている 🔵

## Phase B: search コマンド

### REQ-B01/B02/B03: 基本検索 🔵

**Given**: vault に 3 エントリが存在する (title: "朝の散歩", body: "公園で桜を見た", tags: ["日記"])
**When**: `pq-diary search "桜"` を実行
**Then**: "朝の散歩" エントリがマッチ行付きで表示される

#### テストケース

- [ ] **TC-B01-01**: body にマッチするパターンでエントリが返される 🔵
- [ ] **TC-B01-02**: title にマッチするパターンでエントリが返される 🔵
- [ ] **TC-B01-03**: tags にマッチするパターンでエントリが返される 🔵
- [ ] **TC-B01-04**: テンプレート本文にマッチするパターンで結果が返される 🔵
- [ ] **TC-B01-05**: どのフィールドにもマッチしない場合、空結果で "No matches found" 🟡
- [ ] **TC-B01-06**: 正規表現 `\d{4}-\d{2}-\d{2}` で日付パターンがマッチする 🟡

### REQ-B05: コンテキスト表示 🔵

- [ ] **TC-B05-01**: デフォルトでマッチ行の前後2行が表示される 🔵
- [ ] **TC-B05-02**: `--context 5` で前後5行が表示される 🔵
- [ ] **TC-B05-03**: マッチ行がエントリの先頭の場合、前方は利用可能な行のみ表示 🟡

### REQ-B06: タグフィルタ 🟡

- [ ] **TC-B06-01**: `--tag "仕事"` で仕事タグのエントリのみが検索される 🟡
- [ ] **TC-B06-02**: タグの前方一致 (`仕事` が `仕事/プロジェクト` にマッチ) 🟡

### REQ-B08/B09: セキュリティ 🔵

- [ ] **TC-B08-01**: 検索完了後に復号データが zeroize される 🔵

## Phase C: stats コマンド

### REQ-C01/C02/C03: テキスト統計表示 🔵

**Given**: vault に 10 エントリが存在する
**When**: `pq-diary stats` を実行
**Then**: エントリ総数、執筆頻度、文字数統計、タグ分布がテキストで表示される

#### テストケース

- [ ] **TC-C01-01**: エントリ総数が正しい 🔵
- [ ] **TC-C01-02**: 最初/最後のエントリ日付が正しい 🔵
- [ ] **TC-C01-03**: 文字数統計（合計/平均/最大）が正しい 🟡
- [ ] **TC-C01-04**: タグ分布の上位10タグが降順で表示される 🟡
- [ ] **TC-C01-05**: 空 vault で実行した場合にエラーにならず基本情報が表示される 🟡

### REQ-C04: JSON 出力 🔵

- [ ] **TC-C04-01**: `--json` で valid JSON が stdout に出力される 🔵
- [ ] **TC-C04-02**: JSON に全統計フィールドが含まれる 🔵

### REQ-C05: ヒートマップ 🔵

- [ ] **TC-C05-01**: `--heatmap` で直近1年の日別データがASCII表示される 🔵
- [ ] **TC-C05-02**: 端末幅 80 文字以上で正しく表示される 🟡

## Phase D: import コマンド

### REQ-D01/D02/D03: 基本インポート 🔵

**Given**: ディレクトリ `./notes/` に 5 つの `.md` ファイルが存在する
**When**: `pq-diary import ./notes/` を実行
**Then**: 5 エントリが vault に追加され、サマリーが表示される

#### テストケース

- [ ] **TC-D01-01**: `.md` ファイルがエントリとして取り込まれる 🔵
- [ ] **TC-D01-02**: サブディレクトリの `.md` ファイルも再帰的に取り込まれる 🟡
- [ ] **TC-D01-03**: ファイル名（拡張子除去）がタイトルになる 🔵
- [ ] **TC-D01-04**: frontmatter に title がある場合はそちらが優先される 🔵
- [ ] **TC-D01-05**: `.md` 以外のファイルはスキップされる 🟡

### REQ-D04/D05: リンク・タグ変換 🔵

- [ ] **TC-D04-01**: `[[wiki-link]]` が `[[タイトル]]` に変換される 🔵
- [ ] **TC-D04-02**: `[[link|alias]]` のエイリアス形式が適切に処理される 🟡
- [ ] **TC-D05-01**: `#タグ` がエントリの tags フィールドに取り込まれる 🔵
- [ ] **TC-D05-02**: `#親/子/孫` のネストタグが保持される 🔵

### REQ-D06: frontmatter タグ 🟡

- [ ] **TC-D06-01**: YAML frontmatter の `tags: [a, b]` がエントリの tags に取り込まれる 🟡
- [ ] **TC-D06-02**: frontmatter がないファイルではタグは本文中の `#タグ` のみ 🟡

### REQ-D07: ドライラン 🟡

- [ ] **TC-D07-01**: `--dry-run` で vault に書き込みが行われない 🟡
- [ ] **TC-D07-02**: `--dry-run` で対象ファイル数と変換結果が表示される 🟡

### REQ-D08: Obsidian スキップ 🔵

- [ ] **TC-D08-01**: `.obsidian/` ディレクトリ内のファイルがスキップされる 🔵

### REQ-D09: サマリー表示 🔵

- [ ] **TC-D09-01**: インポート後に取り込み件数が表示される 🔵
- [ ] **TC-D09-02**: スキップ件数が表示される 🔵
- [ ] **TC-D09-03**: 変換されたリンク数・タグ数が表示される 🔵

### REQ-D10/D11/D12: 制約 🟡

- [ ] **TC-D10-01**: 100 ファイルのインポートで write_vault が 1 回のみ呼ばれる 🟡
- [ ] **TC-D11-01**: インポート後に読み込みバッファが zeroize される 🔵
- [ ] **TC-D12-01**: インポート後に元の `.md` ファイルが存在する（削除されていない） 🟡

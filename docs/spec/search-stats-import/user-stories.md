## エピック0: セキュリティ技術的負債の修正

### ストーリー 0.1: vault ファイルの堅牢化 🔵

**信頼性**: 🔵 *S1-S5全スプリントレビュー結果*

**私は** pq-diary 開発者 **として**
**vault reader/writer のセキュリティ脆弱性を修正したい**
**そうすることで** 悪意ある vault ファイルによる OOM 攻撃やデータ破損を防げる

**関連要件**: REQ-A01, REQ-A02, REQ-A03, REQ-A04, REQ-A05

**詳細シナリオ**:
1. vault reader にフィールドサイズ上限チェック (16 MiB) を追加
2. writer の全 `as u32`/`as u8` を `try_from` に置換
3. `init_vault` に空パスワードガードを追加
4. `write_vault` をアトミック書き込み (temp + rename) に変更

**優先度**: Must Have

## エピック1: 全文検索

### ストーリー 1.1: エントリ内容の正規表現検索 🔵

**信頼性**: 🔵 *backlog.md S6 + ユーザヒアリング 2026-04-08*

**私は** pq-diary ユーザー **として**
**暗号化された日記の中から特定のキーワードやパターンを検索したい**
**そうすることで** 過去の記録を素早く見つけられる

**関連要件**: REQ-B01, REQ-B02, REQ-B03, REQ-B04, REQ-B05

**詳細シナリオ**:
1. `pq-diary search "パターン"` を実行
2. パスワードを入力して vault を unlock
3. 全エントリを復号し、body + title + tags + テンプレート本文を正規表現で検索
4. マッチしたエントリの ID、日付、タイトル、マッチ行 + 前後2行を表示
5. 復号データを zeroize して vault を lock

**前提条件**:
- vault が存在し、パスワードが正しいこと
- `regex` クレートが依存関係に追加されていること

**優先度**: Must Have

**信頼性**: 🔵 *backlog.md S6 + ユーザヒアリング 2026-04-08*

**そうすることで** 手動で1つずつコピーする手間を省ける

5. 全エントリをバッチで vault に書き込み
6. サマリー表示: 取り込み 42 件、スキップ 3 件、リンク変換 15 件、タグ変換 8 件

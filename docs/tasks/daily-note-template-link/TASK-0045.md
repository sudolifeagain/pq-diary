## タスク概要

`core/src/link.rs` に `LinkIndex` を実装する。エントリのタイトルとbody内リンクからforward map（タイトル→UUID）とreverse map（UUID→バックリンク元UUID）を構築する。`ResolvedLink` 型でリンク解決結果を返す。

## 単体テスト要件

### TC-045-01: 基本的なインデックス構築 🔵
**Given**: エントリA(body: "[[B]]"), エントリB(body: "")
**When**: LinkIndex::build()
**Then**: resolve("B") → エントリBのUUID、backlinks_for("B") → エントリA

### TC-045-02: 同名タイトル重複の解決 🔵
**Given**: エントリA(title: "メモ"), エントリB(title: "メモ"), エントリC(body: "[[メモ]]")
**When**: resolve("メモ")
**Then**: matches に A, B の2件

### TC-045-03: 未解決リンク 🔵
**Given**: エントリA(body: "[[存在しない]]")
**When**: resolve("存在しない")
**Then**: matches が空

### TC-045-04: 相互リンク 🔵
**Given**: エントリA(body: "[[B]]"), エントリB(body: "[[A]]")
**When**: backlinks_for("A") + backlinks_for("B")
**Then**: それぞれ相手が含まれる

### TC-045-05: all_titles() 🔵
**Given**: 3エントリ (タイトル: "A", "B", "C")
**When**: all_titles()
**Then**: ["A", "B", "C"]

### TC-045-06: 空インデックス 🔵
**Given**: エントリなし
**When**: LinkIndex::build(&[])
**Then**: resolve → 未解決, backlinks_for → 空, all_titles → 空

## 注意事項

- LinkIndex はタイトル文字列を含むため秘密データ。Zeroize必須
- build() の引数は EntryMeta + body文字列のペア（復号済みデータ）
- HashMap のキーは String (タイトル) と [u8; 16] (UUID)

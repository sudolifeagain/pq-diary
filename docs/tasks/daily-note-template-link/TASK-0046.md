## タスク概要

`core/src/lib.rs` の DiaryCore に `link_index: Option<LinkIndex>` フィールドを追加し、テンプレート操作メソッド（new_template, list_templates, get_template, delete_template）とリンク操作メソッド（resolve_links, backlinks_for, all_titles）を追加する。unlock時にLinkIndex構築、lock時にzeroize消去を行う。

### 1. DiaryCore フィールド追加 🔵

```rust
pub struct DiaryCore {

```

### 2. unlock拡張 🔵

**信頼性**: 🔵 *REQ-203・dataflow.mdより*

既存のunlockロジックの後に:
1. list_entries(None) で全エントリ取得
2. 各エントリの body を取得
3. LinkIndex::build() でインデックス構築
4. self.link_index = Some(index)

### 3. lock拡張 🔵

**信頼性**: 🔵 *REQ-403より*

```rust
fn lock(&mut self) {
    self.engine = None;
    self.link_index = None;  // Drop → Zeroize
}
```

### 4. テンプレートメソッド 🔵

**信頼性**: 🔵 *REQ-101〜105・entry操作メソッドパターンより*

既存の entry 操作メソッド（new_entry, list_entries 等）と同じパターンで実装。ensure_unlocked() → template:: 関数呼び出し。

### 5. リンクメソッド 🔵

**信頼性**: 🔵 *REQ-201〜203より*

```rust
pub fn resolve_links(&self, body: &str) -> Vec<ResolvedLink> {
    let index = self.link_index.as_ref().ok_or(NotUnlocked)?;
    let parsed = parse_links(body);
    parsed.iter().map(|l| index.resolve(&l.title)).collect()
}
```

`cli/src/commands.rs` の `cmd_show` を拡張し、本文中の `[[タイトル]]` リンクを解決情報付きで表示し、末尾に `--- Backlinks ---` セクションを追加する。

## 完了条件

- [ ] cmd_show で本文中の `[[タイトル]]` の横にリンク先情報が表示される
- [ ] 一意解決: `[[タイトル]] → [abcd1234]` 形式で表示
- [ ] 複数候補: 候補リストを表示
- [ ] 未解決: `[[タイトル]] (未解決)` と表示
- [ ] 空リンク `[[]]` はそのまま表示

- [ ] バックリンクがない場合はセクションを非表示
- [ ] 全テストがパス

## 実装詳細

### 1. リンク解決表示 🔵

**信頼性**: 🔵 *REQ-201, REQ-211・dataflow.mdより*

cmd_show の本文表示後に:
1. `resolve_links(body)` でリンク解決
2. 各リンクについて:
   - matches.len() == 1 → `[[タイトル]] → [abcd1234]`
   - matches.len() > 1 → `[[タイトル]] → 複数候補:` + リスト
   - matches.len() == 0 → `[[タイトル]] (未解決)`

### 2. バックリンク表示 🔵

**信頼性**: 🔵 *REQ-202・NFR-202より*

```
--- Backlinks ---
  [abcd1234] 2026-04-01 エントリAのタイトル
  [efgh5678] 2026-04-03 エントリBのタイトル
```

- backlinks_for(title) で取得
- 日付の新しい順にソート
- バックリンクが0件の場合はセクション自体を表示しない

## 単体テスト要件

### TC-050-01: 一意リンク解決の表示 🔵
**Given**: エントリA(body: "[[B]]"), エントリB が存在
**When**: show A の表示処理
**Then**: `[[B]]` の横に B のUUIDプレフィックスが表示

### TC-050-02: 未解決リンクの表示 🔵
**Given**: エントリA(body: "[[存在しない]]")
**When**: show A の表示処理
**Then**: `[[存在しない]] (未解決)` と表示

### TC-050-03: 複数候補リンクの表示 🔵
**Given**: エントリA(title: "メモ"), エントリB(title: "メモ"), エントリC(body: "[[メモ]]")
**When**: show C の表示処理
**Then**: 2件の候補が表示

### TC-050-04: バックリンクの表示 🔵
**Given**: エントリA(body: "[[B]]"), エントリB
**When**: show B の表示処理
**Then**: `--- Backlinks ---` にエントリAが表示

### TC-050-05: バックリンクなし時の表示 🔵
**Given**: どこからも参照されていないエントリ
**When**: show の表示処理
**Then**: `--- Backlinks ---` セクションが表示されない

### TC-050-06: リンクなしエントリの表示 🔵
**Given**: `[[` を含まないエントリ
**When**: show の表示処理
**Then**: 通常表示（リンク情報なし）

## 実装詳細

### 1. write_completion_file 🔵

**信頼性**: 🔵 *REQ-302・NFR-102より*

```rust
pub fn write_completion_file(tmpdir: &Path, titles: &[String]) -> Result<PathBuf, DiaryError> {
    let path = tmpdir.join(format!("{}.completion", Uuid::new_v4().as_simple()));
    let content = titles.join("\n");
    fs::write(&path, content.as_bytes())?;
    Ok(path)
}
```

- タイトルのみ（1行1タイトル）
- secure_tmpdir 上に配置

### 2. vim_completion_options 🔵

**信頼性**: 🔵 *ADR-0004・dataflow.mdより*

vim/nvim 起動時に `-c` オプションで:
1. completefunc を PqDiaryComplete に設定
2. PqDiaryComplete 関数を定義（補完ファイルからタイトルを読み込み）
3. `[[` 入力後の Ctrl-X Ctrl-U でトリガー

```vim
set completefunc=PqDiaryComplete
function! PqDiaryComplete(findstart, base)
  if a:findstart
    let line = getline('.')
    let start = col('.') - 1
    while start > 1 && line[start-2:start-1] != '[['
      let start -= 1
    endwhile
    return start
  else
    let titles = readfile('{completion_file_path}')
    return filter(titles, 'v:val =~ "^" . a:base')
  endif
endfunction
```

### 3. エディタ起動フロー更新 🔵

**信頼性**: 🔵 *既存editor.rsパターンより*

- edit/new/today コマンドでエディタ起動前に:
  1. all_titles() でタイトル一覧取得
  2. write_completion_file() で一時ファイル生成
  3. vim_completion_options() でオプション生成
  4. 既存の vim_options と結合
- エディタ終了後:
  5. zeroize_and_delete(completion_file)

### 4. 非vim エディタのフォールバック 🔵

**信頼性**: 🔵 *REQ-303より*

vim/nvim以外のエディタでは:
- write_completion_file() をスキップ
- vim_completion_options() は空Vecを返す
- 通常のエディタ起動のみ

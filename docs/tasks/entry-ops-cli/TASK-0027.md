## 実装詳細

### 1. serde_json 依存追加 🔵

**信頼性**: 🔵 *architecture.md §3 依存追加より*

`core/Cargo.toml` の `[dependencies]` に `serde_json = "1"` を追加する。`EntryPlaintext` の JSON シリアライズ/デシリアライズに必要。

**実装ファイル**: `core/Cargo.toml`

### 2. EntryPlaintext 構造体 🔵

**信頼性**: 🔵 *REQ-081, types.rs EntryPlaintext定義より*

```rust
#[derive(Debug, Serialize, Deserialize, PartialEq)]
pub struct EntryPlaintext {
    pub title: String,
    pub tags: Vec<String>,
    pub body: String,
}
```

暗号化フロー: `EntryPlaintext` → `serde_json::to_vec()` → `CryptoEngine::encrypt()` → `EntryRecord.ciphertext`

**実装ファイル**: `core/src/entry.rs`

### 3. EntryMeta 構造体 🔵

**信頼性**: 🔵 *REQ-002, types.rs EntryMeta定義より*

```rust
pub struct EntryMeta {
    pub uuid_hex: String,
    pub title: String,
    pub tags: Vec<String>,
    pub created_at: u64,
    pub updated_at: u64,
}

impl EntryMeta {
    pub fn id_prefix(&self, len: usize) -> &str {
        &self.uuid_hex[..len.min(self.uuid_hex.len())]
    }
}
```

**実装ファイル**: `core/src/entry.rs`

### 4. Tag ニュータイプ 🔵

**信頼性**: 🔵 *REQ-071〜073, types.rs Tag定義より*

- `new()`: 先頭/末尾の `/` を除去、スペース禁止、空文字禁止、数字のみ禁止
- `as_str()`: 内部文字列への参照を返す
- 許容文字: 英数字、Unicode文字、`_`、`-`、`/` (階層区切り)

**実装ファイル**: `core/src/entry.rs`

### 5. IdPrefix ニュータイプ 🔵

**信頼性**: 🔵 *REQ-041〜043, types.rs IdPrefix定義より*

- `new()`: 最小4文字、`[0-9a-f]` のみ許容。大文字は小文字に正規化
- `matches(&[u8; 16])`: UUID バイト列を hex 化して前方一致比較
- エラーメッセージ: 「IDプレフィックスは4文字以上必要です」

**実装ファイル**: `core/src/entry.rs`

## テスト要件

### EntryPlaintext serdeラウンドトリップ 🔵

**信頼性**: 🔵 *REQ-081より*

- `EntryPlaintext` を `serde_json::to_vec()` → `serde_json::from_slice()` で復元し、元の値と一致することを確認
- タイトル・タグ・本文に日本語 Unicode を含むケースも検証

### Tag バリデーション (正常系) 🔵

**信頼性**: 🔵 *REQ-071〜073より*

- `Tag::new("日記")` → Ok
- `Tag::new("仕事/設計/レビュー")` → Ok (ネストタグ)
- `Tag::new("work_log")` → Ok (アンダースコア)
- `Tag::new("2024-review")` → Ok (ハイフン)
- `Tag::new("/日記/")` → Ok、内部値は `"日記"` (先頭/末尾スラッシュ除去)

### Tag バリデーション (異常系) 🔵

**信頼性**: 🔵 *REQ-072より*

- `Tag::new("")` → Err (空文字)
- `Tag::new("  ")` → Err (スペースのみ)
- `Tag::new("my tag")` → Err (スペース含む)

### IdPrefix バリデーション (正常系) 🔵

**信頼性**: 🔵 *REQ-041より*

- `IdPrefix::new("abcd")` → Ok (最小4文字)
- `IdPrefix::new("0123456789abcdef")` → Ok (16文字)
- `IdPrefix::new("ABCD")` → Ok、内部値は `"abcd"` (小文字正規化)

### IdPrefix バリデーション (異常系) 🔵

**信頼性**: 🔵 *REQ-041より*

- `IdPrefix::new("abc")` → Err (3文字は不足)
- `IdPrefix::new("ghij")` → Err (非hex文字)
- `IdPrefix::new("")` → Err (空文字)

### IdPrefix マッチング 🔵

**信頼性**: 🔵 *REQ-041〜043より*

- 既知のUUID バイト列に対して、先頭4文字のプレフィックスで `matches()` が true を返すことを確認
- 不一致のプレフィックスで false を返すことを確認
- フルUUID (32文字) でマッチすることを確認

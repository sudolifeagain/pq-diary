## タスク概要

エントリの新規作成関数 `create_entry()` を実装する。`EntryPlaintext` を受け取り、JSON シリアライズ → AES-256-GCM 暗号化 → ML-DSA-65 署名 → HMAC-SHA256 計算の一連の暗号パイプラインを通して `EntryRecord` を構築し、vault.pqd に追記する。S3 で実装済みの `vault::reader` / `vault::writer` と `CryptoEngine` を組み合わせた read-modify-write パターンで動作する。

## 実装詳細

### 1. create_entry 関数 🔵

**信頼性**: 🔵 *REQ-001, architecture.md §5.2, types.rs create_entry定義より*

```rust
pub fn create_entry(
    vault_path: &Path,
    engine: &CryptoEngine,
    plaintext: &EntryPlaintext,
) -> Result<Uuid, DiaryError>
```

処理フロー:

1. `Uuid::new_v4()` で UUID を生成
2. `serde_json::to_vec(&plaintext)?` で JSON バイト列に変換
3. `engine.encrypt(&json_bytes)` → `(ciphertext, iv)` を取得
4. `engine.dsa_sign(&ciphertext)` → `signature` を取得
5. `engine.hmac(&record_data)` → `content_hmac` を取得
6. `EntryRecord` を構築:
   - `uuid`: 生成した UUID のバイト列
   - `record_type`: `0x01` (通常エントリ)
   - `legacy_flag`: `0x00`
   - `attachment_count`: `0`
   - `created_at` / `updated_at`: 現在の Unix タイムスタンプ
   - `entry_padding`: `generate_entry_padding()` で生成
   - `ciphertext`, `iv`, `signature`, `content_hmac`: 上記で計算した値
7. `reader::read_vault(vault_path)` で既存データを読み込み
8. `entries.push(new_record)` で新しいエントリを追加
9. `writer::write_vault(vault_path, &header, &entries)` で書き戻し
10. 生成した UUID を返す

**実装ファイル**: `core/src/entry.rs`

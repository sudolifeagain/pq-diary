## タスク概要

vault.pqd 内の全エントリを復号し、一覧表示用の `EntryMeta` ベクタを返す `list_entries()` 関数を実装する。各 `EntryRecord` を `CryptoEngine::decrypt()` で復号し、`serde_json::from_slice()` で `EntryPlaintext` に変換したうえで、UUID・タイトル・タグ・タイムスタンプを `EntryMeta` に格納する。ソート・フィルタは呼び出し側 (cli) で行う設計とし、core 層はすべてのエントリメタデータを返す。

## 実装詳細

### 1. list_entries 関数 🔵

**信頼性**: 🔵 *REQ-002, architecture.md §5.2, types.rs list_entries定義より*

```rust
pub fn list_entries(
    vault_path: &Path,
    engine: &CryptoEngine,
) -> Result<Vec<EntryMeta>, DiaryError>
```

処理フロー:

1. `reader::read_vault(vault_path)` で `(VaultHeader, Vec<EntryRecord>)` を取得
2. 各 `EntryRecord` に対して:
   a. `engine.decrypt(&record.ciphertext, &record.iv)` で復号
   b. `serde_json::from_slice::<EntryPlaintext>(&decrypted)` でデシリアライズ
   c. `EntryMeta` を構築:
      - `uuid_hex`: `hex::encode(&record.uuid)`
      - `title`: `plaintext.title`
      - `tags`: `plaintext.tags`
      - `created_at`: `record.created_at`
      - `updated_at`: `record.updated_at`
3. `Vec<EntryMeta>` を返す

**実装ファイル**: `core/src/entry.rs`

## テスト要件

### 空 vault の一覧取得 🔵

**信頼性**: 🔵 *EDGE-104より*

- 初期化直後の vault に対して `list_entries()` を呼び出し
- 返却値が空ベクタであることを確認
- エラーではないことを確認

### 単一エントリの一覧取得 🔵

**信頼性**: 🔵 *REQ-002より*

- `create_entry()` で 1 件のエントリを作成
- `list_entries()` でメタデータを取得
- 件数が 1 であることを確認
- `title`, `tags`, `uuid_hex`, `created_at`, `updated_at` がすべて正しく設定されていることを確認

### 複数エントリの一覧取得 🔵

**信頼性**: 🔵 *REQ-002より*

- `create_entry()` で 3 件のエントリを異なるタイトル・タグで作成
- `list_entries()` で全件のメタデータを取得
- 件数が 3 であることを確認
- 各エントリのタイトルとタグが正しく対応していることを確認

### 全フィールド検証 🔵

**信頼性**: 🔵 *REQ-002, types.rs EntryMeta定義より*

- 作成したエントリの `EntryMeta` について:
  - `uuid_hex` が 32 文字の hex 文字列であることを確認
  - `title` が作成時のタイトルと一致
  - `tags` が作成時のタグリストと一致
  - `created_at` > 0 (有効なタイムスタンプ)
  - `updated_at` >= `created_at`
  - `id_prefix(4)` が `uuid_hex` の先頭 4 文字と一致

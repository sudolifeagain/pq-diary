## タスク概要

ID プレフィックスでエントリを検索し、一意に特定して復号する `get_entry()` 関数を実装する。プレフィックス解決ロジックとして、一意マッチ → 成功、0 マッチ → エラー、複数マッチ → 候補一覧付きエラーの 3 パターンを処理する。`show` / `edit` / `delete` コマンドの基盤となる重要な関数。

## 実装詳細

### 1. get_entry 関数 🔵

**信頼性**: 🔵 *REQ-003, REQ-041〜043, architecture.md §5.3, types.rs get_entry定義より*

```rust
pub fn get_entry(
    vault_path: &Path,
    engine: &CryptoEngine,
    prefix: &IdPrefix,
) -> Result<(EntryRecord, EntryPlaintext), DiaryError>
```

処理フロー:

1. `reader::read_vault(vault_path)` で `(VaultHeader, Vec<EntryRecord>)` を取得
2. `entries.iter().filter(|r| prefix.matches(&r.uuid)).collect::<Vec<_>>()` でフィルタ
3. マッチ数に応じた処理分岐:
   - **1 件**: `engine.decrypt()` → `serde_json::from_slice()` → `(record, plaintext)` を返す
   - **0 件**: `DiaryError::Entry("エントリが見つかりません".into())` を返す
   - **2 件以上**: マッチした UUID を hex 化して候補一覧を構築し、`DiaryError::Entry(format!("複数のエントリがマッチしました: {}", candidates))` を返す

**実装ファイル**: `core/src/entry.rs`

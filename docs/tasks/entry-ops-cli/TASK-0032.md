## タスク概要

エントリの更新関数 `update_entry()` と削除関数 `delete_entry()` を実装する。`update_entry()` は既存エントリの UUID と `created_at` を保持しつつ、暗号文・署名・HMAC・`updated_at` を再計算して vault.pqd を書き戻す。`delete_entry()` は指定 UUID のエントリを vault.pqd から除去し、パディングを再生成して書き戻す。どちらも read-modify-write パターンで動作する。

## 実装詳細

### 1. update_entry 関数 🔵

**信頼性**: 🔵 *REQ-004, architecture.md §5.2, types.rs update_entry定義より*

```rust
pub fn update_entry(
    vault_path: &Path,
    engine: &CryptoEngine,
    uuid: [u8; 16],
    plaintext: &EntryPlaintext,
) -> Result<(), DiaryError>
```

処理フロー:

1. `reader::read_vault(vault_path)` で `(VaultHeader, Vec<EntryRecord>)` を取得
2. `entries` 内で `uuid` が一致するエントリを検索
3. 一致なし → `DiaryError::Entry("エントリが見つかりません")` を返す
4. 新しい `EntryPlaintext` を `serde_json::to_vec()` → `engine.encrypt()` → `engine.dsa_sign()` → `engine.hmac()`
5. 既存エントリの以下フィールドを更新:
   - `ciphertext`: 新しい暗号文
   - `iv`: 新しい IV
   - `signature`: 新しい署名
   - `content_hmac`: 新しい HMAC
   - `updated_at`: 現在の Unix タイムスタンプ
   - **保持**: `uuid`, `created_at`, `record_type`, `legacy_flag`
6. `writer::write_vault(vault_path, &header, &entries)` で書き戻し

**実装ファイル**: `core/src/entry.rs`

### 2. delete_entry 関数 🔵

**信頼性**: 🔵 *REQ-005, architecture.md §5.2, types.rs delete_entry定義より*

```rust
pub fn delete_entry(
    vault_path: &Path,
    engine: &CryptoEngine,
    uuid: [u8; 16],
) -> Result<(), DiaryError>
```

処理フロー:

1. `reader::read_vault(vault_path)` で `(VaultHeader, Vec<EntryRecord>)` を取得
2. `uuid` が一致するエントリが存在するか確認。存在しない場合はエラー
3. `entries.retain(|r| r.uuid != uuid)` で該当エントリを除外
4. `writer::write_vault(vault_path, &header, &entries)` で書き戻し (パディング再生成)

**実装ファイル**: `core/src/entry.rs`

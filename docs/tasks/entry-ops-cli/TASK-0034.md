## タスク概要

CLIにおけるパスワード取得の3段階フォールバックメカニズムを実装する。`--password` フラグ、`PQ_DIARY_PASSWORD` 環境変数、TTY対話入力の優先順位でパスワードを取得し、いずれも利用不可の場合はエラーを返す。Unix/Windows両対応のエコーなしTTYパスワード読み取りを含む。

## 実装詳細

### 1. `get_password()` 関数 🔵

**信頼性**: 🔵 *要件定義書 REQ-051〜054、ADR-0003*

3段階のフォールバックでパスワードを取得する関数を実装する。

```rust
pub fn get_password(flag_value: Option<&str>) -> Result<PasswordSource, DiaryError>
```

**取得優先順位**:
1. `--password` フラグが指定されている場合: `SecretString` に変換し、stderrに「警告: コマンドラインでのパスワード指定はセキュリティリスクがあります」と出力
2. `PQ_DIARY_PASSWORD` 環境変数が設定されている場合: `SecretString` に変換
3. `stdin.is_terminal()` が true の場合: `read_password_tty()` を呼び出し
4. いずれも該当しない場合: `DiaryError::Password` エラーを返す

**実装ファイル**: `cli/src/password.rs`

### 2. Cargo.toml依存関係追加 🔵

**信頼性**: 🔵 *設計文書 architecture.md*

cli/Cargo.toml にプラットフォーム依存の依存関係を追加する。

```toml
[target.'cfg(unix)'.dependencies]
nix = { version = "0.29", features = ["term"] }

[target.'cfg(windows)'.dependencies]
windows-sys = { version = "0.59", features = ["Win32_System_Console"] }
```

**実装ファイル**: `cli/Cargo.toml`

### 3. `read_password_tty()` - Unix実装 🔵

**信頼性**: 🔵 *設計文書 architecture.md、ADR-0003*

Unix環境でのエコーなしパスワード読み取りを実装する。

- `nix::sys::termios::tcgetattr()` で現在の端末設定を保存
- `ECHO` フラグを無効化して `tcsetattr()` で適用
- stdin からバイト単位で `SecretString` に読み取り
- 完了後（正常終了・エラー問わず）`tcsetattr()` で元の設定を復元
- 復元処理は `Drop` トレイトまたは RAII ガードパターンで確実に実行

**実装ファイル**: `cli/src/password.rs`

### 4. `read_password_tty()` - Windows実装 🔵

**信頼性**: 🔵 *設計文書 architecture.md、ADR-0003*

Windows環境でのエコーなしパスワード読み取りを実装する。

- `GetConsoleMode()` で現在のコンソールモードを保存
- `ENABLE_ECHO_INPUT` を無効化して `SetConsoleMode()` で適用
- `ReadConsoleW()` でUnicode入力を読み取り `SecretString` に変換
- 完了後に元のコンソールモードを復元

**実装ファイル**: `cli/src/password.rs`

## テスト要件

### フラグ優先順位テスト 🔵

- `--password` フラグが環境変数より優先されることを確認
- フラグ指定時に `PasswordSource` が `Flag` バリアントであることを確認
- stderrへの警告メッセージ出力を確認

### 環境変数優先順位テスト 🔵

- 環境変数がTTY入力より優先されることを確認
- `PasswordSource` が `Env` バリアントであることを確認

### TTYパスワード入力テスト 🔵

- フラグ・環境変数ともに未設定でTTY利用可能な場合にTTY入力にフォールバックすることを確認
- エコーが無効化されていることを確認（可能な範囲で）

### エラーケーステスト 🔵

- TTYでない環境でパスワード未指定の場合に `DiaryError::Password` が返ることを確認
- 空のパスワードが適切に処理されることを確認

### セキュリティテスト 🔵

- パスワードが `SecretString` に格納されていることを確認
- `--password` フラグ使用時の警告メッセージ内容を確認

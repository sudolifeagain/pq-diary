## タスク概要

エディタ起動時に使用するセキュアな一時ディレクトリの選定と、ファイル削除時のゼロ化消去を実装する。Unix環境ではメモリベースのファイルシステム（/dev/shm, /run/user/$UID）を優先し、Windows環境ではACLで保護された専用ディレクトリを使用する。ファイル削除時はランダムデータで上書き後に削除することで、復元リスクを低減する。

## 実装詳細

### 1. `secure_tmpdir()` - Unix実装 🔵

**信頼性**: 🔵 *要件定義書 REQ-061、REQ-064*

メモリベースファイルシステムを優先的に選定するセキュア一時ディレクトリ取得関数を実装する。

```rust
pub fn secure_tmpdir() -> Result<PathBuf, DiaryError>
```

**Unix選定順序**:
1. `/dev/shm` が存在し書き込み可能 → `/dev/shm/pq-diary-{uid}/` を作成して返す
2. `/run/user/{uid}` が存在し書き込み可能 → `/run/user/{uid}/pq-diary/` を作成して返す
3. `/tmp` にフォールバック → stderrに「警告: メモリベースの一時ディレクトリが利用できません。/tmp を使用します」と出力 → `/tmp/pq-diary-{uid}/` を作成して返す

ディレクトリのパーミッションは `0o700` (owner only) に設定する。

**実装ファイル**: `cli/src/editor.rs`

### 2. `secure_tmpdir()` - Windows実装 🔵

**信頼性**: 🔵 *要件定義書 REQ-061、REQ-064*

Windows環境でのセキュア一時ディレクトリを実装する。

- `%LOCALAPPDATA%\pq-diary\tmp\` ディレクトリを作成
- ACL設定でオーナーのみ読み書き可能に制限
- ディレクトリが既に存在する場合はそのまま使用

**実装ファイル**: `cli/src/editor.rs`

### 3. `secure_delete()` 関数 🔵

**信頼性**: 🔵 *要件定義書 REQ-065*

ファイルをランダムデータで上書きした後に削除する関数を実装する。

```rust
pub fn secure_delete(path: &Path) -> Result<(), DiaryError>
```

**処理手順**:
1. ファイルサイズを取得
2. `OsRng` でファイルサイズ分のランダムバイト列を生成
3. ファイルを開き（上書きモード）、ランダムデータを書き込み
4. `sync_all()` でディスクにフラッシュ
5. `std::fs::remove_file()` でファイルを削除

**実装ファイル**: `cli/src/editor.rs`

## テスト要件

### 一時ディレクトリ選定テスト 🔵

- Unix: /dev/shm が利用可能な場合に優先選択されることを確認
- Unix: /dev/shm が利用不可で /run/user/$UID が利用可能な場合のフォールバックを確認
- Unix: /tmp フォールバック時に警告メッセージが出力されることを確認
- Windows: %LOCALAPPDATA%\pq-diary\tmp\ が作成されることを確認
- 作成されたディレクトリのパーミッションが適切であることを確認

### セキュア削除テスト 🔵

- テストファイルを作成し、`secure_delete()` 後にファイルが存在しないことを確認
- 上書きが実行されたことを確認（ファイルサイズ分のデータが書き込まれたことの間接的な確認）
- 存在しないファイルに対する `secure_delete()` がエラーを返すことを確認

### エッジケーステスト 🔵

- 空ファイル（0バイト）に対する `secure_delete()` が正常に動作することを確認
- ディレクトリが既に存在する場合に `secure_tmpdir()` が正常に動作することを確認

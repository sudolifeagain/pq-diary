## タスク概要

外部エディタ（$EDITOR）の起動制御と、エディタで編集されたファイルのヘッダーコメント（タイトル・タグ）のパース機能を実装する。vim/nvim検出時のセキュリティオプション自動付与、一時ファイルへのヘッダー書き込みとパース、環境変数によるエディタ指定のフォールバック処理を含む。

## 実装詳細

### 1. `EditorConfig` 構造体 🔵

**信頼性**: 🔵 *設計文書 architecture.md*

エディタ起動に必要な設定を保持する構造体を定義する。

```rust
pub struct EditorConfig {
    pub command: String,
    pub vim_options: Vec<String>,
    pub secure_tmpdir: PathBuf,
}
```

- `$EDITOR` 環境変数からコマンドを取得
- vim/nvim の場合は `vim_options` に `-c "set noswapfile nobackup noundofile"` を設定
- `secure_tmpdir` は TASK-0035 の `secure_tmpdir()` から取得

**実装ファイル**: `cli/src/editor.rs`

### 2. `write_header_file()` 関数 🔵

**信頼性**: 🔵 *要件定義書 REQ-082*

一時ファイルにヘッダーコメント付きの内容を書き出す関数を実装する。

```rust
pub fn write_header_file(tmpdir: &Path, plaintext: &EntryPlaintext) -> Result<PathBuf, DiaryError>
```

**出力フォーマット**:
```
# Title: {title}
# Tags: {tag1}, {tag2}
# ---

{body}
```

- タグが空の場合は `# Tags:` 行を空のままにする
- ファイル名は UUID + `.md` 拡張子

**実装ファイル**: `cli/src/editor.rs`

### 3. `read_header_file()` 関数 🔵

**信頼性**: 🔵 *要件定義書 REQ-083*

ヘッダーコメントをパースしてメタデータと本文を抽出する関数を実装する。

```rust
pub fn read_header_file(path: &Path) -> Result<HeaderComment, DiaryError>
```

**パースルール**:
- `# Title: ` で始まる行 → タイトルを抽出
- `# Tags: ` で始まる行 → カンマ区切りでタグリストを抽出
- `# ---` → セパレータ（以降が本文）
- ヘッダーが不正な形式の場合: title=None, tags=None, body=ファイル全体の内容

**実装ファイル**: `cli/src/editor.rs`

### 4. `launch_editor()` 関数 🔵

**信頼性**: 🔵 *要件定義書 REQ-062, REQ-063*

外部エディタを起動し、編集完了を待機する関数を実装する。

```rust
pub fn launch_editor(tmpfile: &Path, config: &EditorConfig) -> Result<(), DiaryError>
```

**処理詳細**:
- `std::process::Command` でエディタプロセスを起動
- vim/nvim検出時: `-c "set noswapfile nobackup noundofile"` オプションを追加
- 環境変数 `$TMPDIR` (Unix) / `$TEMP` / `$TMP` (Windows) をセキュア一時ディレクトリに設定
- プロセス終了コードが非ゼロの場合: `DiaryError::Editor` を返す
- `$EDITOR` 未設定時のフォールバック: Unix → "vi", Windows → "notepad"

**実装ファイル**: `cli/src/editor.rs`

## テスト要件

### ヘッダー書き込み/パースラウンドトリップテスト 🔵

- タイトル・タグ・本文を `write_header_file()` で書き込み、`read_header_file()` で読み取った結果が元データと一致することを確認
- 空タグ、複数タグ、日本語タグでのラウンドトリップを確認

### 部分ヘッダーテスト 🔵

- タイトルのみ（タグなし）のヘッダーが正しくパースされることを確認
- タグのみ（タイトルなし）のヘッダーが正しくパースされることを確認

### 不正ヘッダーフォールバックテスト 🔵

- ヘッダーが全くないファイルの場合、title=None, tags=None, body=ファイル全体であることを確認
- セパレータ `# ---` がないファイルのフォールバック動作を確認

### vimオプション検出テスト 🔵

- `$EDITOR` が "vim" の場合にセキュリティオプションが付与されることを確認
- `$EDITOR` が "nvim" の場合にセキュリティオプションが付与されることを確認
- `$EDITOR` が "nano" 等の場合にvimオプションが付与されないことを確認
- `$EDITOR` 未設定時のフォールバック値を確認（Unix: "vi", Windows: "notepad"）

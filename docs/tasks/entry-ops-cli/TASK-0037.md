## 実装詳細

### 1. Commands enum 更新 🔵

**信頼性**: 🔵 *設計文書 architecture.md、types.rs*

`cli/src/main.rs` の `Commands` enum に `New` バリアントを追加する。

```rust
/// 新しいエントリを作成
New {
    /// エントリのタイトル
    title: Option<String>,
    /// 本文を直接指定（エディタを起動しない）
    #[arg(short, long)]
    body: Option<String>,
    /// タグ（複数指定可）
    #[arg(short, long)]
    tag: Vec<String>,
}
```

**実装ファイル**: `cli/src/main.rs`

### 2. `cmd_new()` ハンドラ実装 🔵

**信頼性**: 🔵 *要件定義書 REQ-001, REQ-011〜013*

新規エントリ作成の完全なフローを実装する。

```rust
pub fn cmd_new(cli: &Cli, title: Option<String>, body: Option<String>, tags: Vec<String>) -> Result<(), DiaryError>
```

**処理フロー**:
1. `get_password()` でパスワード取得 → Vaultアンロック
2. 本文取得の優先順位:
   - `-b` フラグが指定されている場合: そのまま本文として使用
   - stdinがパイプの場合: `read_to_string()` で全内容を読み取り
   - それ以外: `write_header_file()` → `launch_editor()` → `read_header_file()` → `secure_delete()`
3. タイトルが未指定または空の場合: "Untitled" をデフォルトとして使用
4. `create_entry(vault_path, engine, plaintext)` でエントリを作成
5. 標準出力に `Created: {prefix} "{title}"` を出力
6. `lock()` でVaultをロック

**実装ファイル**: `cli/src/commands.rs`

### 3. モジュール登録 🔵

**信頼性**: 🔵 *設計文書 architecture.md*

`cli/src/main.rs` に `mod commands;` を追加し、`Commands::New` のマッチアームから `cmd_new()` を呼び出す。

**実装ファイル**: `cli/src/main.rs`

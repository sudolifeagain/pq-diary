## タスク概要

エントリ一覧表示（list）と個別エントリ表示（show）のCLIコマンドを実装する。listコマンドはタグフィルタ、キーワード検索、表示件数制限に対応し、更新日時の降順でテーブル形式で表示する。showコマンドはIDプレフィックスによるエントリ特定と全文表示を行う。

### 3. `cmd_list()` ハンドラ実装 🔵

**信頼性**: 🔵 *要件定義書 REQ-002, REQ-031〜033*

エントリ一覧表示の完全なフローを実装する。

```rust
pub fn cmd_list(cli: &Cli, tag: Option<String>, query: Option<String>, number: usize) -> Result<(), DiaryError>
```

**処理フロー**:
1. `get_password()` → Vaultアンロック
2. `list_entries()` で全エントリを取得
3. フィルタリング:
   - `--tag` 指定時: `Tag::is_prefix_of()` による前方一致フィルタ（ネストタグ対応）
   - `-q` 指定時: タイトルの部分一致検索（大文字小文字を区別しない）
4. `updated_at` 降順でソート
5. `-n` で指定された件数に制限（デフォルト: 20）
6. テーブル形式で出力: `{prefix}  {date}  {title}  #{tags}`
7. `lock()` でVaultをロック

**実装ファイル**: `cli/src/commands.rs`

### 4. `cmd_show()` ハンドラ実装 🔵

**信頼性**: 🔵 *要件定義書 REQ-003, REQ-041〜043*

個別エントリの全文表示を実装する。

```rust
pub fn cmd_show(cli: &Cli, id: String) -> Result<(), DiaryError>
```

**処理フロー**:
1. `get_password()` → Vaultアンロック
2. `IdPrefix::new(id)` でプレフィックスを作成
3. `get_entry(prefix)` でエントリを取得
4. 表示内容:
   - タイトル
   - 作成日時 / 更新日時
   - タグ一覧
   - 空行
   - 本文全文
5. エントリが見つからない場合: `DiaryError::EntryNotFound` エラー
6. `lock()` でVaultをロック

**実装ファイル**: `cli/src/commands.rs`

## テスト要件

### タグフィルタテスト 🔵

- `--tag "日記"` でタグが完全一致するエントリのみ表示されることを確認
- `--tag "日記"` でネストタグ `日記/旅行` も前方一致で表示されることを確認
- 存在しないタグでフィルタした場合に空の結果が返ることを確認

### キーワード検索テスト 🔵

- `-q "テスト"` でタイトルに "テスト" を含むエントリが表示されることを確認
- 大文字小文字を区別しない検索が動作することを確認

### 表示件数テスト 🔵

- デフォルトで最大20件が表示されることを確認
- `-n 5` で最大5件に制限されることを確認
- エントリ数が指定件数未満の場合に全件表示されることを確認

### ソートテスト 🔵

- updated_atの降順（新しい順）でソートされていることを確認

### showコマンドテスト 🔵

- 4文字のIDプレフィックスでエントリが取得できることを確認
- 完全なIDでエントリが取得できることを確認
- 存在しないIDでエラーが返ることを確認
- プレフィックスが複数エントリに一致する場合の動作を確認

## タスク概要

既存エントリの編集コマンドを実装する。CLIフラグによるメタデータ（タイトル・タグ）の直接変更と、$EDITORを使用した対話的な全体編集の2モードに対応する。エディタ経由の編集では変更検出を行い、変更がない場合はスキップする。ヘッダーパースエラー時はメタデータを保持し本文のみ更新するフォールバック動作を実装する。

## 実装詳細

### 1. Commands enum 更新 - Edit 🔵

**信頼性**: 🔵 *設計文書 architecture.md、types.rs*

`Commands` enum に `Edit` バリアントを追加する。

```rust
/// エントリを編集
Edit {
    /// エントリID（プレフィックス可）
    id: String,
    /// タイトルを変更
    #[arg(long)]
    title: Option<String>,
    /// タグを追加（複数指定可）
    #[arg(long)]
    add_tag: Vec<String>,
    /// タグを削除（複数指定可）
    #[arg(long)]
    remove_tag: Vec<String>,
}
```

**実装ファイル**: `cli/src/main.rs`

### 2. `cmd_edit()` ハンドラ実装 - フラグモード 🔵

**信頼性**: 🔵 *要件定義書 REQ-004, REQ-014〜016*

CLIフラグによるメタデータ直接変更を実装する。

```rust
pub fn cmd_edit(cli: &Cli, id: String, title: Option<String>, add_tags: Vec<String>, remove_tags: Vec<String>) -> Result<(), DiaryError>
```

**フラグモード処理フロー**:
1. `get_password()` → Vaultアンロック
2. `get_entry(IdPrefix::new(id))` で既存エントリを取得
3. `--title` 指定時: タイトルを更新
4. `--add-tag` 指定時: タグを追加（重複チェック）
5. `--remove-tag` 指定時: タグを削除
6. `update_entry()` でエントリを更新
7. `lock()` でVaultをロック

**実装ファイル**: `cli/src/commands.rs`

### 3. `cmd_edit()` ハンドラ実装 - エディタモード 🔵

**信頼性**: 🔵 *要件定義書 REQ-082, REQ-083*

$EDITORを使用した対話的編集を実装する。

**エディタモード処理フロー**（`--title`, `--add-tag`, `--remove-tag` がいずれも未指定の場合）:
1. `get_password()` → Vaultアンロック
2. `get_entry()` で既存エントリを取得
3. `write_header_file()` で一時ファイルに書き出し
4. `launch_editor()` でエディタを起動
5. `read_header_file()` でパース
6. 変更検出:
   - タイトル、タグ、本文のすべてが元データと一致 → 「変更がありませんでした」と出力してスキップ
   - ヘッダーパースエラー → 元のメタデータを保持し、本文のみを更新
   - 変更あり → `update_entry()` で更新
7. `secure_delete()` で一時ファイルを削除
8. `lock()` でVaultをロック

**実装ファイル**: `cli/src/commands.rs`

## テスト要件

### フラグ編集テスト 🔵

- `--title "新タイトル"` でタイトルのみ変更されることを確認
- `--add-tag "新タグ"` でタグが追加されることを確認
- `--remove-tag "既存タグ"` でタグが削除されることを確認
- 複数フラグの同時指定が正しく動作することを確認
- 本文がフラグ編集時に変更されないことを確認

### エディタ編集テスト 🔵

- エディタ経由でタイトル・タグ・本文が更新されることを確認（エディタモック使用）
- エディタ起動後に一時ファイルがセキュア削除されることを確認

### 変更未検出テスト 🔵

- エディタで何も変更しなかった場合に「変更がありませんでした」が出力されることを確認
- 変更なしの場合に `update_entry()` が呼ばれないことを確認

### ヘッダーパースエラーフォールバックテスト 🔵

- ユーザーがヘッダーを削除した場合に元のメタデータが保持されることを確認
- ヘッダー不正の場合に本文のみ更新されることを確認

### エラーケーステスト 🔵

- 存在しないIDでエラーが返ることを確認
- 不正なパスワードでエラーが返ることを確認

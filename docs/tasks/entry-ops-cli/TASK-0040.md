## タスク概要

エントリ削除コマンドを実装する。削除前にエントリのタイトルと日時を表示し、ユーザーに確認プロンプトを表示する。`--force` フラグまたは `--claude` フラグで確認をスキップできる。確認プロンプトはデフォルト "No" で、明示的に "y" を入力した場合のみ削除を実行する。

### 2. `cmd_delete()` ハンドラ実装 🔵

**信頼性**: 🔵 *要件定義書 REQ-005, REQ-021, REQ-022*

エントリ削除の完全なフローを実装する。

```rust
pub fn cmd_delete(cli: &Cli, id: String, force: bool) -> Result<(), DiaryError>
```

**処理フロー**:
1. `get_password()` → Vaultアンロック
2. `get_entry(IdPrefix::new(id))` で対象エントリを取得
3. エントリのタイトルと日時を表示
4. 確認判定:
   - `--force` フラグが true → 確認スキップ
   - `--claude` グローバルフラグが true → 確認スキップ
   - それ以外 → 確認プロンプトを表示
5. 確認プロンプト表示:
   - `Delete "{title}" ({date})? [y/N]: ` をstderrに出力
   - stdinから1行読み取り
   - "y" または "Y" の場合 → 削除実行
   - それ以外（"n", "N", 空入力、その他） → キャンセル
6. 削除実行時:
   - `delete_entry()` でエントリを削除
   - `Deleted: {prefix} "{title}"` を出力
7. キャンセル時:
   - 「キャンセルしました」を出力
8. `lock()` でVaultをロック

**実装ファイル**: `cli/src/commands.rs`

### 3. 確認プロンプト関数 🔵

**信頼性**: 🔵 *設計文書 architecture.md*

テスト可能な確認プロンプト関数を分離して実装する。

```rust
fn confirm_delete(title: &str, date: &str, reader: &mut impl BufRead) -> Result<bool, DiaryError>
```

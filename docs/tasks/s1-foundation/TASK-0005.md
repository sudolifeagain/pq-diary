## 実装詳細

### 1. Cli 構造体（トップレベル） 🔵

**信頼性**: 🔵 *PRDセクション4.1・REQ-004より*

```rust
#[derive(Parser)]
#[command(name = "pq-diary", about = "Post-quantum cryptography CLI journal")]
struct Cli {
    #[arg(short, long, global = true)]
    vault: Option<String>,
    #[arg(long, global = true)]
    password: Option<String>,
    #[arg(long, global = true)]
    claude: bool,
    #[arg(long, global = true)]
    debug: bool,
    #[command(subcommand)]
    command: Commands,
}
```

**実装ファイル**: `cli/src/main.rs`

### 2. Commands 列挙型 🔵

**信頼性**: 🔵 *PRDセクション4.1の全コマンド体系より*

全サブコマンドを列挙。vault, legacy, daemon, template はネストしたサブコマンドを持つ。

**実装ファイル**: `cli/src/main.rs`

### 3. ディスパッチ関数 🔵

**信頼性**: 🔵 *dataflow.md CLIコマンドディスパッチフローより*

match 式で各コマンドを分岐。Sprint 1 では全コマンドが `unimplemented!("Planned for Sprint N")` を呼ぶ。ただし `--password` フラグ使用時の警告表示は Sprint 1 で実装しない。

**実装ファイル**: `cli/src/main.rs`

## 注意事項

- `--password` は非推奨だがPRDで定義されているため含める。使用時の警告はS4で実装
- clap derive の `#[command(about)]` で各コマンドに短い説明を付ける（英語）
- `unimplemented!()` ではなく `eprintln!` + `process::exit(1)` を使う（パニックを避ける）

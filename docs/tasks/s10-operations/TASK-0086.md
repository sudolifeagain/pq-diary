## タスク概要

`~/.pq-diary/config.toml` / `~/.pq-diary/vaults/` の絶対パスを解決するために、`dirs` クレートを `cli/Cargo.toml` に追加する。後続の TASK-0087 (`AppConfig::default_path()`) および init/sync コマンド実装で `dirs::home_dir()` を利用する。

設計ヒアリング Q1 で「ホームディレクトリ解決は手書きせず `dirs` クレートを採用する」が確定済み。最新メジャー版 (5.x) を使用する。

## 実装詳細

### 1. cli/Cargo.toml への追加

**信頼性**: 🔵 *設計ヒアリング Q1 (2026-05-17)*

`cli/Cargo.toml` の `[dependencies]` セクションに以下を追加する:

```toml
dirs = "5"
```

- バージョン指定は SemVer の最新メジャー (5.x) を採用する。`dirs` クレートは安定しており、5.x は MSRV 1.65 以降をサポート。
- `default-features = false` などの feature 操作は不要 (デフォルトのまま利用)。
- 追加位置はアルファベット順に挿入 (既存依存の順序規約に従う)。

### 2. 検証コマンド

**信頼性**: 🔵 *DIRECT 検証手順 (S1 ~ S9 のパターン踏襲)*

1. `cargo build --workspace --all-targets` で全クレートのビルドを確認
2. `cargo audit` で新規セキュリティ警告がないことを確認 (`cargo install cargo-audit` がインストール済み前提)
3. `cargo tree -p pq-diary` の出力に `dirs v5.x.x` が含まれることを確認

### 3. 利用箇所の確認 (実装は TASK-0087 で行う)

**信頼性**: 🔵 *types.rs L58-70*

本タスクでは `dirs` を import するコードは追加しない。TASK-0087 で `core/src/vault/config.rs` に以下のような利用が入る想定:

```rust
let home = dirs::home_dir()
    .ok_or_else(|| DiaryError::Config("Cannot determine home directory".to_string()))?;
```

ただし、`core/` クレートが `dirs` に依存するかは TASK-0087 の設計で決定する (現状の方針は `cli/` のみ依存)。本タスクでは `cli/Cargo.toml` のみ更新する。

## 注意事項

- 本タスクは DIRECT タスクのため、`cargo test` の新規追加は不要。ビルドと監査のみで完了とする
- `core/` クレートには `dirs` を **追加しない**。CLAUDE.md「core/ にプラットフォーム依存 UI コードを入れない」規約に従い、ホーム解決は CLI 層で実施する
- TASK-0087 で `AppConfig::default_path()` を `core/` に置くか `cli/` に置くかは、TASK-0087 設計時に再確認する。本タスクではあくまで `cli/` に `dirs` 依存を追加するのみ
- 既存依存の `serde` / `toml` / `anyhow` などのバージョンを意図せず上げてしまわないよう、`Cargo.lock` の差分を必ず目視確認すること
- `cargo audit` が未インストールの場合は `cargo install cargo-audit` で先にインストールする
- Rust 規約遵守: 本タスクではコード追加なしのため `unsafe` / `unwrap` / `zeroize` の規約は該当しない

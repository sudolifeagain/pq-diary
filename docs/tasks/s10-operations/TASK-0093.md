## タスク概要

Phase 1 で `not_implemented("info", "Sprint 2")` を返していた `pq-diary info` を本実装する。vault のメタ情報 (名前・ポリシー・エントリ数・作成日時・最終更新日時) を表示し、`--security` フラグ指定時には Argon2 パラメータ・KEM/DSA アルゴリズム名・プロセス強化状態 (mlock/coredump/debugger) を追加表示する。NFR-002 (応答 < 100ms, アンロック後)、NFR-104 (実状態反映、ハードコード "yes" 禁止)、NFR-202 (`pq-diary stats` と統一されたヘッダー + 左寄せ Label スタイル) を満たす。

## 実装詳細

### 1. dispatch 切り替え (cli/src/main.rs)

**信頼性**: 🔵 *cli-commands.md §4 dispatch 変更*

`Commands::Info { security }` のディスパッチを置換する:

```rust
- Commands::Info { .. } => not_implemented("info", "Sprint 2"),
+ Commands::Info { security } => commands::cmd_info(cli, *security),
```

- `Commands::Info` には clap で `#[arg(long)] security: bool` が既に定義済 (Phase 1 スケルトン)。dispatch でのみ `security` を unwrap して `cmd_info` に渡す
- 既存 `not_implemented("info", "Sprint 2")` の呼び出しは完全に消去する

### 2. `cmd_info(cli: &Cli, security: bool) -> anyhow::Result<()>` 実装 (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-401~412 + dataflow.md info シーケンス*

実装順序:

1. **vault path 解決**: `cli.vault` が指定されていればそれ、未指定なら `AppConfig::from_file(AppConfig::default_path()?)?` を読み `config.app.default_vault` から `~/.pq-diary/vaults/{name}/` を解決 (TASK-0087 + TASK-0089 と同じパターン)
2. **パスワード取得 (REQ-404)**: 既存 `get_password(cli, "Password: ")` で 3 段階 (--password > env > TTY) 取得
3. **vault unlock**: `DiaryCore::open(&vault_path)?.unlock(&pwd)?` を呼ぶ。失敗時 `anyhow::bail!("Vault unlock failed")` (TC-401-E01)
4. **vault.toml 読み取り (REQ-401)**: `DiaryCore` 経由で `VaultConfig` (name, policy, argon2 params) を取得。破損時 `DiaryError::Config("Invalid vault.toml: {error}")` が伝搬 (TC-401-E02)
5. **エントリ数取得 (REQ-401, REQ-404)**: `entry::list_entries(&core)?.len()` でアンロック後の vault payload から件数取得 (NFR-002 < 100ms 目標のため list_entries は title/created のみ、本文復号は不要)
6. **created / modified 取得 (REQ-402, REQ-403)**: `fs::metadata(vault_path.join("vault.pqd"))?` で `created()` / `modified()` を取得。Unix で `created()` が `Err` を返す環境 (古い ext4 等) では ctime にフォールバック (REQ-402 ノート)。`chrono::DateTime<Utc>` に変換し ISO 8601 表示
7. **基本情報を出力**: NFR-202 の左寄せ Label スタイルで `=== Vault Info ===` ヘッダー + 6 項目を表示
8. **`--security` 分岐 (REQ-411, REQ-412)**:
   - **REQ-411**: `VaultConfig` から Argon2 params (`memory_cost_kb`, `time_cost`, `parallelism`) を表示。KEM / DSA は固定文字列 `"ML-KEM-768"` / `"ML-DSA-65"` (PRD §3 で固定、vault.toml に格納されているが現状値は固定)
   - **REQ-412**: `cli::security::harden_status()` (TASK-0088) を呼び `HardenStatus { mlock_active, coredump_disabled, debugger_detected }` を取得し `yes`/`no` 表示
   - NFR-104: ハードコード値ではなく実プロセス状態を反映 (`HardenStatus::current()` 内で `/proc/self/status` Linux, `prctl` Linux, `IsDebuggerPresent` Windows 等を呼ぶ — TASK-0088 で実装済)
9. **Ok(()) 返却**

### 3. 出力フォーマット (NFR-202)

**信頼性**: 🔵 *cli-commands.md §4 出力サンプル + ヒアリング2026-05-17 確定提案J*

`pq-diary stats` の `=== Vault Statistics ===` ヘッダー + `Label:    value` 左寄せスタイルと完全に揃える。

**デフォルト (`info`)**:

```
=== Vault Info ===
Name:           default
Policy:         none
Entries:        42
Created:        2026-04-15 09:23:11 UTC
Last updated:   2026-05-17 11:42:03 UTC
```

**`info --security`**:

```
=== Vault Info ===
Name:           default
Policy:         none
Entries:        42
Created:        2026-04-15 09:23:11 UTC
Last updated:   2026-05-17 11:42:03 UTC

=== Security ===
KEM algorithm:        ML-KEM-768
Signature algorithm:  ML-DSA-65
Argon2 memory:        65536 KB
Argon2 time cost:     3
Argon2 parallelism:   1
mlock active:         yes
Coredump disabled:    yes
Debugger detected:    no
```

- Label カラム幅は固定 (16 文字、コロン込み)。`Security` セクションは Label が長いため 22 文字幅に拡張
- 値カラムはタブではなく半角スペースで揃える (CI ログでの一貫表示)
- `Vault Info` セクションと `Security` セクションの間に空行 1 行

### 4. メモリ保護

**信頼性**: 🔵 *CLAUDE.md Rust 規約 + NFR-101*

- パスワードは `SecretString` で受け取り、`cmd_info` スコープ離脱時に Drop で zeroize
- `DiaryCore::unlock` 内で導出される鍵は `ZeroizingKey` で保持 (S2 既存)
- `list_entries` は title/created のみ取得し本文を復号しない (NFR-002 高速化 + 平文露出最小化)

### 5. NFR-002 (info < 100ms, アンロック後) 対応

**信頼性**: 🔵 *NFR-002 + TC-401-03*

- パスワード入力 + Argon2 鍵導出時間は計測対象外 (NFR-002 はアンロック後の応答時間)
- `list_entries` は entries インデックスのみ読み (本文復号せず) で O(N) だが 100 エントリで 100ms 以内に収まる想定
- `fs::metadata` は OS のキャッシュヒットで µs オーダー
- `harden_status()` は `/proc/self/status` 読み + 1 回の `prctl` で ~1ms 以内 (TASK-0088 で計測)
- テストでは Argon2 fast_params を使い計測安定化 (S9 TASK-0085 パターン)

**TC-401-01** 🔵: `tempfile::TempDir` で一時 HOME + AppConfig + default vault (Argon2 fast_params) を作成、エントリ 3 件を追加。`cmd_info(&cli, false)` を呼び stdout を capture (test_helpers の stdout capture か gag crate 利用)。出力に "=== Vault Info ===", "Name: default", "Policy: none", "Entries: 3", "Created: ", "Last updated: " の全 6 項目が含まれることを `assert!(output.contains(...))` で検証。

**TC-401-02** 🔵: TC-401-01 のサブセット。明示的に `"Entries:        3"` (左寄せ揃え後) が含まれることを正確一致 (空白数まで) で検証。

**TC-401-03** 🔵: TC-401-01 と同じセットアップ。事前に `DiaryCore::unlock` まで実行し、その後 vault.toml 読み + entry count + metadata 取得部分 (cmd_info の core ロジック) を `std::time::Instant::now()` で計測。閾値 < 100ms を fast_params 環境で検証。本番パラメータでの 100ms 計測は `#[ignore]` で別途用意。

**TC-411-01** 🔵: TC-401-01 と同じ vault に `cmd_info(&cli, true)` を呼ぶ。出力に "=== Security ===", "Argon2 memory:", "Argon2 time cost:", "Argon2 parallelism:" を含むことを検証。fast_params のため値は (例) `memory_cost_kb=32`, `time_cost=1`, `parallelism=1` だが key の存在のみ検証 (値は環境依存)。

**TC-411-02** 🔵: TC-411-01 の発展。出力に "KEM algorithm:        ML-KEM-768" と "Signature algorithm:  ML-DSA-65" が含まれることを正確一致 (空白揃え込み) で検証。固定文字列のためバージョン非依存。

**TC-412-01** 🔵: `cmd_info(&cli, true)` 出力の "mlock active:" 行を抽出し、`HardenStatus::current().mlock_active` (TASK-0088) と一致することを検証。テスト環境でも実プロセス状態を取得するため、テスト内で `harden_process()` を呼ばず素の状態で実行 (mlock 不可な環境では `no` 期待)。プラットフォーム依存テストは `#[cfg(unix)]` ガードで分岐。

**TC-412-02** 🔵: TC-412-01 と同様、"Coredump disabled:" 行を抽出し `HardenStatus::current().coredump_disabled` と一致を検証。

**TC-412-03** 🔵: 同様、"Debugger detected:" 行を抽出し `HardenStatus::current().debugger_detected` と一致を検証。CI 環境ではデバッガ非接続なので `no` 期待。

## UI/UX 要件

**信頼性**: 🔵 *cli-commands.md §4 出力サンプル + NFR-202 + ヒアリング2026-05-17 確定提案J*

### CLI 出力スタイル (NFR-202)

- **ヘッダー**: `=== Vault Info ===` および `=== Security ===` (stats の `=== Vault Statistics ===` と統一スタイル)
- **Label カラム**: 左寄せ、コロン込みで 16 文字幅 (Vault Info) / 22 文字幅 (Security)
- **値カラム**: Label カラム直後から開始、半角スペースで埋める (タブ禁止)
- **空行**: Vault Info と Security セクションの間に 1 行
- **日時表示**: `YYYY-MM-DD HH:MM:SS UTC` 形式 (chrono `%Y-%m-%d %H:%M:%S UTC`)
- **boolean 表示**: `yes` / `no` (小文字、true/false ではない)
- **stdout のみ**: エラーは stderr、情報表示は stdout

### 対話プロンプト

- TTY 環境: `Password: ` プロンプトを表示し echo オフで入力受付 (既存 `get_password` パターン)
- `--password` フラグ使用時: insecure 警告 (既存パターン)
- `PQ_DIARY_PASSWORD` 環境変数使用時: 警告なし

### エラー出力

- パスワード不正: `Error: Vault unlock failed` (stderr、内部詳細 (HMAC mismatch 等) は表示しない — 情報リーク防止)
- vault.toml 破損: `Error: Invalid vault.toml: {serde_toml error detail}` (stderr、内部詳細表示は OK — 復旧支援のため)

## 注意事項 (CLAUDE.md Rust 規約)

- `unsafe` は使用しない (`cmd_info` 内では直接 prctl/IsDebuggerPresent を呼ばず、TASK-0088 で実装済の `harden_status()` 経由で取得する)
- パスワードは必ず `SecretString` で受け取り、生の `String` で保持しない。`DiaryCore::unlock` には `&SecretString` を渡す
- エントリ本文は `cmd_info` では一切復号しない (件数のみ取得、平文露出最小化)
- `unwrap()` / `expect()` はテストコードのみ許可。本番コードは全て `Result` 返却 + `?` 演算子
- エラーは `cli/` レイヤなので `anyhow` を許可 (`anyhow::bail!`, `anyhow::Context`)。`core/` から伝搬する `DiaryError::Crypto` / `DiaryError::Config` / `DiaryError::Io` は `?` で anyhow に変換
- KEM/DSA アルゴリズム名は将来的に vault.toml から読む可能性があるが、現状 PRD で固定のため文字列定数で OK (REQ-411 が "ML-KEM-768" / "ML-DSA-65" 固定を明記)
- プラットフォーム分岐 (mlock 不可能環境、デバッガ検知非対応環境) は TASK-0088 の `HardenStatus::current()` 内で吸収済。`cmd_info` 側に `#[cfg]` を入れない
- パスワード不正時のエラーメッセージは内部詳細を出さない (HMAC mismatch 等は隠す — 情報リーク防止)。vault.toml 破損時は復旧支援のため内部詳細を出して OK
- doc comment (`///`) を `cmd_info` の公開 API には英語で付与する (S9 TASK-0085 と同じドキュメント規約)
- 日時表示は UTC 固定 (ローカルタイム表示はユーザーの誤解を招くため NFR-202 統一スタイルに従い UTC のみ)

## タスク概要

`pq-diary export <DIR>` コマンドを実装する。指定ディレクトリに全エントリを Markdown ファイル (YAML フロントマター付き) で書き出す機能で、平文出力のため強い警告プロンプトと `--claude` ブロックを伴う。

`Commands::Export` を `Commands::Export(ExportArgs)` に変更して引数 `<DIR>` を受け取り、`cli/src/commands.rs` に `cmd_export()` と 4 つのプライベートヘルパー (`slugify`, `yaml_escape`, `build_export_filename`, `build_export_content`) を追加する。出力ファイル名は `YYYY-MM-DD-{slug}-{id8}.md` 形式、YAML フロントマターは手書き (外部 yaml クレート不使用) で `id`, `title`, `tags`, `created`, `updated` を含む。

### 2. `cmd_export` 関数

**信頼性**: 🔵 *cli-commands.md サブコマンド 5 + dataflow.md 機能 5 シーケンス*

```rust
pub fn cmd_export(cli: &Cli, dir: PathBuf) -> anyhow::Result<()> {
    // 1. --claude ブロック (REQ-511)
    if cli.claude {
        anyhow::bail!("export is not permitted with --claude");
    }

    // 2. DIR 存在チェック (REQ-512)
    if !dir.exists() {
        anyhow::bail!("Directory does not exist: {}", dir.display());
    }

    // 3. 警告プロンプト (REQ-504)
    eprint!("平文を {} に書き出します。続行しますか? [y/N] ", dir.display());
    io::stderr().flush()?;
    let mut answer = String::new();
    io::stdin().read_line(&mut answer)?;
    let answer = answer.trim();

    // 4. y 以外でキャンセル (REQ-513)
    if !answer.eq_ignore_ascii_case("y") {
        println!("キャンセルしました");
        return Ok(());
    }

    // 5. password + unlock
    let pwd = get_password(cli)?;
    let vault_dir = resolve_vault_dir(cli)?;
    let mut core = DiaryCore::new(&vault_dir)?;
    core.unlock(&pwd)?;

    // 6. 全エントリ取得
    let entries = list_entries_with_body(&core)?;

    // 7. 空 vault (EDGE-102)
    if entries.is_empty() {
        println!("No entries to export");
        return Ok(());
    }

    // 8. 各エントリで書き出し
    for entry in &entries {
        let filename = build_export_filename(entry);
        let path = dir.join(&filename);
        // 同名既存チェック (EDGE-004)
        if path.exists() {
            anyhow::bail!("File exists: {}", path.display());
        }
        let content = build_export_content(entry);
        fs::write(&path, content)?;
    }

    // 9. 完了メッセージ (REQ-505)
    println!("Exported {} entries to {}", entries.len(), dir.display());
    Ok(())
}
```

### 3. `slugify` ヘルパー

**信頼性**: 🔵 *EDGE-103, dataflow.md ファイル名生成ロジック*

```rust
fn slugify(title: &str) -> String {
    // 1. trim + 空白を - に置換
    // 2. 制御文字 (U+0000〜U+001F) を除去
    // 3. ファイルシステム禁止文字 (/, \, :, *, ?, ", <, >, |) を除去
    // 4. 空ならば "untitled"
    let s: String = title
        .chars()
        .filter(|c| !c.is_control())
        .map(|c| match c {
            ' ' | '\t' => '-',
            '/' | '\\' | ':' | '*' | '?' | '"' | '<' | '>' | '|' => '_',
            _ => c,
        })
        .collect();
    let s = s.trim_matches('-').to_string();
    if s.is_empty() {
        "untitled".to_string()
    } else {
        s
    }
}
```

**例**:
- `"今日の出来事 #1"` → `"今日の出来事-#1"`
- `""` → `"untitled"`
- `"   "` → `"untitled"`
- `"foo/bar"` → `"foo_bar"`

### 4. `yaml_escape` ヘルパー

**信頼性**: 🔵 *schema.md YAML エスケープ規則*

```rust
fn yaml_escape(s: &str) -> String {
    s.chars()
        .filter(|c| !c.is_control() || *c == '\n')
        .map(|c| match c {
            '"' => "\\\"".to_string(),
            '\\' => "\\\\".to_string(),
            '\n' => "\\n".to_string(),
            _ => c.to_string(),
        })
        .collect()
}
```

エスケープ規則 (schema.md より):
- `"` → `\"`
- `\` → `\\`
- 改行 → `\n` (リテラル)
- 制御文字 (改行除く) → 削除

### 5. `build_export_filename` ヘルパー

**信頼性**: 🔵 *REQ-502, schema.md ファイル名仕様*

```rust
fn build_export_filename(entry: &EntryPlaintext) -> String {
    let date = entry.created_ts_utc().format("%Y-%m-%d");
    let slug = slugify(&entry.title);
    let id8 = &entry.uuid.to_string()[..8];
    format!("{}-{}-{}.md", date, slug, id8)
}
```

**例**:
- `2026-05-17-今日の出来事-1-3c6b775f.md`
- `2026-04-01-untitled-d65004a8.md`

### 6. `build_export_content` ヘルパー

**信頼性**: 🔵 *REQ-503, schema.md YAML フロントマター仕様*

```rust
fn build_export_content(entry: &EntryPlaintext) -> String {
    let mut out = String::new();
    out.push_str("---\n");
    out.push_str(&format!("id: {}\n", entry.uuid));
    out.push_str(&format!("title: \"{}\"\n", yaml_escape(&entry.title)));
    if entry.tags.is_empty() {
        out.push_str("tags: []\n");
    } else {
        out.push_str("tags:\n");
        for tag in &entry.tags {
            out.push_str(&format!("  - \"{}\"\n", yaml_escape(tag)));
        }
    }
    out.push_str(&format!("created: {}\n", entry.created_ts_utc().to_rfc3339()));
    out.push_str(&format!("updated: {}\n", entry.updated_ts_utc().to_rfc3339()));
    out.push_str("---\n\n");
    // 本文はそのままコピー (round-trip 保証)
    out.push_str(entry.body_str());
    out
}
```

**出力例**:
```markdown
---
id: 3c6b775f-4d8e-4c2b-9a1f-8d5e1f0a2b3c
title: "今日の出来事 #1"
tags:
  - "test"
  - "smoke"
created: 2026-05-17T09:23:11+00:00
updated: 2026-05-17T11:42:03+00:00
---

ここからエントリ本文
```

### 7. メモリ管理

**信頼性**: 🔵 *CLAUDE.md Rust 規約 + NFR-102*

- `EntryPlaintext::body` は `SecretBytes` で保持されているが、export は意図的に平文ファイル化する操作
- `body_str()` で取得した `&str` を `String::push_str` でコピーするため、出力ファイル内容はメモリに通常文字列として展開される
- `EntryPlaintext` 自体は `ZeroizeOnDrop` 実装済 (S5) のため、エントリの SecretBytes 部分はスコープ離脱で zeroize される
- **ただし、書き出した平文ファイル自体は zeroize されない** (NFR-102 — 警告プロンプトで明示)

## 注意事項

- `Commands::Export` の enum 形状変更は破壊的なため、CLI 引数パーサーと dispatch の両方を必ず更新する
- YAML フロントマターは手書きとし、外部 `serde_yaml` クレートには依存しない (依存ツリーを最小化)
- `slugify` は Unicode 文字 (日本語等) をそのまま許可する (Obsidian インポーター と対称、note.md L: importer.rs 参照)
- ファイルシステム禁止文字 (Windows: `/ \ : * ? " < > |`) は `_` に置換する (slugify 仕様)
- `body_str()` メソッドが `EntryPlaintext` に存在しない場合は、`expose_secret` 経由で `&str` を取得するヘルパーを追加するか、`body` を `SecretBytes` から `String` 化する処理を実装する (TDD-red で確認)
- `EntryPlaintext` の `created_ts` / `updated_ts` フィールド名と型 (`u64` UNIX タイムスタンプ) は既存実装に従う。`chrono::DateTime<Utc>` への変換は `chrono::DateTime::from_timestamp` を使う
- パフォーマンステスト (TC-501-03) は fast_params() を使用し、S8 で発生したタイミング不安定問題を回避すること
- NFR-102 に従い、出力ファイルは zeroize されないことをユーザーに警告プロンプトで明示する (REQ-504 の警告文がその役割を果たす)
- 警告プロンプトの y/N 判定は `eq_ignore_ascii_case("y")` で大小文字非依存とする (既存 `confirm_full_policy` パターン参考)
- stdin / stdout / stderr のリダイレクトを伴うテストでは `assert_cmd` クレートまたは関数を分離してロジック単体でテストすることを推奨
- 1000 エントリ作成テスト (TC-501-03) は通常テストで遅すぎる場合は `#[ignore]` 付きで実装し、`cargo test -- --ignored` で実行する形でも許容

## タスク概要

CI で CLI 整合性を継続的に保証するため、smoke test スクリプトを Unix (bash) / Windows (PowerShell) の両系統で新規作成する。スクリプトは (1) 全トップレベルコマンドの `--help` exit 0、(2) ヘルプから legacy / daemon の除外、(3) init → new → list → info → export の最小 E2E フロー、の 3 観点を検証する。

設計文書 cli-commands.md L371-421 の骨子をベースに、`PQ_DIARY_PASSWORD` 環境変数によるパスワード入力と `HOME` / `USERPROFILE` の一時化を行うことで、CI 環境で対話入力なしに完走できる構成とする。

完成したスクリプトを `.github/workflows/ci.yml` に統合し、`cargo build --release` 後の検証ステップとして組み込む。

## 実装詳細

### 1. `ci/smoke-test.sh` (Unix 版)

**信頼性**: 🔵 *design cli-commands.md L375-409 骨子*

`set -euo pipefail` で厳格モードを有効化し、PASS / FAIL カウンタを保持して最後に集計する。設計骨子をベースに以下を実装する:

```bash
#!/usr/bin/env bash
set -euo pipefail
BIN="${1:-./target/release/pq-diary}"
PASS=0; FAIL=0
pass() { echo "[PASS] $*"; PASS=$((PASS+1)); }
fail() { echo "[FAIL] $*" >&2; FAIL=$((FAIL+1)); }

# 1. 全ヘルプ exit 0
for cmd in init sync change-password info export new list show edit delete \
           today search stats import vault git-init git-push git-pull \
           git-sync git-status template; do
  if "$BIN" "$cmd" --help >/dev/null 2>&1; then
    pass "$cmd --help"
  else
    fail "$cmd --help"
  fi
done

# 2. ヘルプに legacy / daemon が含まれない
HELP=$("$BIN" --help 2>&1)
if echo "$HELP" | grep -q legacy; then fail "help contains 'legacy'"; else pass "help no 'legacy'"; fi
if echo "$HELP" | grep -q daemon; then fail "help contains 'daemon'"; else pass "help no 'daemon'"; fi

# 3. E2E 最小フロー
TMPDIR=$(mktemp -d)
trap "rm -rf '$TMPDIR'" EXIT
export HOME="$TMPDIR"
export PQ_DIARY_PASSWORD="SmokeTest123!"

"$BIN" init && pass "E2E: init" || fail "E2E: init"
"$BIN" new "smoke" --body "smoke body" && pass "E2E: new" || fail "E2E: new"
"$BIN" list | grep -q smoke && pass "E2E: list contains smoke" || fail "E2E: list contains smoke"
"$BIN" info && pass "E2E: info" || fail "E2E: info"

mkdir "$TMPDIR/out"
echo y | "$BIN" export "$TMPDIR/out"
if [ "$(ls "$TMPDIR/out" | wc -l)" -eq 1 ]; then
  pass "E2E: export 1 file"
else
  fail "E2E: export (expected 1 file)"
fi

echo "===== $PASS passed, $FAIL failed ====="
[ "$FAIL" -eq 0 ]
```

**実装ポイント**:
- 第 1 引数でバイナリパスを指定可能 (デフォルト `./target/release/pq-diary`)。CI と手元検証の両方で再利用可能
- `trap "rm -rf '$TMPDIR'" EXIT` で異常終了時も一時ディレクトリをクリーンアップ
- `HOME` を一時ディレクトリに上書きすることで `~/.pq-diary/` が `$TMPDIR/.pq-diary/` に解決される
- `PQ_DIARY_PASSWORD` 環境変数によりパスワードプロンプトをバイパス (TASK-0089 init 実装で対応想定)
- `echo y | ... export` で export 時の上書き確認プロンプトを yes 応答

### 2. `ci/smoke-test.ps1` (Windows 版)

**信頼性**: 🔵 *design cli-commands.md L411-413*

bash 版と同等の処理を PowerShell で実装する。`USERPROFILE` を一時化し、`$env:PQ_DIARY_PASSWORD` を設定する:

```powershell
#!/usr/bin/env pwsh
[CmdletBinding()]
param(
    [string]$Bin = "./target/release/pq-diary.exe"
)

$ErrorActionPreference = "Continue"
$script:Pass = 0
$script:Fail = 0

function Assert-Pass([string]$Name) {
    Write-Host "[PASS] $Name"
    $script:Pass++
}

function Assert-Fail([string]$Name) {
    Write-Host "[FAIL] $Name" -ForegroundColor Red
    $script:Fail++
}

# 1. 全ヘルプ exit 0
$commands = @(
    'init', 'sync', 'change-password', 'info', 'export',
    'new', 'list', 'show', 'edit', 'delete', 'today',
    'search', 'stats', 'import', 'vault',
    'git-init', 'git-push', 'git-pull', 'git-sync', 'git-status',
    'template'
)
foreach ($cmd in $commands) {
    & $Bin $cmd --help *> $null
    if ($LASTEXITCODE -eq 0) { Assert-Pass "$cmd --help" } else { Assert-Fail "$cmd --help" }
}

# 2. ヘルプに legacy / daemon が含まれない
$help = & $Bin --help 2>&1 | Out-String
if ($help -match 'legacy') { Assert-Fail "help contains 'legacy'" } else { Assert-Pass "help no 'legacy'" }
if ($help -match 'daemon') { Assert-Fail "help contains 'daemon'" } else { Assert-Pass "help no 'daemon'" }

# 3. E2E
$tmp = New-Item -ItemType Directory -Path (Join-Path $env:TEMP "pqd-smoke-$(Get-Random)")
try {
    $env:USERPROFILE = $tmp.FullName
    $env:HOME = $tmp.FullName
    $env:PQ_DIARY_PASSWORD = "SmokeTest123!"

    & $Bin init
    if ($LASTEXITCODE -eq 0) { Assert-Pass "E2E: init" } else { Assert-Fail "E2E: init" }

    & $Bin new "smoke" --body "smoke body"
    if ($LASTEXITCODE -eq 0) { Assert-Pass "E2E: new" } else { Assert-Fail "E2E: new" }

    $list = & $Bin list 2>&1 | Out-String
    if ($list -match 'smoke') { Assert-Pass "E2E: list contains smoke" } else { Assert-Fail "E2E: list contains smoke" }

    & $Bin info
    if ($LASTEXITCODE -eq 0) { Assert-Pass "E2E: info" } else { Assert-Fail "E2E: info" }

    $outDir = Join-Path $tmp.FullName "out"
    New-Item -ItemType Directory -Path $outDir | Out-Null
    "y" | & $Bin export $outDir
    $count = (Get-ChildItem -Path $outDir -File).Count
    if ($count -eq 1) { Assert-Pass "E2E: export 1 file" } else { Assert-Fail "E2E: export (expected 1, got $count)" }
}
finally {
    Remove-Item -Recurse -Force $tmp.FullName -ErrorAction SilentlyContinue
}

Write-Host "===== $script:Pass passed, $script:Fail failed ====="
if ($script:Fail -ne 0) { exit 1 } else { exit 0 }
```

**実装ポイント**:
- PowerShell 7+ (`pwsh`) を前提とする (Windows CI 標準)
- `$LASTEXITCODE` で各サブプロセスの終了コードを判定
- `USERPROFILE` と `HOME` の両方を一時化 (Rust の `dirs` クレートはプラットフォームによって参照変数が異なるため)
- `finally` ブロックで確実にクリーンアップ

### 3. `.github/workflows/ci.yml` への統合

**信頼性**: 🔵 *design cli-commands.md L417-421*

既存 CI ワークフローに以下のジョブステップを追加する。release ビルド完了後に smoke test を実行:

```yaml
- name: Build release binary
  run: cargo build --release --workspace

- name: CLI smoke test (Unix)

  run: |
    chmod +x ./ci/smoke-test.sh
    ./ci/smoke-test.sh ./target/release/pq-diary

- name: CLI smoke test (Windows)

  shell: pwsh
```

`.github/workflows/ci.yml` が未作成の場合は新規作成。既存 CI 設定がある場合はビルドステップの後に追記する。matrix 構成 (ubuntu / windows) があれば `if: runner.os` で分岐する。

## 注意事項

- スクリプトは `PQ_DIARY_PASSWORD` 環境変数経由でのパスワード入力を前提とする。TASK-0089 (init コマンド本実装) で当該環境変数の読み取りが未対応の場合、smoke test も同様に失敗するため、TASK-0089 の完了確認後に本タスクを実施すること
- TASK-0095 完了前に本タスクを実行すると、`--help` 出力に legacy/daemon が含まれるため "help no 'legacy'" / "help no 'daemon'" チェックが FAIL する。依存順を守ること
- bash 版で `mktemp -d` の挙動が macOS と Linux で微妙に異なるが、設計骨子のとおり `mktemp -d` で互換性確保可能
- Windows 版で `USERPROFILE` と `HOME` を両方上書きする理由: Rust の `dirs::home_dir()` は OS によって参照変数が異なるため、両方設定することで CLI 側の実装差異を吸収する
- export の `echo y | ...` パイプはプロンプト応答用。export 実装で `--yes` フラグが追加された場合はそちらを利用するほうがよりロバストになる (TASK-0094 完成時の挙動次第)
- `set -euo pipefail` 下で `|| fail "..."` のパターンを使うと `$?` が伝播してスクリプトが途中終了する場合がある。`if ... ; then ... else ... fi` 形式に書き直しても可
- Rust 規約遵守: 本タスクはシェルスクリプト + YAML のみで Rust コード変更なし。`unsafe` / `unwrap` / `zeroize` 規約には抵触しない
- 既存 CI が無い場合 `.github/workflows/ci.yml` を新規作成し、最低限 `actions/checkout@v4` + `actions-rust-lang/setup-rust-toolchain@v1` + `cargo build --release` + smoke test の構成にすること

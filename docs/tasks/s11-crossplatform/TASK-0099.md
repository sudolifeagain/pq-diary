## タスク概要

`.github/workflows/ci.yml` の `check` ジョブを ubuntu-latest 単独から `ubuntu-latest + macos-latest + windows-latest` の matrix に拡張。fail-fast: false で 1 OS 失敗時も他 OS は最後まで走らせる。

## 完了条件

- [ ] `check` ジョブに `strategy.matrix.os = [ubuntu-latest, macos-latest, windows-latest]` が設定
- [ ] `strategy.fail-fast: false`
- [ ] `runs-on: ${{ matrix.os }}`
- [ ] 各 OS で `cargo build/test/clippy --workspace --locked --all-targets` が実行される
- [ ] `cargo install cargo-audit` / `cargo audit` ステップは check から削除 (TASK-0101 で audit ジョブに移動)
- [ ] PR push 後の CI ログで `check (ubuntu-latest)` / `check (macos-latest)` / `check (windows-latest)` の 3 ジョブが並列表示される

## 実装詳細

`.github/workflows/ci.yml` の `check:` を以下に置換:

```yaml
check:
  name: check
  strategy:
    fail-fast: false
    matrix:
      os: [ubuntu-latest, macos-latest, windows-latest]
  runs-on: ${{ matrix.os }}
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@stable
    - uses: Swatinem/rust-cache@v2
    - run: cargo build --workspace --locked
    - run: cargo test --workspace --locked
    - run: cargo clippy --workspace --locked --all-targets -- -D warnings
```

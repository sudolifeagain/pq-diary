## タスク概要

`cargo audit` は脆弱性 DB を 1 回見るだけで OS 差異がないため、`check` の 3 OS matrix で重複実行するのはムダ。`check` から audit 関連 2 ステップを切り出し、`audit` 独立ジョブを ubuntu のみで走らせる。

## 完了条件

- [ ] `.github/workflows/ci.yml` に `audit:` ジョブが存在 (runs-on: ubuntu-latest)
- [ ] `audit` ジョブが `cargo install cargo-audit --locked` + `cargo audit` の 2 ステップを持つ
- [ ] `check` ジョブから上記 2 ステップが削除されている (TASK-0099 で既に削除済み)
- [ ] PR push 後の CI ログで `audit` ジョブが 1 つだけ表示される (3 OS 重複なし)

## 実装詳細

`.github/workflows/ci.yml` の `check:` の後、`smoke:` の前に以下を挿入:

```yaml
audit:
  name: audit
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@stable
    - uses: Swatinem/rust-cache@v2
    - run: cargo install cargo-audit --locked
    - run: cargo audit
```

## 概要

ファイル末尾パディング(512-4096B)、エントリパディング(0-255B)の生成。検証トークン(32Bランダム→AES-GCM暗号化→ヘッダ格納)の生成・検証。S2のCryptoEngineと連携。

## 完了条件

- [ ] generate_file_padding() → 512-4096B ランダムデータ
- [ ] generate_entry_padding() → 0-255B ランダムデータ
- [ ] generate_verification_token(sym_key) → (iv, ciphertext)
- [ ] verify_token(sym_key, iv, ciphertext) → Result<bool>
- [ ] 全テストパス

## 実装詳細

1. **ファイルパディング生成**: OsRngで512-4096B範囲のランダムデータを生成し、ファイル末尾に付与 🔵 PRDセクション6.1
2. **エントリパディング生成**: OsRngで0-255B範囲のランダムデータを生成し、各エントリ末尾に付与 🔵 PRDセクション9.1
3. **検証トークン生成**: 32Bランダムデータを生成→AES-GCM暗号化→(iv, ciphertext)をヘッダに格納 🔵 REQ-007
4. **検証トークン検証**: ヘッダからiv, ciphertextを取得→AES-GCM復号→成功ならtrue。S2のCryptoEngineと連携 🔵 REQ-008

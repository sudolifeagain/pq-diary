## 実装詳細

### 1. AccessPolicy enum (core/src/policy.rs)

**信頼性**: 🔵 *REQ-001, REQ-002*

`AccessPolicy` enum を以下のバリアントで定義する:

- `None` — vault の復号自体を拒否
- `WriteOnly` — 書き込み操作のみ許可
- `Full` — 全操作を許可

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Default, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum AccessPolicy {
    #[default]
    None,
    WriteOnly,
    Full,
}
```

`Display` と `FromStr` トレイトも実装する。

### 2. OperationType enum (core/src/policy.rs)

**信頼性**: 🔵 *REQ-010*

操作種別を表す enum を定義する:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum OperationType {
    Read,
    Write,
}
```

### 3. classify_operation() (core/src/policy.rs)

**信頼性**: 🔵 *REQ-010*

コマンド名から操作種別を判定する関数を実装する:

```rust
pub fn classify_operation(command: &str) -> OperationType
```

マッピング:
- **Read**: `"list"`, `"show"`, `"search"`, `"stats"`, `"template-show"`, `"template-list"`
- **Write**: `"new"`, `"edit"`, `"delete"`, `"sync"`, `"today"`, `"template-add"`, `"template-delete"`, `"import"`

### 4. PolicyDecision enum (core/src/policy.rs)

**信頼性**: 🔵 *REQ-020~025*

ポリシー評価結果を表す enum を定義する:

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum PolicyDecision {
    Allow,
    DenyNoDecrypt,
    DenyOperation,
}
```

### 5. check_access() (core/src/policy.rs)

**信頼性**: 🔵 *REQ-020~025*

4層のアクセス評価ロジックを実装する:

```rust
pub fn check_access(
    claude: bool,
    policy: AccessPolicy,
    operation: OperationType,
    vault_name: &str,
) -> PolicyDecision
```

評価順序:
1. **Layer 1**: `!claude` → `Allow` (非Claude環境では常に許可)
2. **Layer 2**: `None` → `DenyNoDecrypt` (vault復号を拒否)
3. **Layer 3**: `WriteOnly` + `Read` → `DenyOperation` (読み取り操作を拒否)
4. **Layer 3**: `WriteOnly` + `Write` → `Allow` (書き込み操作を許可)
5. **Layer 4**: `Full` → `Allow` (全操作を許可)

`vault_name` パラメータはエラーメッセージ生成のために使用する。

### 6. PolicyDecision::into_result() (core/src/policy.rs)

**信頼性**: 🔵 *REQ-050*

PolicyDecision を Result に変換するメソッドを実装する:

```rust
impl PolicyDecision {
    pub fn into_result(self) -> Result<(), DiaryError>
}
```

- `Allow` → `Ok(())`
- `DenyNoDecrypt` → `Err(DiaryError::Policy("..."))`
- `DenyOperation` → `Err(DiaryError::Policy("..."))`

既存の `DiaryError::Policy(String)` バリアントを使用する。新しいエラーバリアントは追加しない。

### テスト詳細

**TC-S7-001-01~03**: 各 AccessPolicy バリアントを `serde_json::to_string()` → `serde_json::from_str()` でシリアライズ/デシリアライズし、元の値と一致することを検証する。`rename_all = "snake_case"` により `"none"`, `"write_only"`, `"full"` 文字列に変換されることも確認する。

**TC-S7-001-04~06**: 各 AccessPolicy バリアントに対して `to_string()` → `FromStr::from_str()` のラウンドトリップを検証する。

**TC-S7-001-E01**: `"invalid_policy"` などの無効な文字列を `serde_json::from_str()` に渡してエラーが返ることを検証する。

**TC-S7-001-E02**: 空文字列 `""` を `serde_json::from_str()` に渡してエラーが返ることを検証する。

**TC-S7-010-01**: `"list"`, `"show"`, `"search"`, `"stats"`, `"template-show"`, `"template-list"` を `classify_operation()` に渡し、全て `OperationType::Read` が返ることを検証する。

**TC-S7-010-02**: `"new"`, `"edit"`, `"delete"`, `"sync"`, `"today"`, `"template-add"`, `"template-delete"`, `"import"` を `classify_operation()` に渡し、全て `OperationType::Write` が返ることを検証する。

**TC-S7-020-01~06**: check_access() の Allow ケースを検証する。非Claude環境ではポリシーに関係なく全操作が許可されること、Claude環境でも Full ポリシーでは全操作が許可されること、WriteOnly + Write で許可されることを確認する。

**TC-S7-020-E01~E06**: check_access() の Deny ケースを検証する。Claude環境で None ポリシーの場合は DenyNoDecrypt、WriteOnly + Read の場合は DenyOperation が返ることを確認する。異なる vault 名でも同様のDeny判定となることを検証する。

**TC-S7-050-01**: `DenyNoDecrypt` の `into_result()` が返す `DiaryError::Policy` のメッセージが REQ-050 で定義された形式に合致することを検証する。

**TC-S7-050-02**: `DenyOperation` の `into_result()` が返す `DiaryError::Policy` のメッセージが REQ-050 で定義された形式に合致することを検証する。

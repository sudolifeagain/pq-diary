## 実装詳細

### 1. AccessSection.policy の型変更 (core/src/vault/config.rs)

**信頼性**: 🔵 *REQ-003*

`AccessSection` 構造体の `policy` フィールドの型を `String` から `AccessPolicy` に変更する:

```rust
use crate::policy::AccessPolicy;

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AccessSection {
    pub policy: AccessPolicy,
}
```

`serde(rename_all = "snake_case")` が AccessPolicy enum に付与されているため、vault.toml 内の文字列 (`"none"`, `"write_only"`, `"full"`) は自動的に enum バリアントに変換される。

### 2. VaultConfig::default() の更新 (core/src/vault/config.rs)

**信頼性**: 🔵 *REQ-003*

`VaultConfig` の `Default` 実装を更新し、`policy` フィールドの初期値を `"none".to_owned()` から `AccessPolicy::None` に変更する:

```rust
// Before
access: AccessSection {
    policy: "none".to_owned(),
},

// After
access: AccessSection {
    policy: AccessPolicy::None,
},
```

VaultConfig の Default 実装は config.rs の82〜106行目付近に存在する。

### 3. 既存テストの更新 (core/src/vault/config.rs)

**信頼性**: 🔵 *REQ-003*

既存テスト TC-004-02 (`test_vault_config_default_values`) のアサーションを更新する:

```rust
// Before
assert_eq!(config.access.policy, "none");

// After
assert_eq!(config.access.policy, AccessPolicy::None);
```

既存テスト TC-004-01 (roundtrip) と TC-004-E01 (invalid TOML) は変更不要のはず。roundtrip テストでは TOML 文字列に `policy = "none"` が含まれるため、serde の自動変換で正常にパースされる。

### 4. 後方互換性の検証 (core/src/vault/config.rs)

**信頼性**: 🔵 *REQ-404*

S6以前の vault.toml ファイルで使用されている `policy = "none"` 文字列が、新しい AccessPolicy enum で正しくデシリアライズされることを検証する。`serde(rename_all = "snake_case")` により `"none"` → `AccessPolicy::None`、`"write_only"` → `AccessPolicy::WriteOnly`、`"full"` → `AccessPolicy::Full` に自動変換される。

### 5. DiaryCore::access_policy() アクセサ (core/src/lib.rs)

**信頼性**: 🔵 *design architecture.md*

DiaryCore に現在のアクセスポリシーを返すアクセサメソッドを追加する:

```rust
impl DiaryCore {
    pub fn access_policy(&self) -> AccessPolicy {
        self.config.access.policy
    }
}
```

DiaryCore の `config` フィールド（lib.rs の70行目付近）は既に `VaultConfig` 型であるため、そこから `access.policy` を返す。

### 6. DiaryCore::vault_name() アクセサ (core/src/lib.rs)

**信頼性**: 🔵 *design architecture.md*

DiaryCore に vault 名を返すアクセサメソッドを追加する:

```rust
impl DiaryCore {
    pub fn vault_name(&self) -> &str {
        &self.config.vault.name
    }
}
```

### テスト詳細

**TC-S7-001-07**: S6 以前の vault.toml 形式の TOML 文字列（`policy = "none"` を含む）を `toml::from_str::<VaultConfig>()` でパースし、`config.access.policy == AccessPolicy::None` であることを検証する。`"write_only"` と `"full"` についても同様にパースを確認する。

**TC-004-01 (既存)**: VaultConfig を TOML にシリアライズし、再度デシリアライズして元の値と一致することを検証する。型変更後も serde の自動変換により正常に動作するはず。

**TC-004-02 (既存、更新)**: `VaultConfig::default()` の各フィールドがデフォルト値であることを検証する。`access.policy` のアサーションを `"none"` から `AccessPolicy::None` に変更する。

**TC-004-E01 (既存)**: 不正な TOML 文字列のパースでエラーが返ることを検証する。変更不要。

**TC-S7-068-01**: テスト用の vault を初期化して DiaryCore を生成し、`access_policy()` がデフォルトの `AccessPolicy::None` を返すことを検証する。可能であれば異なるポリシー設定でも検証する。

**TC-S7-068-02**: テスト用の vault を初期化して DiaryCore を生成し、`vault_name()` が初期化時に指定した vault 名を返すことを検証する。

## 実装詳細

### 1. VaultManager::validate_vault_name(name: &str) -> Result<(), DiaryError>

**信頼性**: 🔵 *EDGE-101, EDGE-102*

静的メソッドとして実装する。以下の条件でバリデーションを行い、違反時は `DiaryError::InvalidArgument` を返す。

- 空文字列を拒否
- パストラバーサルを拒否: `'/'`, `'\'`, `".."` を含む場合
- Windows上でファイルシステム無効文字を拒否: `':'`, `'*'`, `'?'`, `'"'`, `'<'`, `'>'`, `'|'`

```rust
pub fn validate_vault_name(name: &str) -> Result<(), DiaryError> {
    if name.is_empty() {
        return Err(DiaryError::InvalidArgument("vault name must not be empty".into()));
    }
    if name.contains('/') || name.contains('\\') || name.contains("..") {
        return Err(DiaryError::InvalidArgument("vault name contains invalid path characters".into()));
    }
    if name.contains(':') || name.contains('*') || name.contains('?')
        || name.contains('"') || name.contains('<') || name.contains('>')
        || name.contains('|')
    {
        return Err(DiaryError::InvalidArgument("vault name contains filesystem-invalid characters".into()));
    }
    Ok(())
}
```

### 2. VaultManager::create_vault(&self, name: &str, password: &[u8], policy: AccessPolicy) -> Result<(), DiaryError>

**信頼性**: 🔵 *REQ-030〜032, EDGE-001, EDGE-103*

vault作成の高レベルラッパー。以下の手順で処理する:

1. `validate_vault_name(name)?` を呼び出し名前のバリデーション
2. vault パスが既に存在するか確認 → 存在する場合は `DiaryError::Vault("already exists")` を返す (EDGE-001)
3. `vaults/` ディレクトリが存在しない場合は `std::fs::create_dir_all()` で自動作成 (EDGE-103)
4. `self.init_vault(name, password)?` を呼び出しvaultを初期化
5. `self.set_policy(name, policy)?` を呼び出しポリシーを設定

**備考**: `set_policy()` は TASK-0070 で実装予定。`create_vault()` の初期実装では vault.toml を直接変更するか、最小限の `set_policy()` スタブを作成して対応する。

### 3. 既存コードとの関係

**信頼性**: 🔵 *既存実装の確認結果*

- 既存の `init_vault()` (行104-191) が vault.pqd の作成を担当する。`create_vault()` はそのラッパーとして機能する
- `VaultManager` は既に `base_dir` フィールドを持ちvaultパス解決に使用している
- `set_policy()` が未実装の場合、`create_vault()` 内で直接 vault.toml の `[access].policy` を書き換える実装を仮置きする

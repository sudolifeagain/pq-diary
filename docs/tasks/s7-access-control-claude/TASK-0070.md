### 2. VaultManager::list_vaults_with_policy(&self) -> Result<Vec<VaultInfo>, DiaryError>

**信頼性**: 🔵 *REQ-033, REQ-201, NFR-201, NFR-002*

パスワード不要で全vaultの名前とポリシーを取得する。

1. `~/.pq-diary/vaults/` ディレクトリのエントリを読み取る
2. 各サブディレクトリについて vault.toml の存在を確認
3. vault.toml が存在する場合: 読み取り → `VaultConfig` にデシリアライズ → 名前とポリシーを抽出
4. vault.toml が破損している場合: `DiaryError::Config` を返す (EDGE-004)
5. 結果を名前でソートして `Vec<VaultInfo>` として返す
6. パスワード不要: vault.toml は平文 (REQ-201)

**備考**: 既存の `list_vaults()` (行201-216) は `Vec<String>` を返す。`list_vaults_with_policy()` はその上位互換として機能する。

### 3. VaultManager::set_policy(&self, name: &str, policy: AccessPolicy) -> Result<(), DiaryError>

**信頼性**: 🔵 *REQ-034, REQ-202, REQ-403*

vault.toml のポリシーフィールドをアトミックに更新する。パスワード不要。

1. vaultの存在を確認: vault.toml が存在するか検証 → 不在の場合 `DiaryError::Vault` (EDGE-002)
2. vault.toml を読み取り → `VaultConfig` にデシリアライズ
3. `config.access.policy = policy` でポリシーを更新
4. アトミック書き込み: vault.toml.tmp にシリアライズして書き込み → `sync_all()` → `rename()` で vault.toml に置換 (REQ-403)

```rust
pub fn set_policy(&self, name: &str, policy: AccessPolicy) -> Result<(), DiaryError> {
    let vault_path = self.base_dir.join("vaults").join(name);
    let toml_path = vault_path.join("vault.toml");
    if !toml_path.exists() {
        return Err(DiaryError::Vault(format!("vault '{}' does not exist", name)));
    }
    let content = std::fs::read_to_string(&toml_path)?;
    let mut config: VaultConfig = toml::from_str(&content)
        .map_err(|e| DiaryError::Config(format!("failed to parse vault.toml: {}", e)))?;
    config.access.policy = policy;
    let new_content = toml::to_string_pretty(&config)
        .map_err(|e| DiaryError::Config(format!("failed to serialize vault.toml: {}", e)))?;
    let tmp_path = toml_path.with_extension("toml.tmp");
    std::fs::write(&tmp_path, &new_content)?;
    let file = std::fs::File::open(&tmp_path)?;
    file.sync_all()?;
    drop(file);
    std::fs::rename(&tmp_path, &toml_path)?;
    Ok(())
}
```

### 4. VaultManager::delete_vault(&self, name: &str, zeroize: bool) -> Result<(), DiaryError>

**信頼性**: 🔵 *REQ-035〜037, EDGE-003*

vaultディレクトリを削除する。オプションでvault.pqdをゼロ化する。パスワード不要。

1. vaultの存在を確認: vaultディレクトリが存在するか検証 → 不在の場合 `DiaryError::Vault` (EDGE-003)
2. `zeroize=true` の場合:
   - vault.pqd ファイルのサイズを取得
   - 同サイズの `OsRng` ランダムデータで上書き
   - `sync_all()` でディスクにフラッシュ
3. `std::fs::remove_dir_all(vault_path)` でディレクトリ全体を削除

```rust
pub fn delete_vault(&self, name: &str, zeroize: bool) -> Result<(), DiaryError> {
    let vault_path = self.base_dir.join("vaults").join(name);
    if !vault_path.exists() {
        return Err(DiaryError::Vault(format!("vault '{}' does not exist", name)));
    }
    if zeroize {
        let pqd_path = vault_path.join("vault.pqd");
        if pqd_path.exists() {
            let metadata = std::fs::metadata(&pqd_path)?;
            let size = metadata.len() as usize;
            let mut random_data = vec![0u8; size];
            use rand::RngCore;
            rand::rngs::OsRng.fill_bytes(&mut random_data);
            std::fs::write(&pqd_path, &random_data)?;
            let file = std::fs::File::open(&pqd_path)?;
            file.sync_all()?;
            drop(file);
        }
    }
    std::fs::remove_dir_all(&vault_path)?;
    Ok(())
}
```

### テスト詳細

**TC-S7-033-01**: 3つのvaultを異なるポリシーで作成し、`list_vaults_with_policy()` を呼び出す。結果が3件であり、名前でソートされていること、各vaultのポリシーが正しいことを検証する。

**TC-S7-033-02**: vaultを1件も作成せずに `list_vaults_with_policy()` を呼び出し、空の `Vec` が返ることを検証する。

**TC-S7-033-03**: `list_vaults_with_policy()` がパスワード引数を取らないことをAPIレベルで確認する。vault.toml のみを読み取り、vault.pqd の復号を行わないことを検証する。

**TC-S7-034-01**: ポリシー `None` で作成したvaultに対して `set_policy(name, AccessPolicy::WriteOnly)` を呼び出し、vault.toml の `[access].policy` が `"write_only"` に変更されていることを検証する。

**TC-S7-034-03**: ポリシー `Full` で作成したvaultに対して `set_policy(name, AccessPolicy::None)` を呼び出し、正常に変更されることを検証する。core層では警告を出さない（警告はCLI層の責務）。

**TC-S7-034-04**: `set_policy()` がパスワード引数を取らないことをAPIレベルで確認する。vault.toml のみを更新し、vault.pqd に触れないことを検証する。

**TC-S7-034-E01**: 存在しないvault名を `set_policy()` に渡し、`DiaryError::Vault` が返ることを検証する。

**TC-S7-035-01**: vaultを作成後、`delete_vault(name, false)` を呼び出し、vaultディレクトリが完全に削除されていることをファイルシステム上で検証する。

**TC-S7-035-04**: vaultを作成後、`delete_vault(name, true)` を呼び出す。削除前にvault.pqd のバイト内容を記録し、zeroize後のファイル内容が元と異なること（ランダムデータで上書きされていること）を検証する。最終的にディレクトリが削除されていることも確認する。

**TC-S7-035-E02**: 存在しないvault名を `delete_vault()` に渡し、`DiaryError::Vault` が返ることを検証する。

**TC-S7-EDGE-004**: vault.toml に無効な内容（例: `"invalid toml content {{{"`）を書き込んだ状態で `list_vaults_with_policy()` または `set_policy()` を呼び出し、`DiaryError::Config` が返ることを検証する。

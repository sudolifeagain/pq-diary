## 実装詳細

### 1. VaultCommands enum 更新 (cli/src/main.rs)

**信頼性**: 🔵 *REQ-030~037*

cli/src/main.rs の VaultCommands enum を以下のように更新する:

```rust
Create { name: String, #[arg(long)] policy: Option<String> }
List   // (変更なし)
Policy { name: String, policy: String }
Delete { name: String, #[arg(long)] zeroize: bool }
```

### 2. ディスパッチ match 更新 (cli/src/main.rs)

**信頼性**: 🔵 *REQ-030~037*

main.rs のディスパッチ match を更新し、新しい cmd_vault_* 関数を呼び出す。既存の `not_implemented()` 呼び出しを削除する。

### 3. cmd_vault_create(cli, name, policy_str) (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-030, REQ-031, REQ-040, NFR-202*

1. ポリシー文字列を AccessPolicy にパースする（デフォルト: None） - REQ-031
2. Full の場合: 警告を表示し [y/N] 確認プロンプトを表示 - REQ-040
3. get_password() で vault.pqd 初期化用パスワードを取得 - REQ-030
4. VaultManager::create_vault(name, pw, policy) を呼び出す - REQ-030
5. 成功メッセージを出力: "Created vault '{name}' at {path} (policy: {policy})" - NFR-202

### 4. cmd_vault_list(cli) (cli/src/commands.rs)

**信頼性**: 🔵 *NFR-201*

1. パスワード入力不要
2. VaultManager::list_vaults_with_policy() を呼び出す
3. NAME + POLICY カラムのテーブルを出力 - NFR-201

### 5. cmd_vault_policy(cli, name, policy_str) (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-034, REQ-040*

1. ポリシー文字列を AccessPolicy にパースする
2. Full の場合: 警告を表示し [y/N] 確認プロンプトを表示 - REQ-040
3. VaultManager::set_policy(name, policy) を呼び出す - REQ-034
4. メッセージを出力: "Policy updated: {name} → {policy}"

### 6. cmd_vault_delete(cli, name, zeroize) (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-035, REQ-036, REQ-037, REQ-102*

1. vault が存在するか確認する
2. デフォルト vault の場合は追加警告を表示 - REQ-102
3. 確認プロンプトを表示（--claude の場合はスキップ） - REQ-036
4. VaultManager::delete_vault(name, zeroize) を呼び出す - REQ-035, REQ-037
5. メッセージを出力: "Deleted vault '{name}'"

### 7. Full ポリシー警告メッセージ (PRD 4.4)

**信頼性**: 🔵 *REQ-040, REQ-041*

Full ポリシー設定時に以下の警告メッセージを表示する:

```
警告: "full"を設定すると日記内容がAnthropicのAPIサーバーを通過します。
      間接プロンプトインジェクション等のリスクがあります。
      本当に許可しますか？ [y/N]:
```

### テスト詳細

**TC-S7-030-03**: Full ポリシーを指定して vault を作成する際、警告メッセージが表示され、"y" 入力で作成が完了することを検証する。

**TC-S7-030-E04**: "invalid" などの無効なポリシー文字列を指定した場合にエラーが返ることを検証する。

**TC-S7-030-E05**: Full ポリシー作成時の警告に "n" を入力した場合、vault が作成されずに中止されることを検証する。

**TC-S7-034-02**: write_only から full にポリシーを変更する際、警告メッセージが表示され、"y" 入力で変更が完了することを検証する。

**TC-S7-034-E02**: ポリシー変更時の Full 警告に "n" を入力した場合、変更されずに中止されることを検証する。

**TC-S7-035-02**: vault 削除の確認プロンプトに "n" を入力した場合、削除されずに中止されることを検証する。

**TC-S7-035-03**: --claude フラグ付きで vault delete を実行した場合、確認プロンプトがスキップされて直接削除されることを検証する。

**TC-S7-035-E01**: デフォルト vault を削除しようとした場合、通常の確認に加えて追加の警告メッセージが表示されることを検証する。

**TC-S7-040-01~04**: None ポリシーおよび WriteOnly ポリシーの作成・変更時には警告が表示されず、Full ポリシーの場合のみ警告が表示されることを検証する。

**TC-S7-NFR-003-01**: vault create コマンドが 3秒以内に完了することを検証する。

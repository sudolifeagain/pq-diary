## 実装詳細

### 1. check_claude_policy() ヘルパー関数 (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-020~025, REQ-101, NFR-101*

commands.rs の先頭にポリシーチェック用ヘルパー関数を作成する:

```rust
fn check_claude_policy(cli: &Cli, operation: OperationType) -> Result<()> {
    if !cli.claude { return Ok(()); }
    let vault_path = resolve_vault_path(cli)?;
    let toml_path = PathBuf::from(&vault_path).join("vault.toml");
    let config = VaultConfig::from_file(&toml_path)?;
    let decision = policy::check_access(
        true, config.access.policy, operation, &config.vault.name,
    );
    decision.into_result().map_err(|e| anyhow::anyhow!("{}", e))
}
```

### 2. 各 cmd_* 関数へのポリシーチェック挿入 (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-020~025, EDGE-006*

各 cmd_* 関数の先頭（get_password() 呼び出しの前）にポリシーチェックを挿入する:

| 関数 | 操作種別 |
|------|---------|
| cmd_new | Write |
| cmd_list | Read |
| cmd_show | Read |
| cmd_edit | Write |
| cmd_delete | Write |
| cmd_search | Read |
| cmd_stats | Read |
| cmd_today | Write |
| cmd_template_add | Write |
| cmd_template_list | Read |
| cmd_template_show | Read |
| cmd_template_delete | Write |
| cmd_import | Write |

### 3. None ポリシーの早期拒否 (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-101, NFR-101, EDGE-006*

None ポリシーの場合、チェックは get_password() の前に実行されるため、パスワードプロンプトは表示されず、vault.pqd の復号も行われない。これにより:

- REQ-101: Claude環境でのアクセス制御が機能する
- NFR-101: 不要な復号処理を回避する
- EDGE-006: パスワードが提供されていても None ポリシーでは拒否する

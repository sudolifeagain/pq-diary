## 実装詳細

### 1. GitOperations 構造体 (core/src/git.rs)

**信頼性**: 🔵 *REQ-001*

Git 操作の実行コンテキストを保持する構造体を定義する:

```rust
pub struct GitOperations {
    vault_dir: PathBuf,
}
```

`vault_dir` は対象 vault のディレクトリパスを保持する。全 Git 操作はこのディレクトリをワーキングディレクトリとして実行する。

### 2. check_git_available() (core/src/git.rs)

**信頼性**: 🔵 *REQ-050, EDGE-001*

システムに git がインストールされているかを確認する関数を実装する:

```rust
pub fn check_git_available() -> Result<(), DiaryError>
```

- `git --version` を `std::process::Command` で実行する
- 成功時: `Ok(())` を返す
- 失敗時 (コマンド未発見 / 実行エラー): `DiaryError::Git("git is not installed or not found in PATH".into())` を返す — EDGE-001

### 3. generate_random_author_email() (core/src/git.rs)

**信頼性**: 🔵 *REQ-003*

プライバシー保護のためのランダムなメールアドレスを生成する関数を実装する:

```rust
pub fn generate_random_author_email() -> String
```

- `OsRng` から 4 バイトのランダム値を取得する
- 16進数文字列 (8文字) に変換する
- `"{8hex}@localhost"` 形式で返す
- 例: `"a3f1b2c4@localhost"`

### 4. generate_gitignore() (core/src/git.rs)

**信頼性**: 🔵 *REQ-002, REQ-051*

暗号化前の平文エントリがリポジトリに含まれないよう .gitignore 内容を生成する関数を実装する:

```rust
pub fn generate_gitignore() -> String
```

- `"entries/*.md\n"` を返す
- 暗号化済みファイルのみが Git 管理対象となることを保証する

### 5. git_init() (core/src/git.rs)

**信頼性**: 🔵 *REQ-001, REQ-003, REQ-004, REQ-005, EDGE-003, EDGE-006*

vault ディレクトリを Git リポジトリとして初期化する関数を実装する:

```rust
pub fn git_init(vault_dir: &Path, remote: Option<&str>) -> Result<(), DiaryError>
```

処理手順:
1. `.git` ディレクトリの存在チェック → 既に存在する場合は `DiaryError::Git("already initialized")` を返す — EDGE-006
2. `git init` を vault_dir で実行する — REQ-001
3. `generate_gitignore()` の内容で `.gitignore` ファイルを作成する — REQ-002
4. `generate_random_author_email()` でランダムメールアドレスを生成する — REQ-003
5. `git config user.name "pq-diary"` を実行する — REQ-003
6. `git config user.email "{generated_email}"` を実行する — REQ-003
7. vault.toml の `[git]` セクションを原子的に更新する (既存の `set_policy()` パターンを再利用) — REQ-004
   - `author_name = "pq-diary"`
   - `author_email = "{generated_email}"`
8. `remote` が `Some(url)` の場合: `git remote add origin <url>` を実行する — REQ-005
9. いずれかの Git コマンド失敗時は `DiaryError::Git` を返す — EDGE-003

### 6. run_git_command() ヘルパー (core/src/git.rs)

**信頼性**: 🔵 *REQ-001*

Git コマンド実行の共通ヘルパー関数を実装する:

```rust
fn run_git_command(vault_dir: &Path, args: &[&str]) -> Result<std::process::Output, DiaryError>
```

- `std::process::Command::new("git")` で Git プロセスを生成する
- `.current_dir(vault_dir)` でワーキングディレクトリを設定する
- `.args(args)` で引数を渡す
- 実行結果を `Output` として返す
- コマンド実行失敗時は `DiaryError::Git` でエラーメッセージを含めて返す

## 実装詳細

### 1. make_author() (core/src/git.rs)

**信頼性**: 🔵 *REQ-011, REQ-052*

vault 設定から Git コミット用の匿名化された author 文字列を生成する関数を実装する:

```rust
pub fn make_author(config: &VaultConfig) -> String
```

- vault.toml の `[git]` セクションから `author_email` を取得する
- `"pq-diary <{author_email}>"` 形式の文字列を返す
- 例: `"pq-diary <a3f1b2c4@localhost>"`
- 実際のユーザー名やメールアドレスを一切含めない — REQ-011

### 2. fuzz_timestamp() (core/src/git.rs)

**信頼性**: 🔵 *REQ-014, REQ-015*

コミット時刻をファジングして行動パターンを秘匿する関数を実装する:

```rust
pub fn fuzz_timestamp(prev: Option<DateTime<Utc>>, fuzz_hours: u64) -> DateTime<Utc>
```

処理ロジック:
1. `fuzz_hours == 0` の場合: `Utc::now()` をそのまま返す (ファジング無効) — REQ-014
2. `OsRng` から `0..=(fuzz_hours * 3600)` 秒の範囲でランダムオフセットを生成する
3. `candidate = prev + Duration::seconds(offset + 1)` を算出する
   - `prev` が `None` の場合は `Utc::now() - Duration::hours(fuzz_hours)` を基準とする
4. `min(candidate, Utc::now())` を返す — 未来のタイムスタンプを防止
5. 単調増加保証: `prev < result` が常に成立する — REQ-015

### 3. get_last_commit_timestamp() (core/src/git.rs)

**信頼性**: 🔵 *REQ-014*

リポジトリの最新コミットの author date を取得する関数を実装する:

```rust
pub fn get_last_commit_timestamp(vault_dir: &Path) -> Option<DateTime<Utc>>
```

- `git log -1 --format=%aI` を実行して最新コミットの ISO 8601 タイムスタンプを取得する
- `chrono::DateTime::parse_from_rfc3339()` でパースし `DateTime<Utc>` に変換する
- コミットが存在しない場合 (空リポジトリ) は `None` を返す
- パースエラー時も `None` を返す (堅牢性)

### 4. generate_extra_padding() (core/src/git.rs)

**信頼性**: 🔵 *REQ-054*

コミットサイズのばらつきを生成してメタデータ分析を困難にするパディングを生成する関数を実装する:

```rust
pub fn generate_extra_padding(max_bytes: usize) -> Vec<u8>
```

- `max_bytes == 0` の場合: 空の `Vec` を返す — REQ-054
- `OsRng` から `0..max_bytes` の範囲でランダムサイズを決定する
- 決定したサイズ分のランダムバイト列を `OsRng` で生成する
- `Vec<u8>` として返す

### 5. chrono 依存の確認・追加 (core/Cargo.toml)

**信頼性**: 🔵 *実装前提*

`core/Cargo.toml` に `chrono` クレートが依存として含まれているか確認する。含まれていない場合は追加する:

```toml
[dependencies]
chrono = { version = "0.4", default-features = false, features = ["clock", "serde"] }
```

`DateTime<Utc>` と ISO 8601 パースに必要。`serde` feature はシリアライズ/デシリアライズ対応のために含める。

### テスト詳細

**TC-S8-075-01**: テスト用の `VaultConfig` を構築し (`[git]` セクションに `author_email = "abcd1234@localhost"` を設定)、`make_author()` を呼び出す。戻り値が `"pq-diary <abcd1234@localhost>"` と一致することを検証する。実際のユーザー名が含まれないことも確認する。

**TC-S8-075-02**: `fuzz_timestamp(None, 0)` を呼び出し、戻り値と `Utc::now()` の差が 1秒以内であることを検証する。ファジングが無効化されていることを確認する。

**TC-S8-075-03**: `prev = Utc::now() - Duration::hours(12)` を設定し、`fuzz_timestamp(Some(prev), 6)` を呼び出す。戻り値が `prev < result` かつ `result <= Utc::now()` を満たすことを検証する。複数回実行して異なるタイムスタンプが返ることも確認する (ランダム性の検証)。

**TC-S8-075-04**: `prev = Utc::now() - Duration::hours(1)` を設定し、`fuzz_timestamp(Some(prev), 6)` を10回連続で呼び出す。毎回 `prev < result` が成立することを検証する。また、同じ `prev` に対して `offset + 1` により厳密に `prev < result` (等号なし) となることも確認する。

**TC-S8-075-05**: `generate_extra_padding(0)` を呼び出し、戻り値が空の `Vec` (`len() == 0`) であることを検証する。

**TC-S8-075-06**: `generate_extra_padding(4096)` を100回呼び出し、毎回 `result.len() <= 4096` であることを検証する。少なくとも1回は `result.len() > 0` となること (ランダムサイズなので確率的に保証) も確認する。サイズの分布が偏りすぎていないことを統計的に検証する (例: 全て0バイトではないこと)。

**TC-S8-075-07**: 一時ディレクトリで `git init` を実行した直後 (コミットなし) の状態で `get_last_commit_timestamp()` を呼び出し、`None` が返ることを検証する。

## 注意事項

- 既存の `DiaryError::Git(String)` バリアントを使用すること。新しいエラーバリアントを追加しないこと
- `OsRng` を使用して暗号品質のランダム値を生成すること (`rand::rngs::OsRng`)。`thread_rng()` は使用しないこと
- `chrono` の `default-features = false` を指定して不要な機能を含めないこと
- タイムスタンプファジングは未来の時刻を生成しないこと (`min(candidate, Utc::now())` による保護)
- TASK-0074 の `run_git_command()` ヘルパーを `get_last_commit_timestamp()` で再利用すること

## 実装詳細

### 1. git_push() 関数シグネチャ (core/src/git.rs)

**信頼性**: 🔵 *REQ-010*

`git_push()` 関数を以下のシグネチャで定義する:

```rust
pub fn git_push(
    vault_dir: &Path,
    config: &GitSyncConfig,
    engine: &CryptoEngine,
    vault_path: &Path,
) -> Result<(), DiaryError>
```

処理フロー:
1. `.git` ディレクトリの存在を確認 → 不在なら EDGE-003 エラー
2. リモートの設定を確認 → 未設定なら EDGE-002 エラー
3. vault.pqd を新しいパディングで再書き込み（Step 1~2）
4. プライバシーパイプライン適用（Step 3~7）
5. git push 実行（Step 8）

### 2. vault.pqd パディング再書き込み (core/src/git.rs)

**信頼性**: 🔵 *REQ-013, REQ-016*

vault.pqd を新しいランダムパディングで再書き込みすることで、コミット間のバイナリ差分からエントリ数やサイズを推測されることを防ぐ:

1. `read_vault()` で既存の vault.pqd から `(VaultHeader, Vec<EntryRecord>)` を読み取る
2. `write_vault()` で同じヘッダとエントリを新しいランダムパディングで再書き込みする
3. `extra_padding_bytes_max > 0` の場合、追加パディングを生成して `write_vault()` に渡す
4. `extra_padding_bytes_max == 0` の場合、追加パディングは付与しない

### 3. git add — ステージング対象ファイル (core/src/git.rs)

**信頼性**: 🔵 *REQ-010*

以下の3ファイルのみをステージングする:

```
git add vault.pqd vault.toml .gitignore
```

`entries/` ディレクトリ等のその他のファイルはステージングしない。暗号化された vault.pqd のみがリポジトリに含まれることを保証する。

### 4. 匿名 author の構築 (core/src/git.rs)

**信頼性**: 🔵 *REQ-011*

vault.toml の `[git_sync]` セクションから `anonymous_name` と `anonymous_email` を読み取り、コミット時の author として使用する:

```rust
fn make_author(config: &GitSyncConfig) -> (String, String) {
    (config.anonymous_name.clone(), config.anonymous_email.clone())
}
```

### 5. 固定コミットメッセージ (core/src/git.rs)

**信頼性**: 🔵 *REQ-012*

vault.toml の `[git_sync]` セクションから `commit_message` を読み取り、全コミットで同一のメッセージを使用する。コミットメッセージの内容からエントリの変更内容を推測されることを防ぐ。

### 6. タイムスタンプファジング (core/src/git.rs)

**信頼性**: 🔵 *REQ-014*

コミットの正確な作成時刻を隠すためにタイムスタンプファジングを適用する:

1. `get_last_commit_timestamp()` で直前のコミットのタイムスタンプを取得する
2. `fuzz_timestamp()` で `fuzz_hours` パラメータに基づいてタイムスタンプにランダムなオフセットを加算する
3. ファジングされたタイムスタンプを ISO 8601 形式でフォーマットする
4. `GIT_AUTHOR_DATE` および `GIT_COMMITTER_DATE` 環境変数にセットする

### 7. git commit の実行 (core/src/git.rs)

**信頼性**: 🔵 *REQ-017*

全プライバシーパラメータを適用して git commit を実行する:

```bash
git commit \
    -c user.name=<anonymous_name> \
    -c user.email=<anonymous_email> \
    --date=<fuzzed_timestamp> \
    -m "<commit_message>"
```

`GIT_AUTHOR_DATE` と `GIT_COMMITTER_DATE` 環境変数により、author date と committer date の両方がファジングされる。

### 8. git push の実行 (core/src/git.rs)

**信頼性**: 🔵 *REQ-010*

現在のブランチをリモートにプッシュする:

```bash
git push origin main
```

カレントブランチが `main` でない場合は、実際のブランチ名を使用する。

### 9. 再書き込みパターンの詳細 (core/src/git.rs)

**信頼性**: 🔵 *REQ-013, REQ-016*

再書き込みの処理フロー:

```
read_vault(vault_path)
    → (VaultHeader, Vec<EntryRecord>)
    → write_vault(vault_path, header, entries, new_random_padding)
```

`write_vault()` は `core/src/vault/writer.rs` に既存の関数。`read_vault()` は `core/src/vault/reader.rs` に既存。再書き込み時はエントリの復号は不要。暗号化された `EntryRecord` をそのまま渡して新しいパディングのみを変更する。

### 10. タイムスタンプファジングの詳細 (core/src/git.rs)

**信頼性**: 🔵 *REQ-014*

```rust
fn get_last_commit_timestamp(vault_dir: &Path) -> Result<Option<i64>, DiaryError>
fn fuzz_timestamp(base: i64, fuzz_hours: u32) -> i64
```

- `get_last_commit_timestamp()`: `git log -1 --format=%ct` で直前コミットの UNIX タイムスタンプを取得。コミットがない場合は `None` を返す
- `fuzz_timestamp()`: `fuzz_hours` が 0 の場合はファジングなし。0 より大きい場合は `[-fuzz_hours, +fuzz_hours]` の範囲でランダムオフセットを加算
- フォーマット: ISO 8601 形式 (`2026-01-15T14:30:00+00:00`)

### テスト詳細

**TC-S8-076-01**: テスト用の vault.pqd を作成し、`git_push()` を呼び出した後にファイルのバイト内容が変更されていることを検証する。パディングが再生成されるため、同一エントリでもバイナリレベルで異なる内容になる。

**TC-S8-076-02**: `git_push()` 実行後に `git log -1 --format=%an,%ae` でコミットの author name と email を取得し、vault.toml の `anonymous_name` / `anonymous_email` と一致することを検証する。

**TC-S8-076-03**: `git_push()` 実行後に `git log -1 --format=%s` でコミットメッセージを取得し、vault.toml の `commit_message` と一致することを検証する。

**TC-S8-076-04**: `fuzz_hours > 0` の設定で `git_push()` を実行し、`git log -1 --format=%ai` でコミットの author date を取得する。壁時計の現在時刻との差が `fuzz_hours` の範囲内であることを検証する。`fuzz_hours = 0` の場合はファジングが適用されないことも確認する。

**TC-S8-076-05**: `extra_padding_bytes_max = 0` の設定で `git_push()` を実行し、vault.pqd のファイルサイズが追加パディングによって増加していないことを検証する。標準パディングのみが再生成されることを確認する。

**TC-S8-076-06**: `.git` ディレクトリが存在しない作業ディレクトリで `git_push()` を呼び出し、`DiaryError` の EDGE-003 に対応するエラーが返ることを検証する。

**TC-S8-076-07**: `.git` は存在するがリモートが設定されていないリポジトリで `git_push()` を呼び出し、`DiaryError` の EDGE-002 に対応するエラーが返ることを検証する。

**TC-S8-076-08**: テスト用の vault ディレクトリに `entries/` ディレクトリとダミーファイルを配置し、`git_push()` 実行後に `git diff --cached --name-only` で vault.pqd, vault.toml, .gitignore のみがステージングされていることを検証する。`entries/` 配下のファイルがステージングされていないことを確認する。

## 注意事項

- `write_vault()` は `core/src/vault/writer.rs` に既存の関数。引数の詳細は実装時に確認すること
- `read_vault()` は `(VaultHeader, Vec<EntryRecord>)` を返す。再書き込み時は `CryptoEngine` による復号は不要（パディングのみ変更）
- git コマンドの実行には `std::process::Command` を使用する。環境変数の設定には `.env()` メソッドを使用する
- テストでは実際の git リポジトリを一時ディレクトリに作成してテストする（`tempfile::tempdir()`）

## 実装詳細

### 1. Commands enum 更新 (cli/src/main.rs)

**信頼性**: 🔵 *REQ-001~005*

Commands enum に Git 系サブコマンドの引数定義を追加・更新する:

```rust
GitInit {
    #[arg(long)]
    remote: Option<String>,
},
GitPush,
GitPull,
GitSync,
GitStatus,
```

- 全 Git 系コマンドの not_implemented() 呼び出しを除去する
- 各サブコマンドから対応する cmd_git_* 関数を呼び出すように更新する

### 2. cmd_git_init(cli, remote: Option<&str>) → Result<()>

**信頼性**: 🔵 *REQ-001, REQ-016~017*

パスワード不要のコマンドとして実装する:

1. vault ディレクトリを resolve_vault_path() で解決する
2. check_git_available()? で git コマンドの存在を確認する
3. git::git_init(vault_dir, remote)? で .git 初期化と vault.toml の author フィールド更新を行う
4. 成功メッセージを表示する（生成された匿名 author 情報を含む）

**注意**: ポリシーチェック不要（vault.pqd にアクセスしない）

### 3. cmd_git_push(cli) → Result<()>

**信頼性**: 🔵 *REQ-002, REQ-022, REQ-025~026*

パスワード必要のコマンドとして実装する（vault.pqd を新パディングで再書込みするため）:

1. check_claude_policy(cli, OperationType::Write)? でポリシーチェックを行う
2. get_password() でパスワードを取得する
3. vault をアンロックする
4. git::git_push(vault_dir, config, engine, vault_path)? で匿名 author・固定メッセージ・タイムスタンプファジングを適用してプッシュする
5. VaultGuard による RAII パターンでロックする
6. 成功メッセージを表示する

### 4. cmd_git_pull(cli) → Result<()>

**信頼性**: 🔵 *REQ-003, REQ-030~031, REQ-040*

パスワード必要のコマンドとして実装する（vault.pqd の復号がマージ時に必要なため）:

1. check_claude_policy(cli, OperationType::Write)? でポリシーチェックを行う
2. get_password() でパスワードを取得する
3. vault をアンロックする
4. result = git::git_pull_merge(vault_dir, config, engine, vault_path, cli.claude)? でプル＆マージを実行する
5. result.conflicts が非空かつ !cli.claude の場合:
   - 競合リストを表示する
   - 各競合に対してインタラクティブプロンプトを表示する: "Keep local (l) or remote (r)?"
   - ユーザー選択に基づいて解決を適用する
6. cli.claude の場合: ローカル側を自動的に優先する（REQ-040）
7. VaultGuard による RAII パターンでロックする
8. マージサマリーを表示する（追加/更新/削除/競合の件数）

### 5. cmd_git_sync(cli) → Result<()>

**信頼性**: 🔵 *REQ-005, REQ-030*

単一パスワード取得で pull → push を連続実行する:

1. check_claude_policy(cli, OperationType::Write)? でポリシーチェックを行う
2. get_password() でパスワードを取得する（1回のみ）
3. vault をアンロックする
4. git_pull_merge() を実行する → 競合があればハンドリング（cmd_git_pull と同様のロジック）
5. git_push() を実行する
6. VaultGuard による RAII パターンでロックする
7. 成功メッセージを表示する

**注意**: cmd_git_pull() → cmd_git_push() の単純な連鎖ではなく、パスワード取得が1回で済むようにインラインで実装する

### 6. cmd_git_status(cli) → Result<()>

**信頼性**: 🔵 *REQ-004*

パスワード不要のコマンドとして実装する:

1. vault ディレクトリを resolve_vault_path() で解決する
2. check_git_available()? で git コマンドの存在を確認する
3. git::git_status(vault_dir)? で git status 出力を取得する
4. 出力を表示する

**注意**: ポリシーチェック不要（vault.pqd にアクセスしない）

## 注意事項

- 既存の VaultGuard RAII パターン（push/pull/sync でのロック/アンロック）に従うこと
- ポリシーチェックは check_claude_policy() を使用し、適切な OperationType を指定すること
- git-push と git-pull は Write 操作としてポリシーチェックする（vault.pqd を変更するため）
- git-init と git-status はポリシーチェック不要（vault.pqd にアクセスしないため）
- cmd_git_sync はパスワード取得を1回に抑えるため、cmd_git_pull/cmd_git_push を呼び出すのではなくインラインで実装すること

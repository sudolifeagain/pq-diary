## 実装詳細

### 1. M-1: not_implemented() → anyhow::bail! (cli/src/main.rs)

**信頼性**: 🔵 *REQ-021, backlog M-1*

未実装コマンドのプレースホルダー関数 `not_implemented()` を `anyhow::bail!` マクロに置換する:

現在の実装:
```rust
fn not_implemented(cmd: &str, sprint: &str) {
    eprintln!("Command '{}' is not yet implemented (planned for {})", cmd, sprint);
    std::process::exit(1);
}
```

変更後:
```rust
fn not_implemented(cmd: &str, sprint: &str) -> anyhow::Result<()> {
    anyhow::bail!("Command '{}' is not yet implemented (planned for {})", cmd, sprint)
}
```

**影響範囲**: 以下の全コマンドハンドラが対象:
- `init`, `sync`, `export`, `change-password`, `info`
- `legacy init`, `legacy rotate`, `legacy set`, `legacy list`
- `legacy-access`
- `daemon start`, `daemon stop`, `daemon status`, `daemon lock`

各呼び出し箇所で戻り値型は既に `anyhow::Result<()>` であるため、`not_implemented(...)` → `not_implemented(...)?` への変更のみで対応可能。

### 2. M-3: PQC dependency pin (core/Cargo.toml)

**信頼性**: 🔵 *REQ-023, backlog M-3*

PQC 依存クレート (ml-kem, ml-dsa) の Git 参照を `branch` から `rev` に固定し、ビルドの再現性を確保する。

**手順**:

1. フォークリポジトリの HEAD コミットハッシュを取得する:
   ```bash
   git ls-remote https://github.com/sudolifeagain/ml-kem refs/heads/pq-diary
   git ls-remote https://github.com/sudolifeagain/ml-dsa refs/heads/pq-diary
   ```

2. core/Cargo.toml を更新する:

   変更前:
   ```toml
   ml-kem = { git = "https://github.com/sudolifeagain/ml-kem", branch = "pq-diary", ... }
   ml-dsa = { git = "https://github.com/sudolifeagain/ml-dsa", branch = "pq-diary", ... }
   ```

   変更後:
   ```toml
   ml-kem = { git = "https://github.com/sudolifeagain/ml-kem", rev = "<commit_hash>", ... }
   ml-dsa = { git = "https://github.com/sudolifeagain/ml-dsa", rev = "<commit_hash>", ... }
   ```

3. `cargo build` で正常にビルドできることを確認する

4. `Cargo.lock` が更新された場合はコミットに含める

### 3. M-5: verify_hmac → Result<bool, DiaryError> (core/src/crypto/hmac_util.rs)

**信頼性**: 🔵 *REQ-024, backlog M-5*

`verify_hmac` 関数の戻り値を `bool` から `Result<bool, DiaryError>` に変更し、鍵生成エラーを呼び出し元に適切に伝搬する。

変更前:
```rust
pub fn verify_hmac(key: &[u8], data: &[u8], expected: &[u8]) -> bool {
    let Ok(mac) = HmacSha256::new_from_slice(key) else {
        return false;
    };
    // ...
    mac.verify_slice(expected).is_ok()
}
```

変更後:
```rust
pub fn verify_hmac(key: &[u8], data: &[u8], expected: &[u8]) -> Result<bool, DiaryError> {
    let mac = HmacSha256::new_from_slice(key)
        .map_err(|e| DiaryError::Crypto(format!("HMAC key error: {}", e)))?;
    // ...
    Ok(mac.verify_slice(expected).is_ok())
}
```

**呼び出し元の更新**:
- 現在の本番コードでの直接呼び出しは0件（将来の利用に備えた関数）
- テストコードでの呼び出しを `Result` 型に対応させる（`.unwrap()` の追加またはアサーション変更）

## 注意事項

- M-1: `std::process::exit(1)` はテスト実行時にテストプロセス全体を終了させるため、現在の実装ではテストが困難である。`bail!` への変更によりテスタビリティが向上する
- M-3: `git ls-remote` の実行にはネットワークアクセスが必要。オフライン環境では事前にコミットハッシュを確認しておくこと
- M-3: Cargo.lock の更新が発生するため、コミットに含めること
- M-5: `HmacSha256::new_from_slice` は HMAC-SHA256 では任意長の鍵を受け付けるため、実際にはエラーパスに到達しにくい。しかし API の堅牢性のためにエラーハンドリングを実装する
- 3つの修正は独立しているため、レビュー時に個別に評価可能

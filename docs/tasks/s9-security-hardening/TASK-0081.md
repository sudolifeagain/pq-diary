## 実装詳細

### 1. H-1: Cli.password SecretString 化 (cli/src/main.rs, cli/src/password.rs)

**信頼性**: 🔵 *REQ-020, backlog H-1*

CLI の `password` フィールドを `secrecy::SecretString` に変更し、パスワードがメモリ上に平文で残留することを防止する。

#### 1a. cli/Cargo.toml への secrecy 依存追加

**信頼性**: 🔵

`secrecy` クレートが cli/Cargo.toml に存在しない場合、追加する:

```toml
[dependencies]
secrecy = { version = "0.10", features = ["serde"] }
```

注: core クレートが既に `secrecy` を使用している場合、バージョンを合わせること。

#### 1b. カスタム clap value parser の実装

**信頼性**: 🔵

clap は `SecretString` をネイティブにサポートしないため、カスタム value parser を実装する:

```rust
fn parse_secret_string(s: &str) -> Result<SecretString, String> {
    Ok(SecretString::from(s.to_string()))
}
```

#### 1c. Cli 構造体の password フィールド変更

**信頼性**: 🔵

```rust
// 変更前
#[arg(short, long)]
pub password: Option<String>,

// 変更後
#[arg(short, long, value_parser = parse_secret_string)]
pub password: Option<SecretString>,
```

#### 1d. get_password() のシグネチャ変更 (cli/src/password.rs)

**信頼性**: 🔵

```rust
// 変更前
pub fn get_password(cli_password: Option<&str>) -> anyhow::Result<String>

// 変更後
pub fn get_password(cli_password: Option<&SecretString>) -> anyhow::Result<String>
```

内部実装:
- `cli_password` が `Some(secret)` の場合: `secret.expose_secret().to_string()` で平文を取得する
- `None` の場合: 従来どおりインタラクティブ入力から取得する

注: `get_password` の戻り値は最終的に `SecretString` にすべきだが、下流の `core` API が `&str` を受け取るため、現時点では `String` のまま維持する。完全な SecretString 化は Phase 2 以降で検討する。

#### 1e. 全 cmd_* 関数の呼び出し箇所更新

**信頼性**: 🔵

各コマンドハンドラで `cli.password` を使用している箇所を更新する:

```rust
// 変更前
let password = get_password(cli.password.as_deref())?;

// 変更後
let password = get_password(cli.password.as_ref())?;
```

**影響範囲**: 全 `cmd_*` 関数のうちパスワードを使用するもの:
- `cmd_new`, `cmd_edit`, `cmd_list`, `cmd_show`, `cmd_delete`, `cmd_search`, `cmd_stats`, `cmd_import`, `cmd_template_*`
- その他パスワードを参照するコマンドハンドラ

### 2. M-2/M-4: Zeroizing 中間バッファ (core/src/crypto/mod.rs)

**信頼性**: 🔵 *REQ-022, backlog M-2/M-4*

KEM/DSA 秘密鍵を `BoxedSlice` に変換する際の中間 `Vec<u8>` バッファを `Zeroizing` でラップし、スコープ終了時に自動ゼロ化する。

#### 2a. KEM 秘密鍵の中間バッファ (core/src/crypto/mod.rs 付近)

**信頼性**: 🔵

```rust
// 変更前 (概略)
let kem_sk_bytes = kem_sk.as_ref().to_vec().into_boxed_slice();

// 変更後
let kem_sk_bytes = {
    let tmp = Zeroizing::new(kem_sk.as_ref().to_vec());
    let mut boxed = vec![0u8; tmp.len()].into_boxed_slice();
    boxed.copy_from_slice(&tmp);
    boxed
};
```

代替アプローチ（よりシンプル）:
```rust
let kem_sk_bytes = {
    let mut buf = Zeroizing::new(vec![0u8; kem_sk.as_ref().len()]);
    buf.copy_from_slice(kem_sk.as_ref());
    buf.to_vec().into_boxed_slice()
};
```

注: `Zeroizing<Vec<u8>>` は `Drop` 時にゼロ化されるため、`buf` がスコープを抜ける際に中間バッファが安全に消去される。最終的な `boxed_slice` は `SecretBytes` 等で保護される前提。

#### 2b. DSA 秘密鍵の中間バッファ (core/src/crypto/mod.rs 付近)

**信頼性**: 🔵

KEM 秘密鍵と同一パターンで DSA 秘密鍵の中間バッファもラップする:

```rust
// 変更前 (概略)
let dsa_sk_bytes = dsa_sk.as_ref().to_vec().into_boxed_slice();

// 変更後
let dsa_sk_bytes = {
    let tmp = Zeroizing::new(dsa_sk.as_ref().to_vec());
    let mut boxed = vec![0u8; tmp.len()].into_boxed_slice();
    boxed.copy_from_slice(&tmp);
    boxed
};
```

#### 2c. Zeroizing インポートの確認

**信頼性**: 🔵

`use zeroize::Zeroizing;` が core/src/crypto/mod.rs に存在することを確認する。存在しない場合は追加する。

## 注意事項

- H-1 は影響範囲が広い（全 `cmd_*` 関数でパスワード参照箇所を更新する必要がある）ため、変更漏れに注意すること
- `secrecy` クレートのバージョンは core クレートで使用しているバージョンと統一すること（ワークスペース内でバージョン不一致があると型の互換性問題が発生する）
- clap は `SecretString` の `FromStr` 実装を検出しないため、`value_parser` 属性の明示的な指定が必須
- `SecretString` は `Clone` を実装しているが、不必要な `clone()` はセキュリティポリシーに反するため最小限に留めること
- M-2/M-4: `Zeroizing::new(vec.to_vec())` は新しいアロケーションを作成するため、元の `to_vec()` によるバッファは依然としてゼロ化されない。最適解は `as_ref()` から直接 `Zeroizing<Vec<u8>>` にコピーし、`to_vec()` を経由しないこと
- `get_password` の戻り値を将来的に `SecretString` に変更する場合、core API のシグネチャ変更が必要になる。この変更はスコープ外とし、backlog に記録すること

## タスク概要

entry.rs のエントリ読み取り時に content_hmac と ML-DSA署名を検証する機能を追加する。現在エントリは読み取り時に整合性チェックが行われておらず、改竄検知ができない。本タスクでは get_entry、list_entries_with_body、search の各関数で HMAC および署名検証を実施し、改竄されたエントリに対して明確なエラーを返すようにする。

## 実装詳細

### 1. get_entry での HMAC 検証 (core/src/entry.rs ~line 378)

**信頼性**: 🔵 *REQ-026, REQ-027*

get_entry 関数内で暗号文を復号する前に、content_hmac の整合性検証を実施する:

- 復号前に暗号文に対して `hmac_util::compute()` で HMAC を計算する
- 計算した HMAC と格納済みの `record.content_hmac` を定数時間比較する
- 不一致の場合は `DiaryError::Crypto("content HMAC verification failed for entry {uuid}".into())` を返す
- content_hmac は暗号文（平文ではない）に対して計算されるため、復号前に検証可能

```rust
// HMAC 検証 (復号前)
let computed_hmac = hmac_util::compute(&ciphertext, &sym_key)?;
if !constant_time_eq(&computed_hmac, &record.content_hmac) {
    return Err(DiaryError::Crypto(
        format!("content HMAC verification failed for entry {}", uuid)
    ));
}
```

### 2. list_entries_with_body での HMAC 検証 (core/src/entry.rs ~line 254)

**信頼性**: 🔵 *REQ-026*

list_entries_with_body 関数内で各エントリの復号前に content_hmac を検証する:

- get_entry と同じパターンで content_hmac を検証する
- 全エントリに対して HMAC + 署名検証を実施する
- list_entries (メタデータのみ、復号なし) では検証をスキップする

### 3. ML-DSA 署名検証 (core/src/entry.rs)

**信頼性**: 🔵 *REQ-025, REQ-027*

HMAC 検証通過後に ML-DSA-65 署名を検証する:

- `engine.dsa_verify(ciphertext, signature)` を呼び出す
- 検証失敗時は `DiaryError::Crypto("signature verification failed for entry {uuid}".into())` を返す
- CryptoEngine に dsa_verify() メソッドが存在しない場合は追加/確認が必要

```rust
// 署名検証 (HMAC 検証後)
if !engine.dsa_verify(&ciphertext, &record.signature)? {
    return Err(DiaryError::Crypto(
        format!("signature verification failed for entry {}", uuid)
    ));
}
```

### 4. 選択的検証

**信頼性**: 🔵 *REQ-025, REQ-026*

各関数での検証ポリシーを以下のように定義する:

| 関数 | 復号 | HMAC 検証 | 署名検証 |
|------|------|-----------|----------|
| list_entries (メタデータのみ) | なし | スキップ | スキップ |
| get_entry | あり | 実施 | 実施 |
| list_entries_with_body | あり | 実施 | 実施 |
| search (ストリーム復号) | あり | 実施 | スキップ可 |

- メタデータのみの一覧表示 (list_entries) では復号しないため、検証不要
- search ではパフォーマンスを考慮し、HMAC 検証はエントリごとに実施する

### 5. エラーハンドリング

**信頼性**: 🔵 *REQ-027*

検証失敗時のエラーハンドリングポリシー:

- 検証失敗は明確なエラーとして返す。サイレントスキップは禁止
- エラーメッセージに対象エントリの UUID を含め、どのエントリが改竄されたか特定可能にする
- HMAC 不一致: `"content HMAC verification failed for entry {uuid}"`
- 署名不正: `"signature verification failed for entry {uuid}"`
- ユーザーは改竄の可能性を認識できる

## 注意事項

- CryptoEngine.dsa_verify() メソッドが存在しない場合は追加/確認が必要
- content_hmac は暗号文（平文ではない）に対して計算されるため、復号前に検証可能
- これは重要なセキュリティ修正: 現在エントリは読み取り時に整合性チェックが行われていない
- 既存の `DiaryError::Crypto(String)` バリアントを使用すること。新しいエラーバリアントを追加しないこと
- 定数時間比較を使用してタイミング攻撃を防止すること

## タスク概要

cli/src/main.rs のプロセス開始時にセキュリティ硬化を適用する。Unix では PR_SET_DUMPABLE=0 によるプロセスメモリアクセス防止、RLIMIT_CORE=0 によるコアダンプ無効化、/proc/self/status による TracerPid チェックを行う。Windows では IsDebuggerPresent によるデバッガ検知を行う。全失敗はワーニングのみとし、プロセス継続を保証する。

## 実装詳細

### 1. harden_process() 関数 (cli/src/security.rs 新規作成)

**信頼性**: 🔵 *REQ-010, REQ-011*

プロセスレベルのセキュリティ硬化を適用する関数を実装する:

```rust
pub fn harden_process() {
    #[cfg(unix)]
    {
        // PR_SET_DUMPABLE=0: /proc/pid/mem アクセスと ptrace を防止
        if let Err(e) = nix::sys::prctl::set_dumpable(false) {
            eprintln!("Warning: PR_SET_DUMPABLE failed: {e}");
        }
        // RLIMIT_CORE=0: コアダンプを無効化
        use nix::sys::resource::{setrlimit, Resource};
        if let Err(e) = setrlimit(Resource::RLIMIT_CORE, 0, 0) {
            eprintln!("Warning: setrlimit(RLIMIT_CORE, 0) failed: {e}");
        }
    }
}
```

- PR_SET_DUMPABLE=0 により、他プロセスからの /proc/pid/mem 読み取りと ptrace アタッチを防止する
- RLIMIT_CORE=0 により、プロセスクラッシュ時のコアダンプファイル生成を抑止し、鍵素材の漏洩を防ぐ
- Windows ではこれらの概念が存在しないため、`#[cfg(unix)]` ガードで囲む
- 全エラーは `eprintln!` でワーニング出力のみ行い、処理を継続する

### 2. check_debugger() 関数

**信頼性**: 🔵 *REQ-012, REQ-013*

デバッガのアタッチ状態を検知し、ユーザーに警告する関数を実装する:

```rust
pub fn check_debugger() {
    #[cfg(unix)]
    {
        // /proc/self/status を読み取り、TracerPid を確認
        if let Ok(status) = std::fs::read_to_string("/proc/self/status") {
            for line in status.lines() {
                if line.starts_with("TracerPid:") {
                    let pid: u32 = line.split(':')
                        .nth(1)
                        .unwrap_or("0")
                        .trim()
                        .parse()
                        .unwrap_or(0);
                    if pid != 0 {
                        eprintln!("Warning: debugger detected (TracerPid: {pid}). Key material may be exposed.");
                    }
                }
            }
        }
    }
    #[cfg(windows)]
    {
        let attached = unsafe {
            windows_sys::Win32::System::Diagnostics::Debug::IsDebuggerPresent()
        };
        if attached != 0 {
            eprintln!("Warning: debugger detected. Key material may be exposed.");
        }
    }
}
```

- Unix: /proc/self/status ファイルの TracerPid フィールドが 0 以外の場合、デバッガがアタッチされていると判定する
- Windows: Win32 API の IsDebuggerPresent() が非ゼロを返す場合、デバッガがアタッチされていると判定する
- デバッガ検知時はワーニングメッセージを表示するのみで、プロセスは終了しない（REQ-013）

## 注意事項

- `nix::sys::prctl` が使用中の nix バージョンに存在しない場合は、代替として `libc::prctl(PR_SET_DUMPABLE, 0)` を使用すること
- Windows では PR_SET_DUMPABLE と RLIMIT_CORE に相当する機能がないため、`#[cfg(unix)]` で適切にガードすること
- /proc/self/status は Linux 固有のファイルであり、macOS 等他の Unix 系 OS では存在しない場合がある。`if let Ok(...)` パターンで安全にハンドリングすること
- check_debugger() 内の `unwrap_or("0")` と `unwrap_or(0)` はパース失敗時のフォールバックであり、本番コードでの unwrap 禁止ルールには抵触しない
- 全失敗はワーニングのみとし、プロセスを終了させないこと（REQ-013 準拠）
- `unsafe` ブロックは IsDebuggerPresent の Win32 API 呼び出しのみに使用すること（CLAUDE.md の unsafe 許可範囲内）

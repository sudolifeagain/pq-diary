## タスク概要

vault reader/writer の堅牢性を向上させる。reader では可変長フィールド読み取り時に 16MiB 上限チェックを追加し、悪意ある/破損した vault ファイルからの防御を強化する。writer では整数キャストを安全な `try_from` に置換し、書き込みをアトミック化して中断時のデータ損失を防ぐ。

## 実装詳細

### 1. reader フィールドサイズ上限チェック (core/src/vault/reader.rs)

`read_header()` と `parse_entry_payload()` で各可変長フィールド読み取り前に以下のチェックを追加する:

```rust
if len > 16 * 1024 * 1024 {
    return Err(DiaryError::Vault("field size exceeds 16MiB maximum"));
}
```

対象フィールド:
- `kem_encrypted_sk`
- `dsa_encrypted_sk`
- `ciphertext`
- `signature`
- `legacy_key_block`
- `record_len`

### 2. writer 安全キャスト (core/src/vault/writer.rs)

全ての `as u32` / `as u8` キャストを安全な変換に置換する:

```rust
// Before
let len = data.len() as u32;

// After
let len = u32::try_from(data.len())
    .map_err(|_| DiaryError::Vault("field length exceeds u32 maximum"))?;
```

### 3. writer アトミック書き込み (core/src/vault/writer.rs)

`write_vault()` をアトミック化する:

1. 同一ディレクトリに一時ファイルを作成 (例: `vault.pqd.tmp`)
2. 一時ファイルに全データを書き込み
3. `sync_all()` でディスクにフラッシュ
4. `std::fs::rename()` で元ファイルにアトミックに置換
5. エラー時は一時ファイルを削除

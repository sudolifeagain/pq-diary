## タスク概要

暗号処理とエントリデータのセキュリティを強化する。空パスワードの拒否、HMAC計算のエラーハンドリング改善、エントリ平文データの確実なゼロ化を実装する。秘密データがメモリ上に残留するリスクを排除し、CLAUDE.md の Rust 規約（秘密データは必ず zeroize / SecretString / SecretBytes で保持）に準拠させる。

## 実装詳細

### 1. 空パスワード拒否 (core/src/vault/init.rs)

`init_vault()` の冒頭に空パスワードチェックを追加する:

```rust
if password.is_empty() {
    return Err(DiaryError::Password("password must not be empty"));
}
```

### 2. HMAC エラーハンドリング改善 (core/src/crypto/hmac_util.rs)

`compute()` の戻り値を `Result<[u8; 32], DiaryError>` に変更する:

```rust
// Before
pub fn compute(key: &[u8], data: &[u8]) -> [u8; 32] { ... }

// After
pub fn compute(key: &[u8], data: &[u8]) -> Result<[u8; 32], DiaryError> { ... }
```

全ての呼び出し側を `?` 演算子に修正する。

### 3. EntryPlaintext ゼロ化 (core/src/entry.rs)

`EntryPlaintext` 構造体に `Zeroize` と `ZeroizeOnDrop` を derive する:

```rust
#[derive(Debug, Clone, Zeroize, ZeroizeOnDrop)]
pub struct EntryPlaintext {
    // ...
}
```

注意: フィールドの型が全て `Zeroize` を実装していることを確認する。`chrono::DateTime` など非対応の型がある場合は、`String` として保持し必要に応じて変換するか、手動で `Zeroize` を実装する。

### 4. list_entries_with_body ラップ (core/src/lib.rs)

`list_entries_with_body()` の戻り値を `Zeroizing` でラップする:

```rust
pub fn list_entries_with_body(&self) -> Result<Zeroizing<Vec<EntryPlaintext>>> { ... }
```

### テスト詳細

**TC-A03-01**: 空文字列のパスワードで `init_vault()` を呼び出し、`DiaryError::Password` が返ることを検証する。

**TC-A03-02**: 非空のパスワードで `init_vault()` を呼び出し、正常に vault が作成されることを検証する。既存テストとの整合性も確認。

**TC-A09-01**: `hmac::compute()` が `Result<[u8; 32], DiaryError>` を返すことを型レベルで検証する。正常ケースで `Ok(...)` が返ること、および（可能であれば）異常ケースで `Err(...)` が返ることを確認する。

**TC-A10-01**: `EntryPlaintext` を作成し、`ManuallyDrop` でラップした上で手動で drop を呼び、フィールドのメモリ内容がゼロ化されていることを検証する。既存の zeroize テストパターン（S4 で実装済み）に準拠する。

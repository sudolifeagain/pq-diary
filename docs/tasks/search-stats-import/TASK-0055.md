## タスク概要

CLI のエディタ起動とパスワード取得におけるセキュリティを修正する。vim/nvim 起動時にバックアップファイルやヒストリファイルの生成を抑止し、環境変数からのパスワード読み取り後に即座に環境変数を削除する。また Unix 環境では一時ファイルのパーミッションを制限する。

## 実装詳細

### 1. vim セキュリティオプション追加 (cli/src/editor.rs)

vim 起動時のオプションに以下を追加する:

```rust
// vim の場合
vim_options.push("nowritebackup");
vim_options.push("viminfo=''");

// nvim の場合
nvim_options.push("nowritebackup");
nvim_options.push("shada=''");
```

これにより以下を防止する:
- `nowritebackup`: バックアップファイル（`.swp` 以外の `~` ファイル）の生成を抑止
- `viminfo=''` / `shada=''`: ヒストリファイルへの情報漏洩を防止

### 2. 環境変数の即時削除 (cli/src/password.rs)

`PQ_DIARY_PASSWORD` 環境変数を読み取った直後に削除する:

```rust
if let Ok(password) = std::env::var("PQ_DIARY_PASSWORD") {
    std::env::remove_var("PQ_DIARY_PASSWORD");
    // password を SecretString に変換して返す
    return Ok(SecretString::from(password));
}
```

これにより、子プロセスやメモリダンプからの環境変数経由のパスワード漏洩リスクを低減する。

### 3. Unix 一時ファイル パーミッション制限 (cli/src/editor.rs)

Unix 環境で一時ファイルを作成する際に、パーミッションを owner のみに制限する:

```rust
#[cfg(unix)]
{
    use std::os::unix::fs::OpenOptionsExt;
    OpenOptions::new()
        .write(true)
        .create(true)
        .mode(0o600)
        .open(&tmp_path)?;
}
```

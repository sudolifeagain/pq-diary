## 実装詳細

### 1. ADR 作成 (docs/adr/0007-win32-console-unsafe.md)

以下の構成で ADR を作成する:

```markdown
# ADR-0007: Win32 Console API の unsafe 使用許可

## ステータス
提案

## コンテキスト
- pq-diary はパスワード入力時にエコーバックを無効化する必要がある
- Unix では termios API を使用 (nix クレート経由で safe API)
- Windows では Win32 Console API を直接呼び出す必要がある
- 対象 API: GetConsoleMode, SetConsoleMode, ReadConsoleW

## 決定
CLAUDE.md の unsafe 許可リストに Win32 Console API を追加する。

## 理由
1. rpassword クレート不採用理由:
   - 追加の依存関係 (依存関係最小化ポリシー)
   - SecretString との統合が不十分
   - zeroize 保証がない
2. windows-sys クレート経由の FFI 呼び出しは本質的に unsafe
3. 呼び出しパターンが限定的 (3関数のみ) でレビュー可能

## SAFETY コメント確認基準
- 各 unsafe ブロックに SAFETY コメントが必須
- ハンドル取得の有効性チェック
- エラー時のコンソールモード復元保証
- バッファサイズの境界チェック

## 代替案
1. rpassword クレート → 不採用 (上記理由)
2. dialoguer クレート → 不採用 (過大な依存関係)
3. 標準入力の raw mode → 不採用 (Windows では不十分)
```

### 2. CLAUDE.md 更新

CLAUDE.md の unsafe 許可リスト行を更新する:

```markdown
# Before
- `unsafe` は mlock/VirtualLock/PR_SET_DUMPABLE のみ許可。それ以外は禁止

# After

```

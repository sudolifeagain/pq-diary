## 実装詳細

### 1. Stats サブコマンド定義 (cli/src/main.rs)

**信頼性**: 🔵 *REQ-C01: pq-diary stats*

clap derive で Stats サブコマンドを追加:

```rust
/// Show vault statistics.
#[derive(Args)]
struct StatsArgs {
    /// Output as JSON.
    #[arg(long)]
    json: bool,

    /// Show ASCII heatmap of writing activity.
    #[arg(long)]
    heatmap: bool,
}
```

`Command` enum に `Stats(StatsArgs)` バリアントを追加する。

### 2. cmd_stats() コマンドハンドラ (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-C03, C04, C05 + 設計文書 VaultGuard パターン*

```rust
pub fn cmd_stats(cli: &Cli, args: &StatsArgs) -> anyhow::Result<()>
```

処理フロー:
1. `get_password()` でパスワード取得
2. `DiaryCore::new()` + `unlock()`
3. `VaultGuard` 取得
4. `core.stats()?` で `VaultStats` 取得
5. 出力モード分岐:
   - `--json` → JSON 出力
   - `--heatmap` → ヒートマップ出力
   - デフォルト → テキスト出力
6. `VaultGuard` drop で自動 lock

### 3. テキスト出力 (デフォルト)

**信頼性**: 🔵 *REQ-C03: 人間が読みやすいフォーマット*

```
=== Vault Statistics ===

Entries:        {entry_count}
Tags:           {tag_count}
First entry:    {first_entry_date (YYYY-MM-DD)}
Last entry:     {last_entry_date (YYYY-MM-DD)}
Active days (30d): {active_days_30d}

Characters:
  Total:        {char_stats.total}
  Average:      {char_stats.average}
  Maximum:      {char_stats.max}

Top Tags:
  1. {tag} ({count})
  2. {tag} ({count})
  ...
```

### 4. --json 出力

**信頼性**: 🔵 *REQ-C04: JSON 形式で stdout に出力*

```rust
if args.json {
    let json = serde_json::to_string_pretty(&stats)?;
    println!("{}", json);
    return Ok(());
}
```

`VaultStats` は `Serialize` を derive しているため、直接 `serde_json::to_string_pretty()` で出力可能。

### 5. --heatmap 出力

**信頼性**: 🔵 *REQ-C05: 直近1年の日別執筆頻度を ASCII ヒートマップ*

`render_heatmap()` 関数を `cli/src/commands.rs` 内に実装する（表示ロジックは CLI 層の責務）:

```rust
fn render_heatmap(daily_activity: &[DailyActivity]) -> String
```

処理:
1. 今日から52週前 (364日前) までの日付範囲を生成
2. 各日について `daily_activity` から count を取得（存在しない日は 0）
3. count に応じて文字を割り当て:
   - 0件: `░`
   - 1件: `▒`
   - 2-3件: `▓`
   - 4件以上: `█`
4. 7行 (曜日) x 52列 (週) のグリッドとして表示
5. 月ラベルをヘッダに表示

表示例:
```
         Jan  Feb  Mar  Apr  May  Jun  Jul  Aug  Sep  Oct  Nov  Dec
Mon  ░░▒▓░░░░▒▒▓▓░░░░░▒░░░░░░░░░▒▒░░░░░▒░░░░░░░░░░░░░░
Tue  ░▒▓▓▒░░░░▒▒░░░░░▒▒▒░░░░▒░░░░░░░░░░░▒░░░░░░░░░░░░░
...
Legend: ░ 0  ▒ 1  ▓ 2-3  █ 4+
```

端末幅80文字以上で正しく表示されることを確認する。

### 6. 空 vault の場合

**信頼性**: 🟡 *EDGE-102: エントリ数: 0 のみ表示*

`entry_count` が 0 の場合でもエラーにせず、基本情報（Entries: 0）を表示する。日付やタグは "N/A" と表示する。

### テスト詳細

**TC-C03-01**: vault にエントリを作成し、`cmd_stats()` をデフォルト（フラグなし）で呼び出して stdout をキャプチャし、"Vault Statistics" ヘッダと "Entries:" ラベルが含まれることを検証する。

**TC-C03-02**: テキスト出力に entry_count, tag_count, first_entry_date, last_entry_date, active_days_30d, Characters (Total/Average/Maximum), Top Tags が含まれることを検証する。

**TC-C04-01**: `--json` フラグで `cmd_stats()` を呼び出し、stdout キャプチャした文字列を `serde_json::from_str::<serde_json::Value>()` でパースしてエラーにならないことを検証する。

**TC-C04-02**: JSON 出力をパースし、"entry_count", "tag_count", "char_stats", "tag_distribution", "daily_activity" キーが存在することを検証する。

**TC-C05-01**: エントリを複数日にわたって作成し、`--heatmap` で呼び出して出力に `░`, `▒`, `▓`, `█` のいずれかの文字が含まれることを検証する。

**TC-C05-02**: ヒートマップ出力の末尾に "Legend:" 行が含まれることを検証する。

**TC-EDGE-01**: エントリ0件の vault に対して `cmd_stats()` を呼び出し、エラーにならず "Entries:" に "0" が表示されることを検証する。

## タスク概要

Markdown ファイルのパーサを実装する。YAML frontmatter からの title/tags 抽出、Obsidian 互換の `[[wiki-link|alias]]` 変換、`#ネスト/タグ` 抽出と本文からの除去を行う。パース結果は `MarkdownFile` 構造体（`Zeroize` derive）として返却し、読み込みバッファは `Zeroizing<String>` で保持する。

### 2. importer モジュール新規作成 (core/src/importer.rs)

**信頼性**: 🔵 *REQ-D02〜D06 + types.rs 参照*

以下の型を定義する:

- **`ImportSource`**: インポートソース設定
  - `directory: PathBuf` — ソースディレクトリパス
  - `dry_run: bool` — プレビューモードフラグ

- **`MarkdownFile`**: パース済み MD ファイル (`Zeroize`, `ZeroizeOnDrop` derive)
  - `title: String` — エントリタイトル
  - `tags: Vec<String>` — 抽出タグ一覧
  - `body: String` — 変換済み本文
  - `source_path: String` — 元ファイルパス

- **`ImportResult`**: インポート結果サマリー
  - `imported: usize` — インポート成功件数
  - `skipped: usize` — スキップ件数
  - `links_converted: usize` — wiki-link 変換数
  - `tags_converted: usize` — タグ抽出数
  - `skip_details: Vec<SkipDetail>` — スキップ詳細

- **`SkipDetail`**: スキップファイル詳細
  - `path: String` — ファイルパス
  - `reason: String` — スキップ理由

### 3. parse_markdown() 関数

**信頼性**: 🔵 *REQ-D03〜D06*

```rust
pub fn parse_markdown(
    content: &str,
    filename: &str,
) -> Result<(MarkdownFile, usize, usize), DiaryError>
```

戻り値: `(MarkdownFile, links_converted_count, tags_converted_count)`

処理フロー:

#### a. YAML frontmatter パース

**信頼性**: 🟡 *Obsidian/Hugo 慣例から推測*

1. コンテンツ先頭が `---\n` で始まるか確認
2. 2つ目の `---\n` までをfrontmatter ブロックとして抽出
3. 手動パース（serde_yaml 依存を避ける）:
   - `title:` 行から title を抽出
   - `tags:` 行から tags を抽出（`[tag1, tag2]` 形式 / `\n- tag1\n- tag2` 形式の両方対応）
4. frontmatter ブロック後の残りを body とする

#### b. タイトル決定

**信頼性**: 🔵 *REQ-D03: frontmatter.title || filename_stem*

```rust
let title = frontmatter_title.unwrap_or_else(|| {
    Path::new(filename).file_stem().unwrap_or_default().to_string_lossy().to_string()
});
```

#### c. wiki-link 変換 (convert_wiki_links)

**信頼性**: 🔵 *REQ-D04 + ADR-0004*

```rust
fn convert_wiki_links(body: &str) -> (String, usize)
```

regex: `\[\[([^\]|]+)(?:\|[^\]]+)?\]\]`
- `[[My Note]]` → `[[My Note]]` (変換不要)
- `[[Other|alias]]` → `[[Other]]` (エイリアス除去)
- 戻り値: (変換後 body, 変換回数)

#### d. タグ抽出 (extract_tags)

**信頼性**: 🔵 *REQ-D05 + backlog.md「#ネスト/タグ 自動変換」*

```rust
fn extract_tags(body: &str) -> (Vec<String>, String, usize)
```

regex: `#([\w/]+)` (Unicode word chars + `/`)
- `#work/project` → Tag "work/project"
- `#personal` → Tag "personal"
- 本文から `#tag` 表記を除去
- 戻り値: (抽出タグ一覧, タグ除去後 body, 抽出回数)

#### e. タグ統合

**信頼性**: 🟡 *REQ-D06: frontmatter tags + extracted tags*

```rust
let mut all_tags = frontmatter_tags;
for tag in extracted_tags {
    if !all_tags.contains(&tag) {
        all_tags.push(tag);
    }
}
```

重複を除去して統合する。

### 4. .obsidian/ スキップ判定

**信頼性**: 🔵 *REQ-D08: .obsidian/ ディレクトリをスキップ*

```rust
pub fn should_skip_path(path: &Path) -> bool {
    path.components().any(|c| c.as_os_str() == ".obsidian")
}
```

ディレクトリ走査時（TASK-0063 で使用）のフィルタ関数として提供する。

### 5. 読み込みバッファの Zeroizing ラップ

**信頼性**: 🔵 *REQ-D11: 読み込みバッファは使用後に zeroize*

ファイル内容を読み込む際は `Zeroizing<String>` で保持し、`parse_markdown()` 完了後に自動 zeroize される:

```rust
let content = Zeroizing::new(std::fs::read_to_string(path)?);
let (md_file, links, tags) = parse_markdown(&content, filename)?;
// content は drop 時に zeroize
```

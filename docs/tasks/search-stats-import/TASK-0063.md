## タスク概要

Markdown パーサ (TASK-0062) の上に、ディレクトリ再帰走査とバッチ書き込み機能を実装する。walkdir で `.md` ファイルを収集し、各ファイルを `parse_markdown()` でパースした後、全エントリを一括で暗号化・署名して `write_vault()` を1回のみ呼び出す。dry-run モードではプレビューのみで書き込みを行わない。

## 実装詳細

### 1. batch_create_entries() 関数 (core/src/importer.rs)

**信頼性**: 🔵 *REQ-D10: バッチ書き込みで write_vault 1回*

```rust
pub fn batch_create_entries(
    vault_path: &Path,
    engine: &CryptoEngine,
    files: Vec<MarkdownFile>,
) -> Result<ImportResult, DiaryError>
```

処理フロー:
1. `read_vault(vault_path)` → `(header, existing_records)`
2. 各 `MarkdownFile` について:
   a. UUID v4 を生成
   b. `EntryPlaintext { id, title, body, tags, created_at, updated_at }` を構築
   c. `serde_json::to_vec()` → JSON バイト列
   d. `engine.encrypt(json)` → (iv, ciphertext)
   e. `engine.sign(ciphertext)` → signature
   f. HMAC 計算
   g. `EntryRecord` を構築 → `existing_records` に追加
3. `write_vault(vault_path, header, all_records)` — **1回のみ**
4. `ImportResult` を構築して返却

### 2. import_directory() 関数 (core/src/importer.rs)

**信頼性**: 🔵 *REQ-D01, REQ-D02: ディレクトリ再帰走査*

```rust
pub fn import_directory(
    vault_path: &Path,
    engine: &CryptoEngine,
    source_dir: &Path,
    dry_run: bool,
) -> Result<ImportResult, DiaryError>
```

処理フロー:
1. `source_dir` の存在チェック（存在しなければ `DiaryError::Import` を返す）
2. `walkdir::WalkDir::new(source_dir)` で再帰走査:
   a. `.obsidian/` パス → スキップ（`should_skip_path()` 使用）
   b. 拡張子が `.md` でないファイル → スキップ
   c. ディレクトリ → スキップ
   d. `.md` ファイル:
      - `Zeroizing::new(std::fs::read_to_string(path)?)` で読み込み
      - `parse_markdown(&content, filename)?` でパース
      - `MarkdownFile` を `Vec<MarkdownFile>` に追加
      - 累計の `links_converted`, `tags_converted` を加算
3. `.md` ファイルが0件の場合、`DiaryError::Import("No Markdown files found")` を返す
4. `dry_run == true` の場合:
   - 書き込みをスキップ
   - `ImportResult { imported: 0, skipped, links_converted, tags_converted, ... }` を返す
5. `dry_run == false` の場合:
   - `batch_create_entries(vault_path, engine, parsed_files)?`
   - `ImportResult` を返す

### 3. DiaryCore::import() メソッド (core/src/lib.rs)

**信頼性**: 🔵 *設計文書 architecture.md: Facade パターン*

```rust
impl DiaryCore {
    pub fn import(
        &self,
        source_dir: &Path,
        dry_run: bool,
    ) -> Result<ImportResult, DiaryError> {
        importer::import_directory(&self.vault_path, &self.engine, source_dir, dry_run)
    }
}
```

### 4. 元ファイル削除禁止

**信頼性**: 🟡 *REQ-D12: import は元ファイルを削除してはならない*

`import_directory()` は `std::fs::read_to_string()` で読み取り専用アクセスのみを行い、元ファイルに対する `write`, `delete`, `rename` 操作を一切行わない。

### テスト詳細

**TC-D01-01**: テスト用ディレクトリに `note1.md` を配置し、`import_directory()` を呼び出してインポート後に `read_vault()` でエントリが1件追加されていることを検証する。エントリのタイトルが "note1" であることも確認する。

**TC-D01-02**: テスト用ディレクトリに `root.md` と `sub/nested.md` を配置し、`import_directory()` を呼び出してインポート結果の `imported` が 2 であることを検証する。サブディレクトリ内のファイルも取り込まれることの確認。

**TC-D01-05**: テスト用ディレクトリに `note.md`, `image.png`, `data.json` を配置し、`import_directory()` を呼び出してインポート結果の `imported` が 1（`note.md` のみ）であることを検証する。`skip_details` に `image.png` と `data.json` が含まれることも確認する。

**TC-D07-01**: テスト用ディレクトリに `note.md` を配置し、`import_directory(dry_run=true)` を呼び出す。呼び出し前後で vault ファイルのバイト内容が同一であること（ファイルハッシュ比較）、かつ `ImportResult.imported` が 0 であることを検証する。

**TC-D10-01**: テスト用ディレクトリに100個の `.md` ファイルを生成し、`import_directory()` を呼び出す。`ImportResult.imported` が 100 であることを検証する。vault ファイルの更新回数は直接検証が困難なため、`batch_create_entries()` 内の `write_vault()` 呼び出しが1回であることをテストの構造（100件一括書き込み後にread_vaultで100件存在確認）で間接的に検証する。

**TC-D11-01**: `MarkdownFile` 構造体が `Zeroize` trait を derive していることを型レベルで確認する。`parse_markdown()` に渡す `Zeroizing<String>` がスコープ終了後に zeroize されることを、drop 後のメモリアクセスではなく、型制約による静的保証で検証する。

**TC-D12-01**: テスト用ディレクトリに `note.md` を配置し、`import_directory()` を呼び出した後、元の `note.md` ファイルが依然として存在し、内容が変更されていないことをファイル内容比較で検証する。

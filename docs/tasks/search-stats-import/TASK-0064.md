## 実装詳細

### 1. Import サブコマンド定義 (cli/src/main.rs)

**信頼性**: 🔵 *REQ-D01: pq-diary import <DIR>*

clap derive で Import サブコマンドを追加:

```rust
/// Import Markdown files from a directory.
#[derive(Args)]
struct ImportArgs {
    /// Source directory containing .md files.
    dir: PathBuf,

    /// Preview import without writing to vault.
    #[arg(long)]
    dry_run: bool,
}
```

`Command` enum に `Import(ImportArgs)` バリアントを追加する。

### 2. cmd_import() コマンドハンドラ (cli/src/commands.rs)

**信頼性**: 🔵 *REQ-D01, D07, D09 + 設計文書 VaultGuard パターン*

```rust
pub fn cmd_import(cli: &Cli, args: &ImportArgs) -> anyhow::Result<()>
```

処理フロー:
1. `args.dir` の存在チェック:
   - 存在しない場合: `"Error: Directory '{}' does not exist"` を表示して終了
   - ディレクトリでない場合: `"Error: '{}' is not a directory"` を表示して終了
2. `get_password()` でパスワード取得
3. `DiaryCore::new()` + `unlock()`
4. `VaultGuard` 取得
5. `core.import(&args.dir, args.dry_run)?` を呼び出し
6. 結果表示（下記フォーマット）
7. `VaultGuard` drop で自動 lock

### 3. サマリー表示

**信頼性**: 🔵 *REQ-D09: インポート結果サマリー*

通常モード (dry_run=false):
```
Imported: {imported}, Skipped: {skipped}, Links converted: {links_converted}, Tags converted: {tags_converted}
```

dry_run モード (dry_run=true):
```
[DRY RUN] Would import {n} files from '{dir}'
  Links to convert: {links_converted}
  Tags to extract: {tags_converted}
  Files to skip: {skipped}
```

### 4. --dry-run プレビュー表示

**信頼性**: 🟡 *REQ-D07: 対象ファイル数と変換結果のプレビュー*

`dry_run=true` の場合、コアモジュールが書き込みなしで `ImportResult` を返すため、CLI 側では結果をプレビュー形式で表示する。スキップされたファイルがある場合は詳細も表示:

```
[DRY RUN] Would import 15 files from '/path/to/notes'
  Links to convert: 23
  Tags to extract: 45
  Files to skip: 3
    - image.png (not a .md file)
    - .obsidian/config.json (.obsidian directory)
    - binary.dat (not a .md file)
```

### 5. エラーハンドリング

**信頼性**: 🟡 *EDGE-002, EDGE-003*

- **存在しないディレクトリ**: `args.dir` のパスが存在しない場合、`DiaryCore` のコンストラクタを呼ぶ前にチェックして明確なエラーメッセージを表示
- **空ディレクトリ**: コアモジュールから `DiaryError::Import("No Markdown files found")` が返された場合、そのまま表示

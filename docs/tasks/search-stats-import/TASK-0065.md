## 実装詳細

### 1. 統合テスト (cli/tests/integration_test.rs)

**信頼性**: 🔵 *全要件の横断検証*

Sprint 6 用の統合テストを `cli/tests/integration_test.rs` に追加する。

#### TC-S6-01: search → list → search で結果一致

**信頼性**: 🔵 *REQ-B01, REQ-B03*

1. vault を初期化し、body に "unique_keyword_2026" を含むエントリを2件、含まないエントリを1件作成
2. `search "unique_keyword_2026"` を実行 → 結果が2件であることを確認
3. `list` を実行 → 全3件が表示されることを確認
4. 再度 `search "unique_keyword_2026"` → 結果が依然として2件（冪等性確認）

#### TC-S6-02: stats 基本統計 (空vault + エントリ追加後)

**信頼性**: 🔵 *REQ-C01, REQ-C02*

1. vault を初期化（エントリ0件）
2. `stats` を実行 → `entry_count` が 0 であることを確認
3. エントリを3件追加（異なるタグ、異なる body 長）
4. `stats` を実行 → `entry_count` が 3、タグ分布・文字数統計が正しいことを確認

#### TC-S6-03: import → list でインポート件数一致

**信頼性**: 🔵 *REQ-D01, REQ-D02*

1. vault を初期化
2. テスト用ディレクトリに `.md` ファイルを5件作成
3. `import <DIR>` を実行 → サマリーの `imported` が 5
4. `list` を実行 → エントリが5件表示されることを確認
5. エントリのタイトルがファイル名から正しく設定されていることを確認

#### TC-S6-04: import --dry-run で vault 未変更

**信頼性**: 🟡 *REQ-D07*

1. vault を初期化してエントリを1件作成
2. vault ファイルのバイト内容を記録
3. テスト用ディレクトリに `.md` ファイルを3件作成
4. `import --dry-run <DIR>` を実行 → プレビュー表示を確認
5. vault ファイルのバイト内容が変更されていないことを確認
6. `list` を実行 → エントリが依然として1件であることを確認

#### TC-S6-05: import wiki-link 変換 + tag 抽出

**信頼性**: 🔵 *REQ-D04, REQ-D05*

1. vault を初期化
2. テスト用 `.md` ファイルを作成:
   ```markdown
   ---
   title: Test Note
   tags: [diary]
   ---
   See [[Other Note|alias]] for details.
   Working on #work/project today.
   ```
3. `import <DIR>` を実行
4. `show <id>` でインポートされたエントリを表示
5. body に `[[Other Note]]` が含まれ（エイリアス除去済み）、`[[Other Note|alias]]` が含まれないことを確認
6. tags に "diary", "work/project" が含まれることを確認

#### TC-S6-06: stats --json が valid JSON

**信頼性**: 🔵 *REQ-C04*

1. vault を初期化してエントリを複数件作成
2. `stats --json` を実行 → stdout をキャプチャ
3. 出力を `serde_json::from_str::<serde_json::Value>()` でパースしてエラーにならないことを確認
4. JSON に "entry_count", "tag_count", "char_stats", "tag_distribution" キーが存在することを確認

### 2. doc comments 追加

**信頼性**: 🔵 *Rust ドキュメント規約*

Sprint 6 で追加した全新規公開型・関数に `///` doc comments を追加する。英語で記述する。

**対象ファイル**:

- **core/src/search.rs**: `SearchQuery`, `SearchMatch`, `ContextBlock`, `SearchResults`, `search_entries()`, `format_context()`
- **core/src/stats.rs**: `VaultStats`, `CharStats`, `TagCount`, `DailyActivity`, `collect_stats()`
- **core/src/importer.rs**: `ImportSource`, `MarkdownFile`, `ImportResult`, `SkipDetail`, `parse_markdown()`, `convert_wiki_links()`, `extract_tags()`, `should_skip_path()`, `batch_create_entries()`, `import_directory()`
- **core/src/lib.rs**: `DiaryCore::search()`, `DiaryCore::stats()`, `DiaryCore::import()`

**doc comment 規約**:
- 1行目: 型/関数の概要（英語、1文）
- 2行目以降: 詳細説明、引数 (`# Arguments`)、戻り値 (`# Returns`)、エラー (`# Errors`)
- 主要な公開APIには `# Examples` セクション
